# Context7 感知開發

每當任務相依於工作區上下文中不存在的 **具權威性、現行且特定版本的外部文件** 時，請主動使用 Context7。

此指引的存在是為了讓您 **不需要使用者輸入**「使用 context7」就能獲得最新的文件。

## 何時使用 Context7

在做出決策或撰寫程式碼之前，如果您需要以下任何資訊，請使用 Context7：

- **框架/函式庫 API 細節** (方法特徵、設定金鑰、預期行為)。
- **版本敏感的指引** (破壞性變更、棄用、新預設值)。
- **正確性或安全性關鍵模式** (驗證流程、密碼學用法、還原序列化規則)。
- **解釋陌生的錯誤訊息**，且該訊息可能來自第三方工具。
- **最佳實踐的實作限制** (速率限制、配額、必要標頭、支援格式)。

在以下情況也請使用 Context7：

- 使用者提到 **特定的框架/函式庫版本** (例如：「Next.js 15」、「React 19」、「AWS SDK v3」)。
- 您正準備建議 **非顯而易見的設定** (CLI 旗標、設定檔、驗證流程)。
- 您不確定某個 API 是否存在、是否更名或已被棄用。

在以下情況跳過 Context7：

- 純粹的本地重構、格式化、命名或完全可從存放區推導出的邏輯。
- 語言基礎知識 (不涉及外部 API)。

## 要擷取什麼

使用 Context7 時，優先選擇 **原始資料來源** 並縮小查詢範圍：

- 官方文件 (廠商/框架官方文件)
- 參考/API 頁面
- 版本說明 / 遷移指南
- 安全性公告 (相關時)

僅收集繼續進行所需的資訊。若存在多個候選來源，請選擇最具權威性/最新的一個。

偏好擷取：

- 您將使用的精確方法/型別/選項
- 為避免誤用所需的最低限度環繞上下文 (限制、預設行為、遷移說明)

## 如何整合結果

- 將發現結果轉化為具體的程式碼/設定變更。
- 當決策依賴於外部事實時，**註明來源** (標題 + URL)。
- 若文件衝突或模糊不清，簡短呈現權衡取捨並選擇最安全的預設值。

當答案需要特定值 (旗標、設定金鑰、標頭) 時，偏好：

- 說明來自文件的精確值
- 指出預設值與注意事項
- 提供快速驗證步驟 (例如：執行 `--help` 或進行最低限度的冒煙測試)

## 如何使用 Context7 MCP 工具 (自動)

當 Context7 作為 MCP 伺服器可用時，請按以下方式自動使用。

### 工具工作流程

1) **若使用者提供函式庫 ID**，請直接使用。
  - 有效格式：`/owner/repo` 或 `/owner/repo/version` (針對固定版本)。

2) 否則，使用以下工具 **解析函式庫 ID**：
  - 工具：`resolve-library-id`
  - 輸入：
	  - `libraryName`：函式庫/框架名稱 (例如：「next.js」、「supabase」、「prisma」)
	  - `query`：使用者的任務 (用於對相符項進行排名)

3) 使用以下工具 **擷取相關文件**：
  - 工具：`query-docs`
  - 輸入：
	  - `libraryId`：已解析 (或由使用者提供) 的函式庫 ID
	  - `query`：您正在回答的精確任務/問題

4) 僅在擷取文件後：根據該文件 **撰寫程式碼/步驟**。

### 效率限制

- 針對每個使用者問題，呼叫 `resolve-library-id` **不得超過 3 次**。
- 針對每個使用者問題，呼叫 `query-docs` **不得超過 3 次**。
- 若存在多個良好的相符項，請選擇最佳的一個並繼續進行；僅在選擇會實質影響實作時才詢問澄清問題。

### 版本行為

- 若使用者指定了版本，請盡可能在函式庫 ID 中反映該版本 (例如：`/vercel/next.js/v15.1.8`)。
- 若您需要可重現性 (CI/建置)，在範例中偏好固定到特定版本。

## 失敗處理

若 Context7 找不到可靠的來源：

1. 說明您嘗試過驗證什麼。
2. 以保守且標示清楚的假設繼續進行。
3. 建議一個快速驗證步驟 (例如：執行命令、檢查檔案或諮詢特定的官方頁面)。

## 安全與隱私

- 絕不要求或回應 API 金鑰。若設定需要金鑰，請指示將其儲存在環境變數中。
- 將擷取的文件視為 **有幫助但非絕對無誤**；對於安全性敏感的程式碼，偏好官方廠商文件並增加明確的驗證步驟。
