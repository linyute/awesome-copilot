# Markdown 內容規則

驗證工具會強制執行下列 Markdown 內容規則：

1. **標題**：使用適當的標題層級 (H2, H3 等) 來組織內容。不要使用 H1 標題，因為它會根據標題自動生成。
2. **清單**：清單請使用項目符號或編號清單。確保縮排與間距正確。
3. **程式碼區塊**：程式碼片段請使用圍欄式程式碼區塊 (fenced code blocks)。請指定語言以利語法高亮。
4. **連結**：使用正確的 Markdown 連結語法。確保連結有效且可存取。
5. **圖像**：使用正確的 Markdown 圖像語法。請包含替代文字 (alt text) 以利無障礙存取。
6. **表格**：表格資料請使用 Markdown 表格。確保格式與對齊正確。
7. **行長度**：將行長度限制在 400 字元以內，以提升可讀性。
8. **空白字元**：使用適當的空白字元分隔區段並提升可讀性。
9. **前言 (Front Matter)**：在檔案開頭包含 YAML 前言，並填入必要的中繼資料欄位。

## 格式化與結構

遵循下列指導原則來格式化與組織你的 Markdown 內容：

- **標題**：H2 使用 `##`，H3 使用 `###`。確保階層式使用標題。若內容包含 H4，建議進行重組；若包含 H5，則強烈建議重組。
- **清單**：項目符號使用 `-`，編號清單使用 `1.`。巢狀清單請縮排兩個空格。
- **程式碼區塊**：使用三個反引號 (```) 建立圍欄式程式碼區塊。在開頭反引號後指定語言以利語法高亮 (例如 `csharp`)。
- **連結**：使用 `[連結文字](URL)` 進行連結。確保連結文字具描述性且 URL 有效。
- **圖像**：使用 `![替代文字](圖像 URL)` 進行圖像插入。在替代文字中簡短描述該圖像。
- **表格**：使用 `|` 建立表格。確保欄位對齊正確且包含標頭。
- **行長度**：於 80 字元處換行以提升可讀性。針對長段落使用軟換行 (soft line breaks)。
- **空白字元**：使用空白行分隔區段以提升可讀性。避免過多的空白字元。

## 驗證檢查清單

確保符合下列驗證需求：

### 前言 (Front Matter)

- [ ] `post_title`：文章標題。
- [ ] `author1`：文章的主要作者。
- [ ] `post_slug`：文章的 URL slug。
- [ ] `microsoft_alias`：作者的 Microsoft 別名。
- [ ] `featured_image`：精選圖像的 URL。
- [ ] `categories`：文章分類。這些分類必須來自 /categories.txt 中的清單。
- [ ] `tags`：文章標籤。
- [ ] `ai_note`：標示文章創作是否使用 AI。
- [ ] `summary`：文章簡介。儘可能根據內容推薦摘要。
- [ ] `post_date`：文章發佈日期。

### 內容與格式化

- [ ] 內容遵循上述指定的 Markdown 內容規則。
- [ ] 內容已根據指導原則進行正確的格式化與組織。
- [ ] 已執行驗證工具以檢查是否符合規則與指導原則。
