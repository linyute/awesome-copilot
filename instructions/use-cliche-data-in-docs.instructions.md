# 在文件中使用陳腔濫調的資料

撰寫或更新工具文件時，**絕對不要包含** 任何從 Prompt、本機設定、指令碼、任務檔案或任何其他實作特定來源所提供的真實資料。文件必須僅使用通用的、廣為人知的預留位置資料，且不得洩漏任何敏感資訊。

## 為什麼這很重要

工具的原始碼與本機設定通常包含真實姓名、真實電子郵件地址、真實組織詳細資訊與真實網域名稱。這些數值對於工具運作是必要的，但它們 **絕對不應出現在公開文件** 中。將真實資料洩漏至文件中可能會暴露：

- 內部企業名稱與聯絡人
- 電子郵件地址與網域名稱
- 客戶識別碼
- 帳戶名稱與憑證
- 揭露私有營運的組織特定術語

## 核心規則

> **若資料來自 Prompt、本機檔案、指令碼、設定檔或任務，則它絕不能進入文件中。**
>
> 文件範例應僅使用廣為人知、虛構或顯而易見的預留位置資料。

## 什麼算真實資料

任何源自下列來源的數值：

- **本機設定檔** (例如 `config.json`, `.env`, 帳戶模組)
- **指令碼與任務檔案** (例如批次指令碼、Shell 指令碼、任務執行器)
- **Prompt Context** (例如使用者在要求代理建構或更新工具時所提供的資料)
- **對應或篩選檔案** (例如 JSON 對應、資料提取規則)
- **Git 忽略的檔案** (例如排除於版本控制之外，包含環境特定數值的檔案)

## 文件建議使用的預留位置資料

在所有文件與範例中，請使用這些通用的陳腔濫調替代方案：

| 類別 | 建議的預留位置範例 |
| --- | --- |
| **人物** | Jane Doe, John Smith, Alice, Bob |
| **電子郵件地址** | `jane.doe@example.com`, `admin@example.org` |
| **組織** | Acme Corp, Contoso, Northwind Traders |
| **網域** | `example.com`, `example.org`, `example.net` |
| **地址** | 123 Main Street, Suite 100, Springfield |
| **電話號碼** | `(555) 123-4567` |
| **帳戶 / 使用者名稱** | `demo-user`, `test-account` |
| **檔案路徑** | `accounts/acme.mjs`, `config/reports.json` |
| **專案名稱** | My Project, Sample App, Demo Tool |

## 如何應用此規則

### 新增功能時

若你使用真實帳戶資料新增功能 (例如以真實客戶命名的指令碼)，請改用虛構帳戶名稱來記錄該功能。

**真實實作檔案**：針對特定企業設定的帳戶模組

**文件範例**：

```javascript
// accounts/acme.mjs — 帳戶設定範例
export default {
  name: 'Acme Corp',
  email: 'reports@example.com',
  folder: 'INBOX',
};
```

### 更新設定文件時

若設定檔參考真實網域、真實路徑或真實憑證，在將其包含於文件中之前，請將每個真實數值取代為預留位置。

**文件範例**：

```json
{
  "host": "imap.example.com",
  "user": "admin@example.com",
  "folder": "INBOX/Reports",
  "outputDir": "./downloads"
}
```

### 撰寫指令碼範例時

若指令碼自動化特定組織的任務，文件範例必須使用通用組織名稱與通用參數。

**文件範例**：

```batch
@echo off
REM 範例：為 Acme Corp 執行提取任務
node extractEmail.mjs --account acme --task download
```

## 程式碼與文件之間的界線

| Context | 是否允許真實資料？ |
| --- | --- |
| 執行階段使用的本機指令碼與設定檔 | 是 |
| 包含環境特定數值的 Git 忽略檔案 | 是 |
| 提供給建構或設定工具的 Prompt 資料 | 是 (僅限於程式碼中) |
| README.md, docs/ 資料夾, 與範本範例 | **否 — 僅使用預留位置** |
| CHANGELOG.md 條目 | **否 — 以通用方式描述變更** |
| 已提交原始碼中的程式碼註解 | **否 — 保持通用** |

## 例外狀況

真實資料中的詞彙 **僅在** 該詞彙為普通英文詞彙，且於句子中使用其一般意義，而 **非** 於範例 Context 中使用時，才可出現在文件中。例如，詞彙 "development" 在句子 "This tool is under active development" 中是可以接受的，即使它同時出現在某個真實組織名稱中。

## 總結

文件是公開的。實作資料是私有的。請將它們區隔開。文件中的每個範例皆應通過簡單的測試：*陌生人閱讀後，是否無法得知此工具背後的真實使用者、客戶或組織？* 若答案為否，請以陳腔濫調的預留位置取代該資料。
