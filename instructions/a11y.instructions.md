# 無障礙標準 (Accessibility Standards)

網頁應用程式開發的全面無障礙規則。每個反模式 (anti-pattern) 都包含嚴重性分類、檢測方法、WCAG 2.2 參考以及修正程式碼範例。

**嚴重性等級：**

- **CRITICAL (關鍵)** — 使用者完全無法存取內容。合併前必須修正。
- **IMPORTANT (重要)** — 對輔助技術使用者造成重大障礙。在同一個開發週期內修正。
- **SUGGESTION (建議)** — 改善輔助技術的使用體驗。規劃於未來迭代中處理。

## WCAG 2.2 快速參考 (AA 等級)

### 可感知性 (Perceivable)

| 準則 | 等級 | 摘要 |
|-----------|-------|---------|
| 1.1.1 非文字內容 | A | 所有非文字內容都有文字替代。裝飾性圖像使用 `alt=""`。 |
| 1.2.1 僅限音訊/影片 | A | 提供逐字稿 (音訊) 或文字替代 (影片)。 |
| 1.2.2 字幕 (預錄) | A | 所有預錄影片都有同步字幕。 |
| 1.3.1 資訊與關係 | A | 結構 (標題、清單、表格、標籤、地標) 需以程式化方式傳達。 |
| 1.3.2 有意義的順序 | A | 當內容呈現順序會影響意義時，內容的視覺與程式排序應一致。 |
| 1.3.3 感官特性 | A | 指令不得僅依賴形狀、大小、位置或聲音。 |
| 1.3.4 方向性 | AA | 內容除非必要，不得限制為單一方向。 |
| 1.3.5 識別輸入目的 | AA | 收集使用者資訊時，輸入欄位應有 `autocomplete` 屬性。 |
| 1.4.1 顏色運用 | A | 顏色不得為傳達資訊的唯一方式。 |
| 1.4.3 對比 (最低要求) | AA | 文字：正常 4.5:1，大型 (18pt / 14pt 粗體) 3:1。 |
| 1.4.4 文字調整 | AA | 文字可放大至 200% 而不損失內容或功能。 |
| 1.4.10 重排 (Reflow) | AA | 內容區段應能在 320px CSS 寬度視埠內適應，無需在兩個維度上捲動即可閱讀。 |
| 1.4.11 非文字對比 | AA | UI 元件與圖形：對相鄰顏色需 3:1。 |
| 1.4.12 文字間距 | AA | 使用者覆寫行高 (1.5x)、段落間距、字元間距與字詞間距調整時，內容或功能不得損失。 |
| 1.4.13 滑鼠懸停或焦點內容 | AA | 滑鼠懸停或焦點時出現的彈出內容應為：可關閉、可懸停、持久。 |

### 可操作性 (Operable)

| 準則 | 等級 | 摘要 |
|-----------|-------|---------|
| 2.1.1 鍵盤 | A | 所有功能皆可透過鍵盤操作。 |
| 2.1.2 無鍵盤陷阱 | A | 使用者可透過鍵盤離開任何元件。 |
| 2.2.1 計時調整 | A | 時間限制可延長或停用。 |
| 2.2.2 暫停、停止、隱藏 | A | 自動更新的內容可暫停。 |
| 2.3.1 三次閃爍 | A | 內容每秒閃爍不得超過 3 次。 |
| 2.4.1 略過區塊 | A | 略過連結以避開重複的導覽。 |
| 2.4.2 頁面標題 | A | 頁面有描述性的 `<title>`。 |
| 2.4.3 焦點順序 | A | 焦點順序保留意義與可操作性。 |
| 2.4.4 連結目的 | A | 連結目的可透過文字或上下文確定。 |
| 2.4.6 標題與標籤 | AA | 標題與標籤描述主題或目的。 |
| 2.4.7 焦點可見 | AA | 鍵盤焦點指示器可見。 |
| 2.4.11 焦點未被遮蔽 | AA | 焦點元素未被其他重疊元素 (如黏性頁首或頁尾) 完全隱藏。 *(WCAG 2.2 新增)* |
| 2.5.1 指標手勢 | A | 多點手勢有單點替代方案。 |
| 2.5.2 指標取消 | A | 若無法中止或反轉，操作應發生在向上事件 (up-event)，除非向下事件操作是必要的。 |

| 2.5.8 目標大小 (最低要求) | AA | 互動控制項應有目標大小，或至少 24x24 CSS px 的間距。 *(WCAG 2.2 新增)* |

### 可理解性 (Understandable)

| 準則 | 等級 | 摘要 |
|-----------|-------|---------|
| 3.1.1 頁面語言 | A | `<html lang="...">` 設定正確。 |
| 3.1.2 部分語言 | AA | 不同語言的內容應標記 `lang` 屬性。 |
| 3.2.1 焦點時 | A | 焦點不會觸發意外的上下文變更。 |
| 3.2.2 輸入時 | A | 變更輸入不會自動觸發意外的上下文變更。 |
| 3.2.6 一致的輔助 | A | 輔助機制在頁面間保持相對順序。 *(WCAG 2.2 新增)* |
| 3.3.1 錯誤識別 | A | 錯誤以文字描述給使用者。 |
| 3.3.2 標籤或說明 | A | 為使用者輸入提供標籤或說明。 |
| 3.3.3 錯誤建議 | AA | 為檢測到的錯誤建議修正方式。 |
| 3.3.4 錯誤預防 | AA | 提交內容可反轉、檢查或確認。 |
| 3.3.7 冗餘輸入 | A | 同一流程中不應重複詢問已提供的資訊。 *(WCAG 2.2 新增)* |
| 3.3.8 無障礙驗證 (最低要求) | AA | 不得使用認知功能測試 (如拼圖驗證碼)。允許貼上與自動填入。 *(WCAG 2.2 新增)* |

### 強健性 (Robust)

| 準則 | 等級 | 摘要 |
|-----------|-------|---------|
| 4.1.2 名稱、角色、值 | A | 所有 UI 元件皆有可存取名稱、角色與狀態。 |
| 4.1.3 狀態訊息 | AA | 狀態訊息應由螢幕閱讀器宣告，無需取得焦點。 |

> **注意：** 4.1.1 剖析 (Parsing) 在 WCAG 2.2 中已廢除 (總是滿足)。其涵蓋的問題現在由 1.3.1 與 4.1.2 處理。

> **WCAG 2.2 新增的 AAA 準則** (AA 不強制要求，但建議)：2.4.12 焦點未被遮蔽 (增強版)、2.4.13 焦點外觀、3.3.9 無障礙驗證 (增強版)。

> **展望未來：** WCAG 3.0 (W3C 無障礙準則) 目前處於工作草案階段 (2026 年 3 月)。它將通過/失敗標準改為銅/銀/金級一致性，並以「成果」(Outcomes) 取代「成功準則」。目前尚未成為標準 — 請持續針對 WCAG 2.2 AA 進行優化。

## 法律執行背景 (2026)

- **歐洲無障礙法案 (EAA)**：自 2025 年 6 月起在所有 27 個歐盟成員國實施。適用於數位產品與服務。罰款最高可達 300 萬歐元。參考 EN 301 549 (對應 WCAG 2.1 AA)。
- **ADA Title II (美國)**：針對服務 50,000 人以上州/地方政府的數位無障礙規則，於 2026 年 4 月生效 (較小機構於 2027 年 4 月)。要求符合 WCAG 2.1 AA。
- **第 508 節 (美國聯邦)**：參考 WCAG 2.0 AA (預計將更新至 2.1/2.2)。

**目標**：WCAG 2.2 AA 涵蓋所有當前法律要求 (2.1 AA 與 2.0 AA 的超集)。

## ARIA 五大原則

1. **優先使用原生 HTML** — 使用 `<button>` 而非 `<div role="button">`。原生元素具備內建的鍵盤、焦點與語意。
2. **禁止修改原生語意** — 不要在 `<button>` 上新增 `role="heading"`。應使用正確的元素。
3. **所有 ARIA 控制項必須可鍵盤操作** — 若 `role="button"`，需處理 Enter 與 Space 鍵事件。
4. **禁止在可焦點元素上使用 `aria-hidden="true"`** — 對輔助技術隱藏但仍可聚焦會產生「幽靈」元素。
5. **所有互動元素需要可存取名稱** — 透過標籤 (label)、`aria-label`、`aria-labelledby` 或視覺文字內容。

## 語意化 HTML 反模式 (S1-S8)

### S1: `<html>` 缺少 `lang` 屬性

- **嚴重性**：CRITICAL
- **檢測**：`<html` 未包含 `lang=`
- **WCAG**：3.1.1 (A)

```html
<!-- 錯誤 -->
<html>

<!-- 正確 -->
<html lang="zh-Hant">
```

Next.js：在 `app/layout.tsx` 中設定。Angular：在 `src/index.html` 中設定。Vue/Nuxt：在 `app.vue` 或 `nuxt.config` 中設定。

### S2: 頁面出現多個 `<h1>`

- **嚴重性**：SUGGESTION
- **檢測**：多個 `<h1>` 元素導致頁面標題結構不明確
- **WCAG**：最佳實務 (支援 1.3.1)

每個頁面建議使用一個 `<h1>` 代表主要主題。使用 `<h2>` 處理區段。多個 `<h1>` 並非嚴格違反 WCAG，但可能混淆螢幕閱讀器的導覽。

### S3: 標題層級間隙

- **嚴重性**：IMPORTANT
- **檢測**：`<h1>` 後直接跟著 `<h3>` (跳過 `<h2>`)
- **WCAG**：1.3.1 (A)

保持邏輯嵌套：`h1 > h2 > h3 > h4`。使用 CSS 設定標題樣式，而非選擇不同的標題層級。

### S4: Div 湯 (Div Soup) — 沒有地標元素 (Landmark Elements)

- **嚴重性**：IMPORTANT
- **檢測**：頁面僅使用 `<div>`，沒有使用 `<nav>`, `<main>`, `<header>`, `<footer>`
- **WCAG**：最佳實務 (支援 1.3.1, 2.4.1)

```html
<!-- 正確 -->
<header>...</header>
<nav aria-label="主要">...</nav>
<main>...</main>
<footer>...</footer>
```

### S5: 未使用 `role="presentation"` 的版面配置表格

- **嚴重性**：IMPORTANT
- **檢測**：未使用 `<th>`, `<caption>` 或 `role="presentation"` 的 `<table>`
- **WCAG**：1.3.1 (A)

使用 CSS Grid/Flexbox 進行版面配置。若必須使用表格進行版面配置，請新增 `role="presentation"`。

### S6: 資料表格缺少標頭

- **嚴重性**：CRITICAL
- **檢測**：包含資料列但沒有 `<th>` 元素的 `<table>`
- **WCAG**：1.3.1 (A)

```html
<!-- 正確 -->
<table>
  <caption>使用者清單</caption>
  <thead>
    <tr><th scope="col">姓名</th><th scope="col">電子郵件</th></tr>
  </thead>
  <tbody>
    <tr><td>Alice</td><td>alice@example.com</td></tr>
  </tbody>
</table>
```

### S7: 連結文字描述不清

- **嚴重性**：IMPORTANT
- **檢測**：`>點擊這裡<|>閱讀更多<|>了解更多<|>這裡<|>更多<|>連結<`
- **WCAG**：2.4.4 (A)

```html
<!-- 錯誤 -->
<a href="/pricing">點擊這裡</a>

<!-- 正確 -->
<a href="/pricing">檢視定價方案</a>
```

### S8: 互動元素未使用語意化 HTML

- **嚴重性**：CRITICAL
- **檢測**：`<div.*(?:onClick|@click|\(click\))`
- **WCAG**：4.1.2 (A)

```tsx
// 錯誤 — 無法聚焦、無角色、無鍵盤支援
<div onClick={handleClick}>提交</div>

// 正確
<button onClick={handleClick}>提交</button>
```

## ARIA 反模式 (A1-A8)

### A1: 原生元素上的冗餘 ARIA

- **嚴重性**：SUGGESTION
- **檢測**：`<button.*role="button"|<nav.*role="navigation"|<a.*role="link"`
- **WCAG**：ARIA 原則 2

移除冗餘的 ARIA。`<button>` 已內建 `role="button"`。

### A2: 可焦點元素上使用 `aria-hidden="true"`

- **嚴重性**：CRITICAL
- **檢測**：在可焦點元素 (button, input, a, [tabindex]) 上使用 `aria-hidden="true"`
- **WCAG**：ARIA 原則 4

不要將可焦點元素留在 `aria-hidden="true"` 內容中，或在可焦點元素上使用 `aria-hidden="true"`。對於支援的原生控制項 (如 `<button>` 或 `<input>`)，若要停用控制項，請使用 `disabled` 屬性。對於 `<a>` 元素或透過 `[tabindex]` 變為可焦點的任意元素，請移除其焦點能力 (例如：移除 `href` 或 `tabindex`)。若內容應完全非互動且對輔助技術隱藏，請使用 `inert`、`hidden` 或從 DOM 中移除。

### A3: 缺少必要 ARIA 屬性

- **嚴重性**：CRITICAL
- **檢測**：`role="tab"` 缺少 `aria-selected`，`role="checkbox"` 缺少 `aria-checked`
- **WCAG**：4.1.2 (A)

各角色需求：`tab` 需要 `aria-selected`；`combobox` 需要 `aria-expanded`/`aria-controls`；`slider` 需要 `aria-valuemin`/`aria-valuemax`/`aria-valuenow`；`checkbox` 需要 `aria-checked`。

### A4: 無效 ARIA 角色值

- **嚴重性**：CRITICAL
- **檢測**：`role="[^"]*"` 使用不存在的值
- **WCAG**：4.1.2 (A)

無效角色會被輔助技術忽略。常見錯誤：`role="input"`, `role="text"`, 拼字錯誤。

### A5: 應使用原生 HTML 處卻使用 ARIA

- **嚴重性**：IMPORTANT
- **檢測**：在 `<div>` 上使用 `role="button"`, 在 `<div>` 上使用 `role="checkbox"`
- **WCAG**：ARIA 原則 1

```html
<!-- 錯誤 — 需要手動鍵盤、焦點與狀態管理 -->
<div role="checkbox" aria-checked="false" tabindex="0">接受條款</div>

<!-- 正確 — 所有行為皆內建 -->
<label><input type="checkbox" /> 接受條款</label>
```

### A6: 圖示按鈕缺少 `aria-label`

- **嚴重性**：CRITICAL
- **檢測**：包含 SVG/圖示子元素且無文字或 `aria-label` 的 `<button`
- **WCAG**：4.1.2 (A)

```html
<!-- 正確 -->
<button aria-label="關閉對話框"><svg aria-hidden="true">...</svg></button>
```

### A7: 在可焦點元素上使用 `role="presentation"`

- **嚴重性**：IMPORTANT
- **檢測**：在互動元素上使用 `role="presentation"`
- **WCAG**：ARIA 原則 4

瀏覽器會忽略可焦點元素上的呈現 (presentation) 角色。

### A8: 動態內容缺少即時區域 (Live Region)

- **嚴重性**：IMPORTANT
- **檢測**：提示/通知元件未使用 `role="alert"`, `role="status"` 或 `aria-live`
- **WCAG**：4.1.3 (AA)

```html
<!-- 正確 — 當內容被注入到 DOM 中預先存在的即時區域元素時，內容會被宣告 -->
<div role="status" aria-live="polite">項目儲存成功</div>
<!-- 對於錯誤，請使用 role="alert" (斷言式) -->
<div role="alert">儲存失敗。請再試一次。</div>
```

## 鍵盤與焦點反模式 (K1-K7)

### K1: 非原生元素 `onClick` 缺少 `onKeyDown`

- **嚴重性**：CRITICAL
- **檢測**：在 `<div>` 或 `<span>` 上使用 `(?:onClick|@click|\(click\))` 且未處理鍵盤事件
- **WCAG**：2.1.1 (A)

請改用 `<button>`。若一定要使用 div：請新增 `role="button"`, `tabIndex={0}`, 並處理 Enter 與 Space 鍵啟動事件。

### K2: 正數 `tabindex` 值

- **嚴重性**：CRITICAL
- **檢測**：`(?:tabindex="[1-9]\d*"|tabIndex=\{[1-9]\d*\})`
- **WCAG**：2.4.3 (A)

僅使用 `tabindex="0"` (新增至 Tab 順序) 與 `tabindex="-1"` (僅程式化聚焦)。

### K3: 沒有 Escape 的焦點陷阱

- **嚴重性**：CRITICAL
- **檢測**：沒有 Escape 鍵處理程式或焦點陷阱的 Modal/覆蓋層
- **WCAG**：2.1.2 (A)

使用內建 `showModal()` 的原生 `<dialog>` — 它能防止鍵盤焦點移至非對話框的 inert 內容。此外，它內建 Escape 鍵關閉功能，且焦點會自動返回至調用元素 (若存在)。若需自訂 Modal 對話框：需在對話框內陷阱 Tab，或對非對話框內容使用 `inert` 屬性 (不可對包含對話框的元素使用 `inert`)，按 Escape 關閉 (除非使用者確認動作是必要的)，並在關閉時將焦點返回給觸發元素，或在觸發元素已不存在時返回至最佳邏輯位置。

### K4: 缺少略過連結 (Skip Link)

- **嚴重性**：IMPORTANT
- **檢測**：沒有作為第一個可焦點元素的略過連結
- **WCAG**：2.4.1 (A)

```html
<a href="#main-content" class="skip-link">跳過至主要內容</a>
<nav>...</nav>
<main id="main-content" tabindex="-1">...</main>
```

```css
.skip-link { position: absolute; top: -40px; left: 0; padding: 8px 16px; background: #000; color: #fff; z-index: 100; }
.skip-link:focus { top: 0; }
```

### K5: 使用 `outline: none` 但沒有替換方案

- **嚴重性**：CRITICAL
- **檢測**：`outline:\s*none|outline:\s*0\b` 但沒有 `:focus-visible` 替換
- **WCAG**：2.4.7 (AA)

```css
/* 正確 */
button:focus-visible { outline: 2px solid #005fcc; outline-offset: 2px; }
```

### K6: 滑鼠專屬互動

- **嚴重性**：IMPORTANT
- **檢測**：`onMouseOver|onMouseEnter|@mouseenter` 但沒有鍵盤對等操作
- **WCAG**：2.1.1 (A)

成對使用懸停與焦點事件。將 `onFocus`/`onBlur` 與 `onMouseEnter`/`onMouseLeave` 搭配使用。

### K7: 自訂 Modal 關閉後焦點未返回

- **嚴重性**：IMPORTANT
- **檢測**：自訂 Modal 對話框關閉時未恢復焦點給觸發元素
- **WCAG**：2.4.3 (A)

儲存觸發元素的參考，或在關閉時返回至邏輯上的最佳位置。Modal 關閉時，呼叫 `triggerElement.focus()`。

## 表單反模式 (F1-F6)

### F1: 輸入欄位缺少關聯標籤

- **嚴重性**：CRITICAL
- **檢測**：`<input|<select|<textarea` 未使用 `<label>`, `aria-label`, 或 `aria-labelledby`
- **WCAG**：1.3.1 (A), 3.3.2 (A)

```html
<!-- 正確 -->
<label for="email">電子郵件地址</label>
<input id="email" type="email" placeholder="you@example.com" />
```

### F2: 錯誤訊息未連結到輸入欄位

- **嚴重性**：CRITICAL
- **檢測**：輸入欄位附近的錯誤元素未使用 `aria-describedby`
- **WCAG**：3.3.1 (A)

```html
<input id="email" type="email" aria-describedby="email-error" aria-invalid="true" />
<span id="email-error" class="error">電子郵件格式無效</span>
```

### F3: 必要欄位僅以顏色或 `*` 標示

- **嚴重性**：IMPORTANT
- **檢測**：未使用 `aria-required="true"` 或 HTML `required` 的 `required` 或 `*`
- **WCAG**：3.3.2 (A), 1.4.1 (A)

```html
<label for="name">姓名 <span aria-hidden="true">*</span></label>
<input id="name" type="text" required />
<p class="form-note">標記 * 為必填欄位</p>
```

### F4: 沒有錯誤摘要或焦點未移至第一個錯誤

- **嚴重性**：IMPORTANT
- **檢測**：表單提交處理程式在驗證失敗時未管理焦點
- **WCAG**：3.3.1 (A)

提交失敗時，將焦點移至第一個無效欄位，或顯示並聚焦於錯誤摘要。

### F5: 沒有無障礙替代方案的 CAPTCHA

- **嚴重性**：IMPORTANT
- **檢測**：無後備方案的拼圖/圖像驗證碼；密碼欄位 `autocomplete='off'` 或阻止貼上的 JavaScript
- **WCAG**：3.3.8 (AA)

使用 reCAPTCHA v3 (隱形)、hCaptcha 無障礙模式或替代身份驗證。絕對不要在密碼欄位阻止貼上或自動填入 — 這違反 WCAG 3.3.8。

### F6: 佔位符 (Placeholder) 作為標籤

- **Severity**: IMPORTANT
- **Detection**: `placeholder=` 未搭配 `<label>`, `aria-label`, 或 `aria-labelledby`
- **WCAG**: 3.3.2 (A)

務必搭配可見的 `<label>`。佔位符是提示，而非標籤。

## 視覺與顏色反模式 (V1-V5)

### V1: 文字對比度不足

- **嚴重性**：CRITICAL
- **檢測**：文字顏色組合低於 4.5:1 (正常) 或 3:1 (大型)
- **WCAG**：1.4.3 (AA)

```css
/* 錯誤 — #999 在 #fff 上約為 2.5:1 */
.text { color: #999; background: #fff; }

/* 正確 — #595959 在 #fff 上為 7.0:1 */
.text { color: #595959; background: #fff; }
```

### V2: 僅以顏色傳達資訊

- **嚴重性**：CRITICAL
- **檢測**：錯誤/成功狀態僅以顏色區分
- **WCAG**：1.4.1 (A)

新增次要指示器：圖示、文字、圖案、底線或邊框。

### V3: 固定字型大小導致無法調整

- **嚴重性**：IMPORTANT
- **檢測**：`font-size:\s*[0-9]*px` (不包括根/基礎字型)
- **WCAG**：1.4.4 (AA)

字型大小使用 `rem` 或 `em`。基礎字型可使用 `px`，但內容字型應為相對單位。

### V4: 內容在 320px 無法重排

- **嚴重性**：IMPORTANT
- **檢測**：固定寬度容器，窄視埠下出現水平捲軸
- **WCAG**：1.4.10 (AA)

使用響應式版面配置 (Grid, Flexbox)。在 320px CSS 寬度下進行測試。避免使用固定寬度容器。

### V5: 動畫沒有 `prefers-reduced-motion`

- **嚴重性**：SUGGESTION
- **檢測**：`animation:|transition:` 未使用 `prefers-reduced-motion` 媒體查詢
- **WCAG**：2.3.3 (AAA)

最佳實務與 AAA 增強。將非必要動畫放在 `prefers-reduced-motion` 之後，確保請求減少動作的使用者不會被迫體驗互動觸發的特效。

```css
@media (prefers-reduced-motion: no-preference) {
  .card { transition: transform 0.3s ease; }
  .card:hover { transform: scale(1.05); }
}
```

## 媒體反模式 (D1-D4)

### D1: 資訊性圖像缺少 Alt 文字

- **嚴重性**：CRITICAL
- **檢測**：`<img|<Image` 缺少 `alt=` 屬性
- **WCAG**：1.1.1 (A)

Alt 文字決策樹：裝飾性 = `alt=""`；包含文字 = 包含該文字；功能性 = 描述動作；資訊性 = 描述內容。

### D2: 裝飾性圖像使用非空的 Alt

- **嚴重性**：SUGGESTION
- **檢測**：裝飾性圖像包含有意義的 alt 文字
- **WCAG**：1.1.1 (A)

裝飾性圖像使用 `alt=""`。裝飾性 SVG 新增 `aria-hidden="true"`。

### D3: 影片缺少字幕

- **嚴重性**：CRITICAL
- **檢測**：`<video` 缺少 `<track kind="captions">`
- **WCAG**：1.2.2 (A)

```html
<video src="/tutorial.mp4" controls>
  <track kind="captions" src="/tutorial-zh-tw.vtt" srclang="zh-tw" label="繁體中文" default />
</video>
```

### D4: 音訊/影片自動播放

- **嚴重性**：IMPORTANT
- **檢測**：`autoplay` 屬性但未設定 `muted`
- **WCAG**：1.4.2 (A)

絕對不要自動播放音訊。若影片自動播放，請靜音並提供控制項。

## 框架特定：React / Next.js (RX1-RX4)

### RX1: `<label>` 缺少 `htmlFor`

- **嚴重性**：IMPORTANT
- **檢測**：JSX 中的 `<label.*for="` (應為 `htmlFor`)
- **WCAG**：1.3.1 (A), 3.3.2 (A)

### RX2: SPA 路由變更後缺少焦點管理

- **嚴重性**：IMPORTANT
- **檢測**：導覽時缺少焦點管理或即時區域
- **WCAG**：4.1.3 (AA)

路由變更後，聚焦於主要標題，或透過即時區域宣告新的頁面標題。Next.js (v13+) 內建路由宣告器，會讀取 `document.title`、`<h1>`、pathname。確保每個頁面皆有唯一的 `<title>`。

### RX3: Fragment 根節點導致重新渲染時焦點遺失

- **嚴重性**：SUGGESTION
- **檢測**：`<>...</>` 根節點搭配條件渲染導致 DOM 結構重組
- **WCAG**：2.4.3 (A)

使用 `key` 屬性保持 DOM 識別，或使用 `useRef` + `useEffect` 手動恢復焦點。

### RX4: 注入 HTML 缺少 ARIA 考量

- **嚴重性**：IMPORTANT
- **檢測**：富文字渲染缺少無障礙驗證
- **WCAG**：1.3.1 (A)

對注入的 HTML 進行清理與驗證，檢查標題階層、alt 文字與 ARIA 結構。

## 框架特定：Angular (NG1-NG4)

### NG1: `(click)` 於 `<div>` 缺少角色與鍵盤支援

- **嚴重性**：CRITICAL
- **檢測**：在 `<div>` 或 `<span>` 使用 `(click)`，但未定義 `role=`, `tabindex`, `(keydown)`
- **WCAG**：2.1.1 (A), 4.1.2 (A)

請改用 `<button>`。若一定要用 div：新增 `role="button"`, `tabindex="0"`, `(keydown.enter)`, `(keydown.space)`。

### NG2: Modal 元件缺少 `cdkTrapFocus`

- **嚴重性**：IMPORTANT
- **檢測**：Modal 元件缺少 `cdkTrapFocus`
- **WCAG**：2.1.2 (A)

```html
<div class="modal" cdkTrapFocus [cdkTrapFocusAutoCapture]="true">...</div>
```

Angular CDK 的 `Dialog` 服務會自動處理焦點陷阱與恢復。

### NG3: 路由變更缺少 `LiveAnnouncer`

- **嚴重性**：IMPORTANT
- **檢測**：Angular Router 導覽未使用 `LiveAnnouncer`
- **WCAG**：4.1.3 (AA)

```typescript
router.events.pipe(filter(e => e instanceof NavigationEnd)).subscribe(() => {
  liveAnnouncer.announce(titleService.getTitle(), 'polite');
});
```

### NG4: 範本驅動表單 (Template-Driven Forms) 缺少無障礙驗證

- **嚴重性**：IMPORTANT
- **檢測**：表單顯示錯誤但未設定 `[attr.aria-invalid]` 或 `[attr.aria-describedby]`
- **WCAG**：3.3.1 (A), 3.3.3 (AA)

將 `[attr.aria-invalid]` 與 `[attr.aria-describedby]` 綁定至表單控制項狀態。

## 框架特定：Vue (VU1-VU3)

### VU1: `@click` 於非互動元素缺少角色與鍵盤操作

- **嚴重性**：CRITICAL
- **檢測**：在 `<div>` 或 `<span>` 使用 `@click` 但未定義 `role=`, `tabindex`, `@keydown`
- **WCAG**：2.1.1 (A), 4.1.2 (A)

請改用 `<button>`。或新增 `role="button"`, `tabindex="0"`, `@keydown.enter`, `@keydown.space.prevent`。

### VU2: `v-if` 切換缺少焦點管理

- **嚴重性**：IMPORTANT
- **檢測**：`v-if` 切換時未透過 `nextTick` 管理焦點
- **WCAG**：2.4.3 (A)

```vue
<script setup>
import { ref, watch, nextTick } from 'vue';
const showPanel = ref(false);
const panel = ref(null);
watch(showPanel, async (val) => {
  if (val) { await nextTick(); panel.value?.focus(); }
});
</script>
```

### VU3: `v-html` 注入缺少無障礙結構

- **嚴重性**：IMPORTANT
- **檢測**：`v-html` 渲染使用者或 CMS 內容
- **WCAG**：1.3.1 (A)

在注入前，對 HTML 的標題階層、alt 文字與 ARIA 結構進行清理與驗證。

## 鍵盤互動參考

| 按鍵 | 預期行為 |
|-----|-------------------|
| `Tab` | 將焦點移至 DOM 順序中的下一個可焦點元素 |
| `Shift+Tab` | 將焦點移至上一個可焦點元素 |
| `Enter` | 啟動按鈕與連結 |
| `Space` | 啟動按鈕、切換核取方塊、選擇選項按鈕 (radio) |
| `Escape` | 關閉對話框、提示框、彈出視窗、下拉選單 |
| `Arrow Up/Down` | 在選單、清單方塊、選項按鈕群組、標籤頁中導覽 |
| `Arrow Left/Right` | 在標籤列、滑桿、選項按鈕群組中導覽 |
| `Home` | 移至清單、選單或標籤列的第一個項目 |
| `End` | 移至清單、選單或標籤列的最後一個項目 |

### 各元件特定模式

| 元件 | Tab 進入 | 內部導覽 | 啟動 | 離開 |
|--------|-----------|-------------|----------|------|
| 標籤列 | 聚焦啟用中的標籤 | 箭頭左/右 | 自動或 Enter | Tab 離開 |
| 選單 | 聚焦第一個項目 | 箭頭上/下 | Enter | Escape |
| 對話框 | 聚焦第一個元素 | Tab 在內部循環 | Enter 於按鈕 | Escape |
| 組合框 | 聚焦輸入框 | 箭頭上/下 | Enter 選擇 | Escape 關閉 |
| 樹狀結構 | 聚焦第一個節點 | 箭頭鍵 | Enter/Space | Tab 離開 |

## 無障礙檢查清單 (POUR)

### 可感知性 (Perceivable)
- [ ] 所有圖像皆有適當的 alt 文字 (描述性，或裝飾性為空白)
- [ ] 影片有同步字幕
- [ ] 頁面使用語意化地標：`<header>`, `<nav>`, `<main>`, `<footer>`
- [ ] 標題遵循邏輯階層 (h1 > h2 > h3，無間隙)
- [ ] 文字對比度達 4.5:1 (正常) / 3:1 (大型)
- [ ] UI 元件對比度達 3:1
- [ ] 資訊不單僅以顏色傳達
- [ ] 內容在 320px 下可重排，無水平捲軸
- [ ] `<html lang="...">` 設定正確
- [ ] 文字可調整至 200% 而內容不損失

### 可操作性 (Operable)
- [ ] 所有功能可透過鍵盤存取
- [ ] 無鍵盤陷阱 (Escape 可關閉覆蓋層)
- [ ] 提供略過連結作為第一個可焦點元素
- [ ] 所有互動元素上皆有可見的焦點指示器
- [ ] 焦點順序與視覺順序相符
- [ ] 焦點未被黏性頁首/頁尾遮蔽
- [ ] Modal 關閉後焦點返回至觸發元素
- [ ] 觸控目標至少為 24x24 CSS px
- [ ] 動畫尊重 `prefers-reduced-motion`
- [ ] 內容每秒閃爍不超過 3 次

### 可理解性 (Understandable)
- [ ] 所有表單輸入皆有關聯的 `<label>` 或 `aria-label`
- [ ] 錯誤訊息透過 `aria-describedby` 連結至輸入欄位
- [ ] 必要欄位以 `required` 或 `aria-required` 標示
- [ ] 提交失敗時有錯誤摘要或聚焦至第一個錯誤
- [ ] 焦點或輸入時不會發生意外的上下文變更

### 強健性 (Robust)
- [ ] 所有互動元素皆有可存取名稱、角色與狀態
- [ ] ARIA 角色具備必要屬性
- [ ] 可焦點元素上無 `aria-hidden="true"`
- [ ] 動態內容透過即時區域宣告
- [ ] SPA 路由變更向螢幕閱讀器宣告
- [ ] 原生 HTML 元素上無冗餘 ARIA
