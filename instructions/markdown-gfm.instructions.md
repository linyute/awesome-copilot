# GitHub Flavored Markdown (GFM)

在撰寫或審查 `.md` 檔案時，請依照 [GFM 規範](https://github.github.com/gfm/) 套用下列規則。GFM 是 CommonMark 的嚴格超集。GFM 規範僅供參考，請勿下載 GFM 規範。

## 前言

- 行尾以換行 (`U+000A`)、歸位字元 (`U+000D`) 或檔案結尾定義。空白行僅包含空格或 Tab。
- Tab 在區塊結構中表現為 4 個空格的停駐點，但在內容中不會被展開。
- 將 `U+0000` 取代為替代字元 `U+FFFD`。

## 葉節點區塊 (Leaf Blocks)

- **主題分隔線**：行內 3 個以上匹配的 `-`, `_` 或 `*` 字元，且縮排 0–3 個空格。行內不可有其他字元。可中斷段落。
- **ATX 標題**：1–6 個 `#` 字元，後接一個空格或行尾。選用的閉合 `#` 序列 (前方需有空格)。允許 0–3 個空格縮排。
- **Setext 標題**：以 `=` (層級 1) 或 `-` (層級 2) 底線標示的文字。無法中斷段落 — 前方的段落後需有空白行。
- **縮排程式碼區塊**：縮排 4 個以上空格的行。無法中斷段落。內容為字面文字，不解析為 Markdown。
- **圍欄式程式碼區塊 (Fenced code blocks)**：以 3 個以上反引號或波浪號開啟 (不可混用)。閉合圍欄必須使用相同字元且數量至少相同。於開啟圍欄後指定語言識別碼。內容為字面文字。
- **HTML 區塊**：由開啟/閉合標籤條件定義的七種類型。類型 1–6 可中斷段落；類型 7 不可。內容原樣傳遞為原始 HTML。
  - 類型 1：`<script>`, `<pre>`, 或 `<style>` (不區分大小寫) — 於匹配的閉合標籤結束。
  - 類型 2：`<!--` 註解 — 於 `-->` 結束。
  - 類型 3：`<?` 處理指令 — 於 `?>` 結束。
  - 類型 4：`<!` + 大寫字母 (例如 `<!DOCTYPE>`) — 於 `>` 結束。
  - 類型 5：`<![CDATA[` — 於 `]]>` 結束。
  - 類型 6：區塊層級 HTML 標籤 (`<div>`, `<table>`, `<p>`, `<h1>`–`<h6>`, `<ul>`, `<ol>`, `<section>` 等) — 於空白行結束。
  - 類型 7：任何其他在其自身行上的完整開啟或閉合標籤 — 於空白行結束。無法中斷段落。
- **連結參考定義**：`[label]: destination "title"`。標籤匹配不區分大小寫。重複標籤以第一個定義為準。無法中斷段落。
- **段落**：無法解釋為其他區塊結構的連續非空白行。開頭最多 3 個空格會被移除。
- **空白行**：區塊間的空白行會被忽略；決定清單是緊密 (tight) 或寬鬆 (loose)。
- **表格 (擴充)**：標頭列、分隔列 (`---`, `:---:`, `---:`)，以及零個或多個資料列。使用 `|` 分隔儲存格。文字內的管線符號需跳脫為 `\|`。標頭與分隔列的欄數必須匹配。遇第一個空白行或其他區塊層級結構時中斷。

## 容器區塊 (Container Blocks)

- **區塊引用 (Block quotes)**：以前綴 `>` (後可選擇性接空格) 的行。僅允許對段落文字進行懶續行 (Lazy continuation)。空白行會分隔連續的區塊引用。
- **清單項目**：項目符號 (`-`, `+`, `*`) 或編號標記 (1–9 位數字 + `.` 或 `)` )。內容欄由標記寬度 + 至第一個非空白字元間的空格決定。子清單必須縮排至內容欄。中斷段落的有序清單必須以 `1` 開頭。
- **任務清單項目 (擴充)**：清單項目段落開頭的 `- [ ]` (未勾選) 或 `- [x]` (已勾選)。`-` 與 `[` 之間需要空格。可以嵌套。
- **清單**：相同型別的清單項目序列。變更項目符號或有序標記會啟動新清單。若任一項目由空白行分隔，則清單為寬鬆。

## 內聯元素 (Inlines)

- **反斜線跳脫**：ASCII 標點字元前的 `\` 會渲染為該字面字元。在程式碼區塊、程式碼行或自動連結中不識別。
- **實體與數值字元參考**：`&amp;`, `&#123;`, `&#x7B;` — 有效的 HTML5 實體。在程式碼區塊或程式碼行中不識別。無法取代結構化字元。
- **程式碼行 (Code spans)**：反引號分隔的內聯程式碼。行尾會轉換為空格。同時存在開頭與結尾空格時會被移除。程式碼行內的反斜線跳脫視為字面文字。
- **強調與加強強調**：`*`/`_` 為 `<em>`，`**`/`__` 為 `<strong>`。`_` 不允許用於詞內強調。適用左側/右側對齊分隔符運作規則。當一個分隔符既可開啟又可閉合時 (除非兩個長度皆為 3 的倍數)，分隔符運作長度總和不得為 3 的倍數。
- **刪除線 (擴充)**：`~~text~~` — 一或兩個波浪號。不跨段落。三個以上波浪號不會建立刪除線。
- **連結**：內聯 `[text](url "title")` 或參考 `[text][label]` / `[text][]` / `[text]`。連結文字可包含內聯元素，但不可包含其他連結。`<…>` 中的目的地允許空格。連結文字與 `(` 或 `[` 之間不得有空白。
- **圖像**：`![alt](src "title")` — 與連結語法相同，前方加上 `!`。Alt 文字為描述內容的純字串。
- **自動連結**：角括號內的 `<URI>` 或 `<email>`。Scheme 必須為 2–32 個字元，以 ASCII 字母開頭。
- **自動連結 (擴充)**：裸露的 `http://`, `https://`, `www.` URL 與裸露的電子郵件位址會自動連結，無須角括號。排除結尾標點符號；括號平衡。
- **原始 HTML**：開啟/閉合標籤、註解 (`<!-- -->`)、處理指令 (`<? ?>`)、宣告 (`<!…>`)、CDATA (`<![CDATA[…]]>`) 原樣傳遞。
- **禁止的原始 HTML (擴充)**：`<title>`, `<textarea>`, `<style>`, `<xmp>`, `<iframe>`, `<noembed>`, `<noframes>`, `<script>`, `<plaintext>` 會將開頭的 `<` 取代為 `&lt;`。
- **強制換行**：行尾前的兩個以上空格或 `\`。在程式碼行或 HTML 標籤中不識別。
- **軟換行**：行尾未預先接兩個以上空格或 `\` 的行。瀏覽器中渲染為空格。

## 驗證檢查清單

- [ ] ATX 標題使用 1–6 個 `#` 後接一個空格。
- [ ] 圍欄式程式碼區塊指定語言識別碼，並使用匹配的圍欄字元與數量。
- [ ] 表格包含標頭與分隔列，欄數匹配。對齊方式於分隔符中使用 `:` 設定。
- [ ] 任務清單項目在 `-` 與 `[ ]` 或 `[x]` 之間有空格。
- [ ] 強調於詞內使用 `*`；`_` 僅用於詞邊界。
- [ ] 刪除線精確使用 `~~` (非 3 個以上波浪號)。
- [ ] 連結使用 `[text](url)` 或參考語法，且在 `(` 或 `[` 前無空白。
- [ ] 無禁止的原始 HTML 標籤 (`<script>`, `<style>`, `<title>`, `<textarea>`, `<xmp>`, `<iframe>`, `<noembed>`, `<noframes>`, `<plaintext>`)。
