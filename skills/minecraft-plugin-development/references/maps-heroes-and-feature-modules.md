# 地圖、英雄與功能模組

在建構地圖系統、競技場輪替、職業系統、工具包 (Kits) 或模組化遊戲功能時，請參考此文件。

## 真實外掛程式中的地圖模式

### 每一場遊戲的地圖執行個體 (Map-instance) 用法

觀察到的特徵：

- 每一場遊戲獨立的地圖執行個體
- 透過設定允許地圖篩選
- 與活動地圖執行個體掛鉤的遊戲物件
- 比賽模式可以限制使用的地圖

這適用於每個遊戲都擁有自己的世界和目標的隔離比賽執行個體。

### 持續對戰場地的地圖輪替

觀察到的特徵：

- 一次只有一個活動的戰鬥世界
- 將來源地圖複製到臨時活動世界中
- 輪替後卸載並刪除舊世界
- 輪替警告在更換前廣播
- 輪替後重新整理出生點排行榜

這適用於世界按計時器輪替的持續共享模式。

### 空島競技場 (Sky-island arena) 地圖用法

觀察到的特徵：

- 一個遊戲擁有一份複製的地圖執行個體
- 團隊被分配到設定好的島嶼出生位置
- 箱子按島嶼、中間或中心角色分組
- 倒數計時控制籠子移除、遊戲開始、補給時間和遊戲結束
- 可見性和聊天範圍僅限於同一遊戲執行個體中的玩家

這適用於每個比賽都需要隔離島嶼、戰利品、觀戰者和清理工作的 SkyWars 風格模式。

### 地牢節點 (Dungeon-node) 地圖用法

觀察到的特徵：

- 一個遊戲擁有一個大廳地圖加上臨時的戰鬥或事件節點地圖
- 路徑選擇是根據階段 Metadata 產生的
- 玩家在推進前進行投票或確認
- 每個戰鬥節點擁有自己的波次佇列和活動怪物組
- 舊的節點地圖僅在玩家移動到新地圖或大廳後才卸載

這適用於 PvE 類 Rogue (Roguelike)、地牢、波次或 Boss 進度外掛程式。

## 地圖架構指引

- 每個執行個體的小遊戲：
  - 使用由遊戲物件擁有的 `GameMap`
- 共享輪替戰場：
  - 使用具有一個活動世界加上輪替計時器的 `MapManager`
- 臨時複製的世界：
  - 務必在卸載前將玩家傳送出去
  - 卸載後清理資料夾
  - 建立世界後重新套用遊戲規則 (Gamerules)
- 空島比賽：
  - 按團隊或島嶼角色對出生點和箱子位置進行分組
  - 每個比賽重置籠子、背包、計分板和觀戰者
- 地牢或路徑節點模式：
  - 將每個節點視為一個具有明確載入、進入、清除和卸載步驟的臨時階段
  - 保持怪物所有權與遊戲掛鉤，而不僅僅是與世界掛鉤

## 職業與英雄系統模式

觀察到的英雄系統特徵：

- `HeroRegistry` (英雄註冊表)
- `HeroService` (英雄服務)
- 按主題分組的定義
- 英雄階級進度
- 英雄技能設定和處理器
- 選擇器 GUI 與分配邏輯分離

觀察到的小遊戲能力選擇相似之處：

- 品牌和特殊物品的功能類似於模組化的玩家能力
- 選擇限制和類別是明確的
- 比賽規則可以限制可用的選擇

指引：

- 將定義與執行階段分配分離
- 使用註冊表來管理可發現的內容
- 將解鎖規則和階級儲存在資料中，而不是硬編碼在監聽器分支中
- 分離以下內容：
  - 什麼是職業
  - 如何選擇職業
  - 如何套用職業
  - 如何觸發其主動技能

## 功能模組模式

適合拆分為獨立套件的候選模組：

- 地圖輪替
- 英雄或職業系統
- 物品能力
- Boss 系統
- 比賽規則
- 商店和 GUI
- 計分板
- 進度系統

不要將所有這些都合併到一個監聽器或一個 "game utils" 類別中。

## 實用的啟發式方法

如果一項功能具備以下全部三項，則它值得擁有自己的模組：

- 自訂資料模型
- 設定或定義
- 一個或多個監聽器、命令或排程任務
