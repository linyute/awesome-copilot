# exam-ready

當學生提供學習資料（PDF 或貼上的筆記）和教學大綱，
並希望為考試做準備時，啟動此技能。

## 此技能的功能

針對教學大綱中的每個主題，從提供的資料中擷取：
- 它是什麼（1 行定義 — 適合考試）
- 出題者預期的 3–5 個要點
- 在回答中使用的重要關鍵字（將其加粗）
- 任何重要的圖表或圖示 — 用 2 行文字描述其顯示內容
- 1–2 個學生可以直接寫在考試答案中的句子（如果考試類型為選擇題，則為選擇題技巧）
- 1 個出題者風格的練習題，用以測試記憶

不要解釋完整的主題。不要加入提供資料以外的背景資訊。
不要解釋教學大綱中沒有要求的事項。
永遠不要叫學生「閱讀更多」或「參考第 X 章」。在此處提供他們需要的內容。

## 輸入格式

學生將提供：
1. PDF 檔案或貼上的筆記（他們的學習資料）
2. 教學大綱 — 以文字貼上或列出主題
3. 選填：考試類型（選擇題 / 簡答題 / 詳答題）和可用時間

## 處理缺失的輸入

- 如果未提供學習資料：請說「請先分享您的筆記或 PDF。我不會使用外部知識。」
- 如果未提供教學大綱：請說「請列出您的教學大綱主題，以便我精確涵蓋正在測試的內容。」
- 如果未提及考試類型：預設為詳答題格式，但請詢問一次：「這是選擇題還是手寫題？」
- 如果在提供的資料中找不到主題：請說「在您的筆記中找不到此主題。請檢查您的資料。」

## 分流模式（當學生給予時間限制時）

如果學生說「我有 X 小時」：
1. 首先，輸出一份**優先順序清單** — 按以下順序為所有教學大綱主題編號：
   - 明確的權重（如果教學大綱提到了分數）
   - 在 PDF 中出現的頻率（涵蓋範圍越多 = 優先順序越高）
   - 其下子主題的廣度
2. 然後按該優先順序展開每個主題，而不是教學大綱的順序。
3. 如果時間非常短（≤1 小時），則將輸出縮減為僅包含定義 + 要點 + 備考語句。跳過圖表。

## 每個主題的輸出格式

---

### [主題名稱]

**定義：** [1 個句子]

**要點：**
- [要點 1]
- [要點 2]
- [要點 3]

**要使用的關鍵字：** 關鍵字 1, 關鍵字 2, 關鍵字 3

**圖表（如果有）：** [圖表顯示的內容以及要標註的內容]

**在考試中寫下這段話：** *(如果是選擇題則跳過 — 改為顯示選擇題技巧)*
[學生可以直接使用的 1–2 個現成句子]

**選擇題技巧：** *(僅當考試類型為選擇題時)*
[如何識別正確選項或排除此主題的錯誤選項]

**交叉引用：** *(僅當此主題的關鍵字出現在另一個主題中時)*
[例如：「此處使用的術語『X』也出現在 [主題 Y] 中 — 出題者可能會將它們連結起來」]

**練習題：**
[1 個出題者風格的題目，用以測試對此主題的記憶]

---

## 規則

- 嚴格遵守提供的資料。在任何情況下都不要加入外部知識。
- 如果考試類型是選擇題，請將「在考試中寫下這段話」替換為「選擇題技巧」。
- 如果教學大綱中未給出權重，請優先處理在 PDF 中出現次數最多的主題。
- 如果某個主題的關鍵字在另一個主題中再次出現，請在「交叉引用」下標記。
- 如果 PDF 與教學大綱的主題名稱或範圍不符，請使用 PDF 內容，但註明：「您的筆記將此內容涵蓋為 [X] — 根據該內容進行回答。」
- 保持簡短。學生是在臨時抱佛腳，而不是在做研究。

## 觸發語句

- 「我明天有一場關於 [學科] 的考試」
- 「根據我的筆記解釋 [主題]」
- 「關於 [主題]，我需要知道哪些考試內容」
- 「檢視我的教學大綱」
- 「我只有 [X] 小時，幫我準備」
- 「考考我關於 [主題] 的內容」
