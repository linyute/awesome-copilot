# LinkedIn 貼文格式化工具 (LinkedIn Post Formatter)

使用 Unicode 字體和經實證的結構模式，將原始內容、想法或技術材料轉化為精緻且經過參與度優化的 LinkedIn 貼文。

## 概覽

LinkedIn 僅支援純文字 — 不支援 Markdown 渲染，也不支援富文本格式。此技能使用 Unicode 數學字母數字符號來模擬粗體、斜體和粗斜體文字，這些文字可在無需任何外部工具的情況下，直接在 LinkedIn 編輯器中原生呈現。

## Unicode 字體參考

在將純文字轉換為 Unicode 樣式的 LinkedIn 文字時，請先載入並使用 `references/unicode-charmap.md` 作為權威的字元映射參考。

套用這些字元映射以在純文字中建立視覺強調：

### 粗體 (數學無襯線粗體 - Mathematical Sans-Serif Bold)

對關鍵片語、區段標題和強調詞使用粗體。

| 純文字 | Unicode 粗體 |
|-------|-------------|
| A-Z   | 𝗔-𝗭         |
| a-z   | 𝗮-𝘇         |
| 0-9   | 𝟬-𝟵         |

### 斜體 (數學無襯線斜體 - Mathematical Sans-Serif Italic)

對細微的強調、技術術語或引用使用斜體。

| 純文字 | Unicode 斜體 |
|-------|---------------|
| A-Z   | 𝘈-𝘡           |
| a-z   | 𝘢-𝘻           |

### 粗斜體 (數學無襯線粗斜體 - Mathematical Sans-Serif Bold Italic)

謹慎使用以獲得最大程度的強調。

| 純文字 | Unicode 粗斜體 |
|-------|-------------------|
| A-Z   | 𝘼-𝙕               |
| a-z   | 𝙖-𝙯               |

## 視覺分隔線

使用這些字元建立視覺結構：

- **區段分隔線**：`━━━━━━━━━━━━━━━━━━━━━━`（方框繪製粗水平線）
- **項目符號**：`◈`（帶點菱形）或 `◎`（靶心）
- **箭頭流程**：`↓` 用於垂直流程，`→` 用於水平延續
- **子項目**：`↳` 用於縮進的子項
- **編號項目**：使用粗體 Unicode 數字 `𝟭. 𝟮. 𝟯.` 等。

## 格式化規則

1. **換行很重要**：LinkedIn 會摺疊多個空行。段落之間請使用單個空行。
2. **鉤子需在「折疊線」上方**：前 2-3 行必須吸引讀者點擊「查看更多」。將價值前置。
3. **短段落**：每段最多 1-3 句話。文字牆會扼殺參與度。
4. **謹慎使用粗體**：強調關鍵片語和標題，而非整個段落。
5. **斜體用於細微差別**：對技術術語、內部想法或細微強調使用斜體。
6. **標籤置於末尾**：最後一行放 5-8 個相關標籤。貼文中間不要放標籤。
7. **本文中不使用表情符號**，除非使用者明確要求。例外：CTA 中可使用一個策略性表情符號（♻️ 用於轉發）。
8. **字數限制**：LinkedIn 貼文上限為 3000 字元。為獲得最佳參與度，建議目標設為 1500-2500 字元。
9. **本文中不放置 URL**：LinkedIn 會降低帶有連結貼文的觸及率。請將連結放在留言中。在 CTA 中提到「連結在留言處」或「在下方獲取」。

## 參與度優化

- **奏效的開場鉤子**：問題、強烈主張、「如果你在做 X...」、反直覺觀點、令人驚訝的數據。
- **奏效的結尾 CTA**： 「♻️ 𝗥𝗲𝗽𝗼𝘀𝘁 如果...」、「儲存此貼文供日後參考」、「標記需要此內容的人」、「你的看法為何？👇」
- **留白是你的好朋友**：密集的文字會被快速滑過。通透、易於掃視的佈局能勝出。
- **「查看更多」鉤子**：在桌面版上，LinkedIn 會在約 210 個字元後截斷貼文。確保前兩行能產生足夠的好奇心引導點擊。

## 流程

1. 分析來源內容（文字、HTML、圖片或想法）。
2. 識別最佳貼文結構模式（鉤子→內容→CTA、清單文、故事→啟示、資源分享）。
3. 提取核心訊息和 3-5 個關鍵點。
4. 使用 `references/unicode-charmap.md` 對標題和強調詞套用 Unicode 粗體/斜體格式化。
5. 在各區段之間加入視覺分隔線。
6. 為開場撰寫引人入勝的鉤子。
7. 在末尾加入 CTA 和標籤。
8. 驗證貼文已就緒，可直接複製貼上至 LinkedIn。
