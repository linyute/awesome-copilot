# 縮放以提升查詢量

問題：當查詢具有極大的 limit (例如：1000) 且存在多個分片 (例如：10) 時，樸素的做法是每個分片都必須回傳完整的 1000 個結果 — 總共傳輸和合併 10,000 個計分點。由於資料隨機分佈在自動分片中，這種做法非常浪費資源。

## 核心構想

與其要求每個分片都回傳完整的 limit，不如要求每個分片回傳一個根據卜瓦松分佈 (Poisson distribution) 統計計算出的較小 limit，然後進行合併。這種做法是安全的，因為自動分片保證了隨機、獨立的資料分佈。

## 何時啟用

- 分片數量大於 1
- 正在使用自動分片 (所有查詢的分片共用相同的分片鍵)
- 請求的 limit + offset >= SHARD_QUERY_SUBSAMPLING_LIMIT (128)
- 查詢不是精確搜尋 (exact search)

## 關鍵權衡

 此策略以極小的機率回傳略微不完整的結果為代價，大幅減少了分片間的資料傳輸量，特別是對於跨多個分片的高 limit 查詢。1.2 倍的安全性係數和 99.9% 的卜瓦松閾值將錯誤率保持在極低水平 — 與 HNSW 等近似向量索引已經引入的不精確性相當。
