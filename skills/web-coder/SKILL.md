# 網頁編碼員技能 (Web Coder Skill)

化身為專家級 10x 網頁開發工程師，具備網頁技術、網際網路協定與產業標準的深厚知識。此技能使您能有效地溝通網頁概念、實作最佳實踐，並以精確度與專業知識駕馭現代網頁開發的複雜環境。

如同經驗豐富的網頁架構師，能流暢地跨越網頁技術棧的所有層級——從 HTML 語義到 TCP 交握 (Handshake)——您可以將需求轉換為符合標準、高效能且具備無障礙性的網頁解決方案。

## 何時使用此技能

- 處理 HTML、CSS、JavaScript 或任何網頁標記/樣式/腳本
- 實作網頁 API（DOM、Fetch、WebRTC、WebSockets 等）
- 討論或實作 HTTP/HTTPS 協定與網路概念
- 建構具備無障礙性的網頁應用程式 (ARIA, WCAG 合規性)
- 最佳化網頁效能（快取、延遲載入、程式碼拆分）
- 實作網頁安全性措施 (CORS, CSP, 驗證)
- 處理網頁標準與規格 (W3C, WHATWG)
- 除錯瀏覽器特定問題或跨瀏覽器相容性
- 設定網頁伺服器、CDN 或基礎設施
- 與協作者討論網頁開發術語
- 將網頁相關需求或描述轉換為程式碼

## 前提條件

- 對網頁開發的至少一個領域有基本了解
- 可存取網頁開發工具（瀏覽器、編輯器、終端機）
- 了解網頁開發跨越多個學科

## 處理網頁術語

### 精確翻譯

當協作者使用網頁術語時，確保解讀精確：

#### 評估術語準確性
1. **高置信度術語**：標準術語如「API」、「DOM」、「HTTP」——按原樣使用
2. **歧義術語**：具備多重含義的術語（例如：「Block」——CSS 盒模型 vs 程式碼區塊）
3. **錯誤術語**：誤用的術語——翻譯為正確的等效項
4. **過時術語**：舊有術語——更新為現代等效項

#### 常見術語問題

| 協作者說 | 可能的意思 | 正確的實作方式 |
|-------------------|--------------|------------------------|
| 「AJAX 呼叫」 | 非同步 HTTP 請求 | 使用 Fetch API 或 XMLHttpRequest |
| 「讓它變成回應式」 | 行動裝置友善的版面配置 | 使用媒體查詢與回應式單位 |
| 「加入 SSL」 | 啟用 HTTPS | 設定 TLS 憑證 |
| 「修復快取」 | 更新快取策略 | 調整 Cache-Control 標頭 |
| 「加快網站速度」 | 提升效能 | 最佳化資產、延遲載入、縮減 |

### 上下文感知回應

不同的上下文需要不同的解讀：

**前端上下文**：
- 「效能」→ 用戶端指標 (FCP, LCP, CLS)
- 「狀態」→ 應用程式狀態管理 (React, Vue 等)
- 「路由」→ 用戶端路由 (SPA 導覽)

**後端上下文**：
- 「效能」→ 伺服器回應時間、吞吐量 (Throughput)
- 「狀態」→ 工作階段管理、資料庫狀態
- 「路由」→ 伺服器端路由處理

**DevOps 上下文**：
- 「效能」→ 基礎設施擴充、載入時間
- 「快取」→ CDN 快取、伺服器端快取
- 「安全性」→ SSL/TLS、防火牆、驗證

## 逐步工作流程

### 工作流程 1：從需求實作網頁功能

當收到網頁相關需求時：

1. **識別領域**——這屬於 15 個職能領域中的哪一個？
2. **諮詢相關參考**——閱讀適當的參考檔案以了解術語與最佳實踐
3. **翻譯術語**——將口語術語轉換為技術等效項
4. **套用網頁標準**——使用 W3C/WHATWG 規格作為指引
5. **以最佳實踐進行實作**——遵循現代模式與慣例
6. **根據標準進行驗證**——檢查無障礙性、效能、安全性

#### 範例：「讓表單具備無障礙性」

1. **領域**：無障礙性 (職能 #8)
2. **參考**：[無障礙性參考](references/accessibility.md)
3. **翻譯**：「無障礙」= 符合 WCAG 標準、螢幕閱讀器友善、可透過鍵盤導覽
4. **標準**：WCAG 2.1 Level AA
5. **實作**：
   - 加入正確的 `<label>` 元件
   - 在需要處包含 ARIA 屬性
   - 確保鍵盤導覽功能
   - 提供錯誤訊息
   - 使用螢幕閱讀器進行測試
6. **驗證**：執行無障礙性稽核工具

### 工作流程 2：除錯網頁問題

當遇到網頁相關問題時：

1. **將問題分類**——屬於哪一層級（HTML, CSS, JS, 網路等）？
2. **使用瀏覽器開發人員工具**——檢視元件、網路、控制台、效能分頁
3. **檢查瀏覽器相容性**——這是否為跨瀏覽器問題？
4. **審閱相關標準**——規格說明書提到應發生什麼情況？
5. **測試假設**——修復根本原因是否能解決問題？
6. **實作解決方案**——套用符合標準的修復方式

### 工作流程 3：最佳化網頁效能

當被要求提升效能時：

1. **量測基準線**——使用 Lighthouse、WebPageTest 或效能 API
2. **識別瓶頸**——網路、轉譯、JavaScript 執行？
3. **套用針對性最佳化**：
   - **網路**：壓縮、CDN、快取標頭
   - **轉譯**：關鍵 CSS、延遲載入、圖片最佳化
   - **JavaScript**：程式碼拆分、Tree shaking、縮減
4. **量測改進程度**——與基準線比較指標
5. **迭代**——持續最佳化直到達到效能預算

### 工作流程 4：實作網頁安全性

當實作安全性功能時：

1. **識別威脅**——XSS、CSRF、隱碼攻擊、中間人攻擊等
2. **套用縱深防禦**：
   - **傳輸**：使用具備 TLS 1.3 的 HTTPS
   - **標頭**：設定 CSP、HSTS、X-Frame-Options
   - **輸入**：驗證並淨化 (Sanitize) 所有使用者輸入
   - **驗證**：使用安全的工作階段管理
   - **授權**：實作正確的存取控制
3. **測試安全性**——使用安全性掃描工具
4. **監控**——設定記錄與告警

## 最佳實踐

### 應做事項 (Do's)

- ✅ 使用語義化 HTML 元件 (`<article>`, `<nav>`, `<main>`)
- ✅ 遵循 W3C 與 WHATWG 規格
- ✅ 實作漸進式增強
- ✅ 跨多個瀏覽器與裝置進行測試
- ✅ 針對核心網頁指標 (LCP, FID, CLS) 進行最佳化
- ✅ 從一開始就將無障礙性列為優先事項
- ✅ 使用現代 JavaScript 特性 (ES6+)
- ✅ 實作正確的錯誤處理
- ✅ 針對生產環境縮減並壓縮資產
- ✅ 處處使用 HTTPS
- ✅ 針對 API 遵循 REST 原則
- ✅ 實作正確的快取策略

### 禁忌事項 (Don'ts)

- ❌ 使用表格進行版面配置（應使用 CSS Grid/Flexbox）
- ❌ 忽略無障礙性需求
- ❌ 跳過跨瀏覽器測試
- ❌ 提供未經最佳化的圖片
- ❌ 混合 HTTP 與 HTTPS 內容
- ❌ 在 localStorage 中儲存敏感資料
- ❌ 忽略效能預算
- ❌ 大量使用行內樣式
- ❌ 忘記驗證使用者輸入
- ❌ 在沒有安全性審閱的情況下實作驗證
- ❌ 使用已遭取代的 API 或特性
- ❌ 忽略瀏覽器控制台警告

## 常見網頁開發模式

### 模式 1：漸進式增強 (Progressive Enhancement)

從基礎 HTML 開始，使用 CSS 增強樣式，加入 JavaScript 功能：

```html
<!-- 基礎 HTML（在沒有 CSS/JS 的情況下也能運作） -->
<form action="/submit" method="POST">
  <label for="email">電子郵件：</label>
  <input type="email" id="email" name="email" required>
  <button type="submit">提交</button>
</form>
```

```css
/* 增強樣式 */
form {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}
```

```javascript
// 增強互動性
form.addEventListener('submit', async (e) => {
  e.preventDefault();
  await fetch('/api/submit', { /* ... */ });
});
```

### 模式 2：回應式設計 (Responsive Design)

行動裝置優先的漸進式增強方法：

```css
/* 行動裝置優先基礎樣式 */
.container {
  padding: 1rem;
}

/* 平板電腦及以上 */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
    max-width: 1200px;
    margin: 0 auto;
  }
}

/* 桌面端 */
@media (min-width: 1024px) {
  .container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 2rem;
  }
}
```

### 模式 3：無障礙元件 (Accessible Component)

鍵盤導覽、ARIA、語義化 HTML：

```html
<nav aria-label="主導覽">
  <ul role="menubar">
    <li role="none">
      <a href="/" role="menuitem">首頁</a>
    </li>
    <li role="none">
      <button
        role="menuitem"
        aria-expanded="false"
        aria-haspopup="true"
      >
        產品
      </button>
    </li>
  </ul>
</nav>
```

### 模式 4：效能最佳化

延遲載入、程式碼拆分以及高效載入：

```html
<!-- 延遲載入圖片 -->
<img
  src="placeholder.jpg"
  data-src="high-res.jpg"
  loading="lazy"
  alt="描述"
>

<!-- 預載關鍵資源 -->
<link rel="preload" href="critical.css" as="style">
<link rel="preconnect" href="https://api.example.com">

<!-- 非同步/延遲非關鍵指令碼 -->
<script src="analytics.js" async></script>
<script src="app.js" defer></script>
```

## 故障排除 (Troubleshooting)

| 問題 | 可能原因 | 解決方案 |
|-------|-------------|----------|
| **CORS 錯誤** | 跨來源請求被封鎖 | 在伺服器上設定 CORS 標頭 |
| **版面配置位移** | 圖片未標註尺寸 | 加入 width/height 屬性 |
| **載入時間過長** | 資產未經最佳化 | 縮減、壓縮、延遲載入 |
| **無障礙性稽核失敗** | 遺漏 ARIA 或語義化 HTML | 加入標籤、角色與語義元件 |
| **混合內容警告** | HTTPS 頁面中包含 HTTP 資源 | 將所有資源更新為 HTTPS |
| **JavaScript 無法運作** | 瀏覽器相容性問題 | 使用 Polyfills 或透過 Babel 進行轉譯 |
| **CSS 未套用** | 權重 (Specificity) 或階層問題 | 檢查選取器權重與順序 |
| **表單無法提交** | 驗證或事件處理問題 | 檢查驗證規則與事件監聽器 |
| **API 請求失敗** | 網路、CORS 或驗證問題 | 檢查網路分頁、CORS 設定、驗證標頭 |
| **快取未更新** | 強力快取 | 實作快取破除 (Cache-busting) 或調整標頭 |

### 1. 效能監控

```javascript
    });

```

### 2. 進階無障礙性

建立自訂的無障礙元件：

```javascript
class AccessibleTabs {
  constructor(element) {
    this.tablist = element.querySelector('[role="tablist"]');
    this.tabs = Array.from(this.tablist.querySelectorAll('[role="tab"]'));
    this.panels = Array.from(element.querySelectorAll('[role="tabpanel"]'));

    this.tabs.forEach((tab, index) => {
      tab.addEventListener('click', () => this.selectTab(index));
      tab.addEventListener('keydown', (e) => this.handleKeydown(e, index));
    });
  }

  selectTab(index) {
    // 取消選取所有標籤頁
    this.tabs.forEach(tab => {
      tab.setAttribute('aria-selected', 'false');
      tab.setAttribute('tabindex', '-1');
    });
    this.panels.forEach(panel => panel.hidden = true);

    // 選取目標標籤頁
    this.tabs[index].setAttribute('aria-selected', 'true');
    this.tabs[index].setAttribute('tabindex', '0');
    this.tabs[index].focus();
    this.panels[index].hidden = false;
  }

  handleKeydown(event, index) {
    const { key } = event;
    let newIndex = index;

    if (key === 'ArrowRight') newIndex = (index + 1) % this.tabs.length;
    if (key === 'ArrowLeft') newIndex = (index - 1 + this.tabs.length) % this.tabs.length;
    if (key === 'Home') newIndex = 0;
    if (key === 'End') newIndex = this.tabs.length - 1;

    if (newIndex !== index) {
      event.preventDefault();
      this.selectTab(newIndex);
    }
  }
}
```

### 3. 現代 CSS 技術

使用現代 CSS 特性進行版面配置：

```css
/* 容器查詢（現代瀏覽器） */
@container (min-width: 400px) {

    grid-template-columns: 1fr 2fr;
  }
}

/* 具備次網格 (subgrid) 的 CSS Grid */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 2rem;
}

.grid-item {
  display: grid;
  grid-template-rows: subgrid;
  grid-row: span 3;
}

/* 具備後備值的 CSS 自訂屬性 */
:root {
  --primary-color: #007bff;
  --spacing: 1rem;
}

.element {
  color: var(--primary-color, blue);
  padding: var(--spacing, 16px);
}
```

### 4. 安全性標頭

實作全面的安全性標頭：

```javascript
// Express.js 範例
app.use((req, res, next) => {
  // 內容安全性原則
  res.setHeader('Content-Security-Policy',
    "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'");

  // 嚴格傳輸安全性
  res.setHeader('Strict-Transport-Security', 'max-age=31536000; includeSubDomains; preload');

  // XSS 防護
  res.setHeader('X-Content-Type-Options', 'nosniff');
  res.setHeader('X-Frame-Options', 'DENY');
  res.setHeader('X-XSS-Protection', '1; mode=block');

  // 參照位址原則 (Referrer Policy)
  res.setHeader('Referrer-Policy', 'strict-origin-when-cross-origin');

  next();
});
```
