# 定位策略 (Positioning Strategy)

尋找並佔領可防禦的市場地位。將大眾化的訊息轉化為清晰的差異化 — 或者至少在投入資源之前，先測試你的差異化是否真的能引起共鳴。

## 何時使用

**觸發情境：**
- 「我們的訊息聽起來和競爭對手一模一樣」
- 「品牌知名度很強但轉換很弱」
- 「銷售團隊無法解釋我們有何不同」
- 「買家認為我們可以被輕易取代」
- 「我們應該在品牌重塑前先重新定位嗎？」
- 「我們該如何測試定位主張？」

**背景：**
- 產品大同小異的競爭市場
- 無法產生轉換的訊息
- 新產品發布
- 既有產品的重新定位
- 銷售團隊回報買家感到困惑

## 核心框架

### 1. 一個詞就能改變一切（「自動化」帶來的問題）

**模式：**

在一款自動化 AI 產品的早期企業溝通中，我們將其定位為「自動化 AI 代理人 (autonomous AI agent)」。

開發者：「很酷，但有點可怕。」
經理：「這會取代我們的團隊嗎？」
交易進展：緩慢。充斥著「我們會再考慮看看」。

**變更：**

改動一個詞：「自動化 (autonomous)」→「AI 團隊成員 (AI teammate)」

同樣的產品，同樣的能力，但不同的框架。

**結果：**

開發者：「這能幫到我。」
經理：「這能讓我的團隊更有生產力。」
交易進展：明顯變快。

**為何這很重要：**

定位不在於你做了什麼，而在於你 **沒說什麼**。

我們本可以說「取代開發者」（就某些任務而言技術上是真的），但這會扼殺每一筆企業交易。

**框架：選詞形塑買家心理**

**會嚇跑企業的詞彙：**
- 自動化 (Autonomous)（隱含：失去控制，取代人類）
- 取代 (Replaces)（威脅：工作保障）
- 全自動 (Fully automated)（移除：人類判斷）
- AI 優先 (AI-first)（意指：不明確，流行語）

**能促成轉換的詞彙：**
- 團隊成員 (Teammate)（隱含：協作，協助）
- 增強 (Augments)（協助：讓人變得更好）
- 一切由您控制 (You stay in control)（安心：人類監督）
- 處理重複性工作 (Handles repetitive work)（具體：節省時間）

**如何測試選詞：**

不要用猜的，去測試。

**測試 1：外展電郵 A/B 測試**
- 寄給 100 位潛在客戶版本 A（「自動化代理人」）
- 寄給 100 位潛在客戶版本 B（「AI 團隊成員」）
- 衡量：回覆率、預約會議率
- 訊號強度：高（真實的買家意圖）

**測試 2：網站首頁 A/B 測試**
- 版本 A：目前的定位
- 版本 B：新的選詞
- 衡量：關鍵行動呼籲 (CTA) 的點擊率
- 時長：最少 1-2 週
- 訊號強度：中等（有興趣但未承諾）

**測試 3：銷售通話腳本**
- 一半的客戶經理 (AE) 使用定位 A
- 另一半使用定位 B
- 衡量：演示到試用的轉換率
- 訊號強度：高（真實的銷售週期）

**常見錯誤：**

根據內部共識而非客戶回饋來更改定位。你的團隊不是買家。

### 2. 在承諾前先測試（「爬、走、跑」定位推行策略）

**模式：**

定位變更伴隨著風險：品牌混亂、銷售步調不一、客戶流失（如果既有客戶認不出你）。

**透過階段性推行來降低風險：**

**爬 (Crawl) 階段（1-2 週）：驗證**

在不投入產品/組織資源的情況下測試訊息。

**行動：**
- A/B 測試網站標題（新版 vs 現有版）
- 對冷啟動潛在客戶執行兩個不同定位角度的外展電郵序列
- 詢問現有客戶：「如果我們將自己描述為 [新定位]，你還認得出我們嗎？」

**衡量：**
- 追蹤網站變體的點擊率 (CTR)
- 追蹤外展序列的回覆率
- 記錄質性回饋

**通過/終止標準：**
- 至少有一個定位變體在回覆率上優於現有版 20% 以上
- 現有客戶不會說「等等，這不是你們在做的」
- 若有明確贏家則繼續；若平手，則拉長測試時間或測試新角度

**走 (Walk) 階段（2-3 週）：對齊**

如果測試獲得驗證，則將產品與銷售與新定位對齊（但尚未公開重塑品牌）。

**行動：**
- 重寫網站文案（首頁、企業頁面、CTA）
- 製作銷售賦能工具：更新的投售簡報 (pitch deck)、通話腳本、電郵範本
- 更新文件以符合新敘事
- 建立特定使用案例的範例

**衡量：**
- 銷售團隊對訊息易用性的回饋
- 網站參與度的分析（尚非轉換率）
- 細分市場分析（誰在回應？）

**通過/終止標準：**
- 銷售團隊表示新訊息更易於使用
- 點擊率指標優於現有版
- 沒有重大的客戶困惑
- 進入跑階段

**跑 (Run) 階段（2-3 週及持續進行）：規模化**

全面投入。這就是品牌重塑。

**行動：**
- 針對各個使用案例發布專屬落地頁面
- 針對各個定位角度發動外展活動
- 更新所有面對客戶的材料
- 針對新敘事培訓客戶成功團隊
- 宣布重新定位（若合適）

**衡量：**
- 按定位角度劃分的案源量
- 按定位角度劃分的贏單率
- 各管道的獲客成本 (CAC) 效率
- 客戶留存率（我們有失去任何人嗎？）

**常見錯誤：**
- 跳過爬階段（在未經驗證前直接進行全面品牌重塑）
- 同時進行多個階段（如果訊息在推行中期改變會造成混亂）
- 等待完美的產品才重新定位（產品應追隨定位，而非領先定位）
- 未能在每個階段進行衡量（無法確定測試是否「獲勝」）

### 3. 定位清晰度診斷

**模式：**

如果你的訊息與競爭對手的訊息極度相似，那麼你面臨的是定位問題，而非產品問題。

**定位失效的表現：**
- 所有競爭對手描述的價值主張幾乎一模一樣
- 差異化需要解釋複雜的技術細節
- 買家認為產品可以互換
- 行銷指標（點擊率、參與度）低於產業基準
- 銷售對話常因比較問題而偏離主題

**如何執行：**

**步驟 1：競爭對手訊息審計**
- 收集 5-7 個直接競爭對手的首頁標題
- 識別共通的主張（這些已商品化）
- 對應競爭對手聲稱具備獨特價值之處

**範例：**

如果每個人都說「最快」、「最可靠」、「最易於使用」— 這些只是入場券，而非差異化。

**步驟 2：評估你的實際優勢**
- 有什麼是你做得到而競爭對手在不投入大量研發下做不到的？
- 你最好的客戶為什麼選擇你？（去問他們）
- 你具備哪些結構性優勢？（部署模型、資料所有權、定價、網絡效應等）

**步驟 3：尋找服務不足的定位**
- 現有的解決方案在哪些方面辜負了使用者？
- 什麼問題是每個人都忽視的？
- 哪個買家細分市場服務不足？

**步驟 4：提出清晰的主張**

必須是：
- 你現在就能擁有的（而非未來的路線圖）
- 競爭對手無法輕易複製的（結構性優勢）
- 能與你最好的客戶群體產生共鳴的

**常見錯誤：**
- 聲稱你在每個人都在做的事情上做得「更好」（令人難以置信）
- 定位在競爭對手已具備的功能上
- 同時採用多個定位（請選一個）
- 在重新定位前等待完美的產品

---

### 4. 市場定位架構（三層）

**第一層：市場背景 (Market Context)**
- 市場正在經歷什麼問題？
- 為什麼現在會經歷這個問題？
- 如果問題不解決會發生什麼事？

**範例：**
「基礎架構團隊在混合環境中管理日益複雜的部署。組織紛紛採用微服務與分散式系統。這創造了傳統監控工具無法處理的維運複雜性。」

**第二層：定位陳述 (Positioning Statement)（1-2 句話）**
- **我們服務誰**：哪個客戶細分市場？
- **我們解決什麼問題**：具體的痛點
- **我們有何不同**：為什麼我們相對於替代方案很重要
- **證據**：為什麼他們應該相信我們？

**範例：**
「我們透過 [核心能力] 協助平台團隊更快出貨，該能力即時連結了 [工作流程 A]、[工作流程 B] 與 [業務成果]。」

**第三層：敘事 (Narrative)**

將定位擴展為故事：
- 世界為何正在改變
- 現有的解決方案為何無效
- 為什麼我們的方法更好
- 與我們合作的未來是什麼樣子

**如何執行：**

在測試前寫下全部三層。先使用「爬、走、跑」方法論測試第二層（定位陳述）。如果獲得驗證，再建立第三層。

### 5. 標題與副標題測試

**原則：** 清晰的定位需要可測試的結構：標題（你是什麼？）+ 副標題（為誰而做？為什麼？）。

**主要標題格式：**
- 「具備 [形容詞] 的 [類別]，能 [差異化點]」
- 「專為 [特定使用案例] 打造的 [產品]」
- 「能帶來 [核心利益] 的 [產品]」

**範例：**
- 「專為 [工作流程] 打造的可客製化平台」
- 「為自動化團隊打造的基礎架構」
- 「取代 [現有業者] 的企業級替代方案」

**紅燈訊號：**
- 使用競爭對手的名稱（具防衛性，不自信）
- 太過技術化（買家聽不懂）
- 主張多種利益（請選一個）
- 模糊（「X 的未來」— 令人難以置信）

**副標題的目的：**

釐清對象、原因、以及與現狀有何不同。

**範例：**
- 「隨處部署。即時擴展。你的基礎架構，你說了算。」
- 「致深受重複性工作所苦的團隊。自動化處理 80% 的工作，人類處理剩下的 20%。」
- 「企業級品質。無供應商鎖定。與您現有的技術堆疊相容。」

**如何測試：**

針對標題 + 副標題的組合進行 A/B 測試：
- 變體 A：目前的訊息
- 變體 B：新的定位角度
- 變體 C：不同的差異化主張

衡量點擊率、回覆率、轉換率。

根據資料而非意見挑選贏家。

---

### 6. 定位防禦力評估

**原則：** 定位只有在競爭對手無法輕易複製時才有價值。

**防禦力階層：**

**1. 結構性優勢（最強）**
- 難以複製：獨特的資料所有權、部署靈活性、定價模型、網絡效應
- 範例：「為受監管產業打造，具備地端部署能力」（如果不重構架構就無法複製）

**2. 市場地位（若為首創則強）**
- 如果你率先佔領並擴大規模，則具備防禦力
- 範例：「首款專為 [特定工作流程] 打造的 AI 平台」（仿效者會顯得像是衍生物）

**3. 產品功能（弱）**
- 容易複製：使用者體驗 (UX)、特定能力
- 範例：「更快的 API 呼叫」（競爭對手在 6 週內就能推出速度提升的功能）

**如何評估：**

針對每個定位主張，請自問：

1. 競爭對手能否透過單一產品衝刺週期來複製？（是 = 無防禦力）
2. 我們具備結構性優勢嗎？（否 = 暫時性的定位）
3. 考量目前的產品，這是否可信？（否 = 先不要宣稱）
4. 我們能否在競爭對手反應前佔領此位置？（否 = 太慢了）

**常見錯誤：**

定位在競爭對手可以輕易跟進的功能上。這會產生定位跑步機效應 — 你總是在防守，從未真正擁有。

## 決策樹

### 我們應該重新定位嗎？

```
品牌知名度很強但轉換很弱嗎？
├─ 是 → 定位問題，測試新角度
└─ 否 → 繼續...
    │
    我們的訊息聽起來和競爭對手一樣嗎？
    ├─ 是 → 定位問題
    └─ 否 → 非定位問題
```

### 我們應該測試哪種定位角度？

```
我們具備競爭對手無法複製的結構性優勢嗎？
├─ 是 → 針對結構性優勢進行定位
└─ 否 → 繼續...
    │
    我們是該類別的首創嗎？
    ├─ 是 → 針對類別所有權進行定位
    └─ 否 → 尋找服務不足的細分市場 / 使用案例
```

### 我們應該從爬階段移至走階段嗎？

```
新定位的表現是否優於現有版 20% 以上？
├─ 是 → 移至走階段（對齊階段）
└─ 否 → 繼續...
    │
    測試運行時間是否足夠（2 週以上）？
    ├─ 否 → 繼續測試
    └─ 是 → 嘗試不同的定位角度或維持現狀
```

## 常見錯誤

**1. 在每個人都在做的事情上聲稱做得「更好」**
   - 令人難以置信。請尋找不同的角度。

**2. 定位在容易被複製的功能上**
   - 競爭對手會跟進。你需要結構性優勢。

**3. 在重新定位前等待完美的產品**
   - 產品工作應追隨定位，而非領先定位。

**4. 同時測試太多定位角度**
   - 無法判斷什麼奏效。請一次測試一個。

**5. 跳過驗證階段**
   - 未經測試就直接進行全面品牌重塑 = 風險。

**6. 對所有買家畫像使用同一套定位**
   - 不同的畫像在意不同的事。

**7. 無法產生差異化的通用定位**
   - 「同類最佳 (Best-in-class)」、「創新 (innovative)」= 毫無意義。

## 快速參考

**爬、走、跑測試：**
- 爬（1-2 週）：A/B 測試訊息，衡量回覆率
- 走（2-3 週）：將銷售/產品與獲勝角度對齊
- 跑（持續進行）：全面重新定位，衡量案源/轉換

**能促成轉換的選詞：**
- ✅ 團隊成員、增強、一切由您控制
- ❌ 自動化、取代、全自動

**定位審計步驟：**
1. 收集競爭對手訊息
2. 識別你的實際優勢（詢問客戶）
3. 尋找服務不足的定位
4. 提出清晰、具防禦力的主張

**防禦力階層：**
1. 結構性優勢（獨特資料、部署靈活性、定價模型）
2. 市場地位（類別所有權）
3. 產品功能（最弱，容易複製）

**測試階層（訊號強度）：**
1. 外展回覆率（最高）
2. 銷售通話轉換率（高）
3. 網站點擊率（中等）
