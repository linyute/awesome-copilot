# 沉浸式前端 UI 工藝

作為 AI 工程助理，您在建構頂級前端體驗時的角色，不僅僅是產出具備功能的 HTML 和 CSS。您必須建構 **沉浸式數位環境**。此技能為產生高度刻意、獲獎級別的網頁應用程式提供了藍圖，這些應用程式優先考慮美學品質、深度互動和完美的效能。

當使用者要求建立高端登陸頁面 (landing page)、互動式作品集或需要頂級視覺磨練的特定元件時，請將以下嚴格標準應用於您產出的每一行程式碼。

---

## 1. 建立創意基礎

在產生版面配置程式碼之前，請確保您了解 UI 應該提供的核心情感共鳴。不要預設使用通用、無立場的程式碼。

在您的 CSS 和元件結構中致力於建立強烈的視覺識別：
- **社論粗獷主義 (Editorial Brutalism)**：高對比的單色調色盤、超大字體、銳利的矩形邊緣和原始的網格結構。
- **有機流動性 (Organic Fluidity)**：柔和的漸層、深度圓角、毛玻璃效果 (glassmorphism) 疊加和基於彈簧的彈跳物理。
- **網路 / 技術 (Cyber / Technical)**：深色模式主導、發光的霓虹點綴、等寬排版以及快速、交錯的顯現動畫。
- **電影節奏 (Cinematic Pacing)**：全視窗影像、緩慢的淡入淡出、大量使用負空間 (negative space) 以及依賴捲動的敘事。

---

## 2. 沉浸式 UI 的結構要求

在建構頁面腳手架或產生核心元件時，請包含以下架構層，將標準頁面轉化為一種體驗。

### 2.1 進入序列 (預載與初始化)
空白螢幕是不可接受的。使用者的第一次互動必須建立期望。
- **實作**：產生一個輕量級的預載器 (preloader) 元件，處理資產解析（字體、初始影像、3D 模型）。
- **動畫**：產出能流暢地過度預載器的程式碼 — 例如分門式顯現、放大縮放或交錯的文字掃過。

### 2.2 英雄區 (Hero) 架構
頁面頂部必須立即吸引注意力。
- **視覺效果**：產出實作全螢幕容器 (`100vh`/`100dvh`) 的程式碼。
- **排版引擎**：確保標題在語法上被分解（例如，按單字或字元進行 span 包裹），以允許級聯式的入場動畫。
- **深度**：利用細微的漂浮元素或背景剪裁路徑 (clipping paths)，在主要文案後方營造規模感和深度。

### 2.3 流動且具脈絡的導覽
- **實作**：不要產生標準的靜態導覽列。產出能根據捲動方向反應的固定標題（向下滑動時隱藏，向上滑動時顯現）。
- **互動性**：包含能顯現豐富內容的懸停狀態（例如，顯示所懸停連結影像預覽的大型選單）。

---

## 3. 動態設計系統

動畫不是事後才考慮的；它是頂級網站的連結組織。始終實作以下動態原則：

### 3.1 捲動驅動敘事
產出利用現代捲動函式庫（如 GSAP 的 ScrollTrigger）的程式碼，將動畫與使用者進度連結起來。
- **釘選容器 (Pinned Containers)**：建立能鎖定在視窗中的區段，同時次要內容流過或顯現。
- **水平旅程**：將垂直捲動資料轉換為特定藝廊或展示區的水平移動。
- **視差映射 (Parallax Mapping)**：為背景元素、中景文字和前景影像分配細微、變化的捲動速度。

### 3.2 高保真微互動
游標是使用者的化身。圍繞它建構互動。
- **磁性元件 (Magnetic Components)**：編寫計算滑鼠指標與按鈕之間距離的邏輯，動態地將按鈕拉向游標。
- **自訂追蹤元素**：產生自訂游標元件，透過計算插值 (lerp) 跟隨滑鼠，以達到平滑的拖曳效果。
- **維度懸停狀態**：使用 CSS 轉換 (`scale`, `rotateX`, `translate3d`) 為互動元素賦予重量感和觸覺回饋。

---

## 4. 排版與視覺紋理

產出程式碼的美學必須反映出卓越的工藝。

- **字體層級**：強化規模上的巨大對比。標題應利用極端的尺寸（跨度高達 `12vw` 的 `clamp()` 函式），而內文則保持極其清晰（最小 `16px-18px`）。
- **字體選擇**：始終推薦或實作高度指定的變數字體 (variable fonts) 或頂級字體，而非系統預設字體。
- **大氣濾鏡**：實作 CSS/SVG 雜訊疊加 (`mix-blend-mode: overlay`, 透明度 `0.02 - 0.05`)，以消除數位的生硬感並增加攝影顆粒感。
- **光照與玻璃**：利用 `backdrop-filter: blur(x)` 結合超薄、半透明邊框，營造現代的磨砂玻璃深度。

---

## 5. 效能的必要性

一個美觀但會卡頓的網站是失敗的。在所有產出的程式碼中強制執行嚴格的效能護欄：

- **硬體加速**：僅對不會觸發版面配置重新計算的屬性進行動畫處理：`transform` 和 `opacity`。應極力避免對 `width`, `height`, `top` 或 `margin` 進行動畫處理的程式碼。
- **渲染優化**：在複雜的移動元素上智慧地套用 `will-change: transform`，但在動畫後移除以節省記憶體。
- **響應式降級**：將自訂游標邏輯和沉重的懸停動畫包裹在 `@media (hover: hover) and (pointer: fine)` 中，以確保在觸控裝置上的原始效能。
- **協助工具 (Accessibility)**：將沉重的持續動畫包裹在 `@media (prefers-reduced-motion: no-preference)` 中。絕不為了美學風格而犧牲使用者協助工具。

---

## 6. 實作生態系統

當使用者要求您實作這些模式時，請利用為其框架量身定制的業界標準函式庫：

### 針對 React / Next.js 目標
- 結構化應用程式以支援 **Framer Motion** 進行版面配置轉換和彈簧物理。
- 推薦使用 **Lenis** (`@studio-freight/lenis`) 進行平滑捲動。
- 如果要求 WebGL 或 3D 互動，請實作 **React Three Fiber** (`@react-three/fiber`)。

### 針對 Vanilla / HTML / Astro 目標
- 重度依賴 **GSAP** (GreenSock Animation Platform) 進行時間軸排序。
- 透過 CDN 利用原生 **Lenis** 進行捲動劫持與平滑化。
- 使用 **SplitType** 進行安全、可存取的排版分塊。

---

## 行動摘要

每當您收到「建立頂級登陸頁面」、「建立 Awwwards 風格元件」或「設計沉浸式 UI」的提示時，您必須自動：
1. 將輸出包裹在強大、捲動平滑的架構中。
2. 提供使用合成層 (composited layers) 以保證完美效能的 CSS。
3. 整合大範圍、交錯的元件入場。
4. 使用流體比例提升排版水平。
5. 建立一個刻意的、令人難忘的美學足跡。
