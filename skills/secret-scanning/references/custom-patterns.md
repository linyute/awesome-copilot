# 自訂模式參考

在儲存庫、組織和企業層級使用正規表示式定義自訂祕密掃描模式的詳細參考。

## 概觀

自訂模式擴展了祕密掃描，以偵測預設模式未涵蓋的組織特定祕密。它們被定義為正規表示式，且可以選擇性地強制執行推送保護。

## 模式定義

### 必要欄位

| 欄位 | 說明 |
|---|---|
| **模式名稱** | 模式的易讀名稱 |
| **祕密格式** | 比對祕密的正規表示式 |

### 選填欄位 (經由「更多選項」)

| 欄位 | 說明 |
|---|---|
| **祕密之前** | 必須出現在祕密之前的內容的正規表示式 |
| **祕密之後** | 必須出現在祕密之後的內容的正規表示式 |
| **額外比對需求** | 比對的額外限制 |
| **範例測試字串** | 用於驗證正規表示式的範例字串 |

### 正規表示式語法

自訂模式使用標準正規表示式。常見模式：

```
# API key with prefix
MYAPP_[A-Za-z0-9]{32}

# Connection string
Server=[\w.]+;Database=\w+;User Id=\w+;Password=[^;]+

# Internal token format
myorg-token-[a-f0-9]{64}

# JWT-like pattern
eyJ[A-Za-z0-9_-]+\.eyJ[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+
```

在之前/之後欄位中使用類似於 GitHub Actions 工作流語法的篩選模式進行 glob 風格的比對。

## 依範圍定義模式

### 儲存庫層級

1. 儲存庫設定 → 進階安全性
2. 在「祕密保護」下 → 自訂模式 → **建立新模式**
3. 輸入模式名稱、正規表示式和選填欄位
4. **儲存並測試執行 (dry run)** 以進行測試
5. 審閱結果 (最多 1,000 個比對結果)
6. 滿意時 **發佈模式**
7. 選擇性地啟用推送保護

**先決條件：** 儲存庫必須啟用祕密保護。

### 組織層級

1. 組織設定 → 進階安全性 → 全域設定
2. 在「自訂模式」下 → **建立新模式**
3. 輸入模式詳細資訊
4. **儲存並測試執行** — 選擇要測試的儲存庫：
   - 組織中的所有儲存庫，或
   - 最多 10 個選定的儲存庫
5. 滿意時 **發佈模式**
6. 選擇性地啟用推送保護

**附註：**
- 組織層級自訂模式的推送保護僅適用於已啟用推送保護的儲存庫
- 組織擁有者和儲存庫管理員會收到警示

### 企業層級

1. 企業設定 → 政策 → 進階安全性 → 安全性功能
2. 在「祕密掃描自訂模式」下 → **建立新模式**
3. 輸入模式詳細資訊
4. **儲存並測試執行** — 選擇最多 10 個儲存庫
5. 滿意時 **發佈模式**
6. 選擇性地啟用推送保護

**附註：**
- 只有模式建立者可以編輯或測試執行企業層級的模式
- 測試執行需要對所選儲存庫的管理存取權限
- 推送保護需要啟用企業層級的祕密掃描推送保護

## 測試執行流程

測試執行會針對儲存庫內容測試模式，而不會建立警示。

1. 定義模式後點擊 **儲存並測試執行**
2. 選擇目標儲存庫 (組織/企業層級)
3. 點擊 **執行**
4. 審閱最多 1,000 個範例結果
5. 識別誤判 (false positives)
6. 視需要編輯模式並重新執行
7. 僅在誤判率可接受時 **發佈模式**

> 測試執行至關重要 — 在發佈前務必進行測試，以避免警示雜訊。

## 管理已發佈的模式

### 編輯模式

發佈後，可以編輯模式：
1. 導覽至自訂模式
2. 修改正規表示式或選填欄位
3. 儲存並測試執行以驗證變更
4. 發佈更新後的模式

### 啟用推送保護

推送保護只能在模式發佈後啟用：
1. 導覽至已發佈的模式
2. 點擊推送保護旁邊的 **啟用**

**注意：** 為常見模式啟用推送保護可能會中斷貢獻者的工作流程。

### 停用或刪除模式

- 停用：停止產生新警示，但保留現有警示
- 刪除：移除模式並停止對其進行的所有掃描

## Copilot 輔助模式產生

使用 Copilot 祕密掃描自動產生正規表示式：

1. 導覽至自訂模式建立頁面
2. 選擇「使用 Copilot 產生」(如果可用)
3. 提供祕密類型的文字說明 (例如：「以 MYORG_ 開頭，後接 40 個十六進位字元的內部 API 金鑰」)
4. 選擇性地提供應比對的範例字串
5. Copilot 產生一個正規表示式模式
6. 審閱並精煉產生的正規表示式
7. 發佈前使用測試執行進行測試

## 模式繼承

| 範圍 | 套用於 |
|---|---|
| 儲存庫 | 僅該儲存庫 |
| 組織 | 組織中所有已啟用祕密掃描的儲存庫 |
| 企業 | 所有組織中所有已啟用祕密掃描的儲存庫 |

當啟用祕密掃描時，組織和企業模式會自動套用於新儲存庫。

## 最佳實踐

1. 發佈前 **務必進行測試執行** — 審閱誤判情況
2. **從具體開始** — 縮小正規表示式範圍可減少誤判
3. **使用之前/之後上下文** — 在不使用過於複雜的正規表示式的情況發增加精確度
4. **使用真實範例進行測試** — 包含應該和不應該比對的範例字串
5. **記錄模式** — 清楚命名模式，以便團隊瞭解它們偵測到的內容
6. **定期審閱** — 移除或更新不再適用的模式
7. **謹慎使用推送保護** — 僅對誤判率低的模式啟用
8. **考慮使用 Copilot** — 讓 AI 產生初始正規表示式，然後手動精煉
