# 議題範本 (Issue Templates)

複製並自訂這些範本以作為議題內容。

## 錯誤回報範本 (Bug Report Template)

```markdown
## 描述 (Description)
[請清楚描述發生的錯誤]

## 重現步驟 (Steps to Reproduce)
1. [第一個步驟]
2. [第二個步驟]
3. [依此類推...]

## 預期行為 (Expected Behavior)
[預期應發生的結果]

## 實際行為 (Actual Behavior)
[實際發生的情況]

## 環境 (Environment)
- 瀏覽器：[例如：Chrome 120]
- 作業系統：[例如：macOS 14.0]
- 版本：[例如：v1.2.3]

## 螢幕截圖/日誌 (Screenshots/Logs)
[若適用請提供]

## 額外上下文 (Additional Context)
[任何其他相關資訊]
```

## 功能請求範本 (Feature Request Template)

```markdown
## 摘要 (Summary)
[對功能的單行描述]

## 動機 (Motivation)
[為什麼需要這個功能？它能解決什麼問題？]

## 建議的解決方案 (Proposed Solution)
[此功能應如何運作？]

## 驗收準則 (Acceptance Criteria)
- [ ] [準則 1]
- [ ] [準則 2]
- [ ] [準則 3]

## 考慮過的替代方案 (Alternatives Considered)
[考慮過的其他方法以及未被採用的原因]

## 額外上下文 (Additional Context)
[模型圖、範例或相關議題]
```

## 任務範本 (Task Template)

```markdown
## 目標 (Objective)
[需要完成的工作內容]

## 詳情 (Details)
[工作的詳細描述]

## 檢查表 (Checklist)
- [ ] [子任務 1]
- [ ] [子任務 2]
- [ ] [子任務 3]

## 相依性 (Dependencies)
[任何阻礙因素或相關工作]

## 備註 (Notes)
[額外的上下文或考量因素]
```

## 極簡範本 (Minimal Template)

適用於簡單的議題：

```markdown
## 描述 (Description)
[內容與原因]

## 任務 (Tasks)
- [ ] [任務 1]
- [ ] [任務 2]
```
