# 自我學習系統

將專案學習產物維護在指定的教訓/記憶目錄下（例如：`.github/SecurityLessons` 和 `.github/SecurityMemories`）。

## 何時建立

### 教訓 (Lesson)

在以下情況建立教訓：

- 掃描產生了需要手動修正的誤報 (false positive)
- 發現類別、STRIDE 類別或缺陷類型在初次分析時遺漏，但在自我評論迴圈中被發現
- 發現了工具或方法論的限制
- 特定語言的規則發生誤判
- 無法解析 SCA 相依性

### 記憶 (Memory)

在以下情況建立記憶：

- 發現了架構決策、安全性慣例或技術堆疊細節
- 識別出相依性管理模式、特定領域的威脅模式或威脅執行者設定檔
- 發現了專案編碼慣例、框架慣用型法或已知的誤報模式
- 任何對未來掃描同一程式碼庫有用的程式碼庫專屬知識

## 治理規則 (Governance Rules)

1. **去重檢查**：在建立新的教訓或記憶前，搜尋現有檔案是否有類似內容。更新現有記錄而非建立重複項。
2. **衝突解決**：若新證據與現有的作用中教訓/記憶衝突，將舊的標記為 `已棄用 (deprecated)`，並建立更新版本且附帶 `取代 (Supersedes)` 參考。
3. **掃描啟動時重複使用**：在每次分析開始時，檢查教訓/記憶目錄中的適用背景。在開始分析前套用相關指引。
