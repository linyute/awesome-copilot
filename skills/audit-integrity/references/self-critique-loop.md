# 自我評論迴圈

在完成初始分析後，在交付輸出前執行**強制性的第二次檢視**。

## 通用檢查 (所有代理)

1. **證據檢查**：每項發現都必須引用具體的參考（檔案:行號、元件、架構元素、CVE ID、規則鍵值）。刪除任何沒有佐證證據的發現。
2. **覆蓋率檢查**：驗證與代理方法論相關的所有類別、階段或掃描類型都已明確評估。對於每個無問題的類別，請註明「未偵測到」，而非默默省略。
3. **緩解/修復檢查**：每個「關鍵」和「高」風險的發現都必須有具體、可實施的修復方案 — 而非通用的建議。

## 領域特定擴充

每個代理都會在上述通用清單中增加領域檢查：

### STRIDE 威脅建模

4. **STRIDE 完整性**：你是否為每個信任邊界和資料流評估了所有六個 STRIDE 類別 (S/T/R/I/D/E)？
5. **信任邊界稽核**：重新驗證每個識別出的信任邊界是否至少有一個經過評估的資料流穿過。

### STRIDE-LM (橫向移動)

4. **STRIDE-LM 完整性**：你是否為每個資產和信任邊界評估了所有七個類別 (S/T/R/I/D/E/LM)？
5. **控制措施覆蓋率**：每個「關鍵/高」風險威脅是否都對應到一個控制功能（資產清單/收集/偵測/保護/管理/回應）？
6. **橫向移動稽核**：重新追蹤所有識別出的跳板路徑。驗證從受損入口點到高價值資產之間不存在不受控的路徑。

### 程式碼審查威脅建模

4. **STRIDE 完整性**：為每個信任邊界和資料流評估所有六個 STRIDE 類別。
5. **信任邊界稽核**：每個信任邊界都有經過評估的資料流穿過。

### 程式碼品質 (SonarQube 風格)

4. **問題類型覆蓋率**：明確評估了所有五種問題類型（錯誤、漏洞、熱點、異味、重複）。
5. **評級合理性檢查**：在最終確定品質閘門 (Quality Gate) 判定前，確認 A–E 評級與發現數量一致。

### SAST/SCA

4. **污點追蹤 (Taint trace) 完整性**：在探索階段識別出的每個入口點是否都已追蹤至終點 (Sinks)？
5. **相依性清單 (Manifest) 覆蓋率**：在探索階段識別出的所有相依性清單是否都已進行審核？

### 多工具管線

4. **階段覆蓋率**：所有可交付的檔案是否已產生並儲存？
5. **交叉關聯**：SAST 的發現是否得到 SCA 發現的佐證 → 提升獲得佐證項目的優先順序。
6. **去重**：同一項發現不會出現在多個工具的輸出中。
7. **藍圖完整性**：每個「關鍵/高」風險發現都出現在立即修復層級中。
