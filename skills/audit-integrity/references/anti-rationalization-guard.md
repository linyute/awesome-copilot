# 反合理化守衛

以下合理化藉口**絕不**是跳過、省略或調降發現等級的有效理由：

## 通用合理化 (所有代理)

| 如果你認為... | 強制性回應 |
| ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| 「初次分析未發現問題/威脅」 | 在得出「乾淨」的結論前，必須對所有類別進行系統化評估。擴大範圍並完成完整的矩陣。 |
| 「這看起來沒問題，跳過深度分析」 | 「看起來沒問題」不是證據。證據 = 程式碼追蹤、架構參考或規則匹配。執行檢查。 |
| 「在實務中風險可能較低」 | 風險等級是基於 影響 × 可能性 (CVSS/可利用性)。任何調降等級都必須有明確證據支撐。 |
| 「這是一個誤報」 | 將其標記為潛在誤報但仍須包含在內 — 不要默默隱瞞。記錄理由以供人工審查。 |
| 「這不在範圍內」 | 明確說明原因，並引用聲明的範圍或評估邊界。 |
| 「這裡不需要控制/緩解措施」 | 明確說明「未識別出缺口 — 理由：[X]」。沉默不代表保證。 |

## SAST/SCA 特定

| 如果你認為... | 強制性回應 |
| ---------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| 「SCA CVE 在此處不可利用」 | 包含該 CVE 並附帶記錄的背景註釋 — 不要默默隱瞞。 |
| 「此階段可以跳過」 | 所有階段皆為強制性。記錄因缺少輸入而確實無法完成的任何階段。 |
| 「考慮到背景，嚴重性應該較低」 | 嚴重性是基於 CVSS/可利用性。任何調降等級都必須有明確證據支撐。記錄理由，不要隱瞞。 |

## 程式碼品質特定

| 如果你認為... | 強制性回應 |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| 「團隊稍後會重構此部分」 | 技術債務在今天仍計入債務比率。請準確記錄。 |
| 「品質閘門 (Quality Gate) 失敗是誤報」 | 將其作為一項發現包含在內，記錄疑似誤報的理由，並標記以供人工審查。 |

## 威脅建模特定

| 如果你認為... | 強制性回應 |
| ---------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| 「此威脅已由架構緩解」 | 記錄具體的補償性控制措施，並驗證其是否確實實施 — 不要僅憑假設。 |
| 「此類別在此處沒有適用的威脅」 | 明確說明「未識別出適用的威脅 — 理由：[X]」。不要默默省略。 |
| 「此處不太可能發生橫向移動」 | 記錄防止滲透轉移的具體架構控制措施，並驗證其是否實施 — 不要僅憑假設。 |
| 「此威脅執行者不會以此為目標」 | 記錄該排除項目的依據。內部威脅和供應鏈執行者必須始終納入考慮。 |
