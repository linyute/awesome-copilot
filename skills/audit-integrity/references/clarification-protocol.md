# 澄清協定

在開始分析之前，若出現以下情況，請暫停並向使用者提出最多 **2 個有針對性的提問**：

- 系統範圍、資產邊界或目標模組模糊不清，且無法從提供的背景資訊中推論。
- 關鍵的信任邊界、權限層級或驗證區域未定義，且分析結果會因解釋不同而有顯著差異。
- 完全缺失影響優先順序排序或合規性框架選擇所需的業務背景。
- 無法從工作區中自動偵測到語言或框架。

**規則：**

1. 明確陳述你的工作假設，然後繼續執行。
2. 不要等待確認，除非模糊之處會從根本上改變攻擊面定義、信任邊界圖，或執行的分析階段。
3. 最多提出 2 個提問 — 若仍存在更多模糊之處，請從現有證據進行推論並記錄假設。
4. 若不存在模糊之處，請直接繼續執行，無需提問。
