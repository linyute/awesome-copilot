# 模組 2：鍵盤快捷鍵 (Module 2: Keyboard Shortcuts)

## 導覽與編輯 (Navigation & Editing)

| 快捷鍵 | 行動 |
|----------|--------|
| `@` | 提及檔案 — 將其內容包含為上下文 |
| `Ctrl+S` | 提交提示同時保留輸入文字 |
| `Shift+Tab` | 循環切換模式：互動 (Interactive) → 計畫 (Plan) |
| `Ctrl+T` | 切換模型推理顯示 |
| `Ctrl+O` | 展開最近的時序表 (無輸入時) |
| `Ctrl+E` | 展開所有時序表 (無輸入時) / 移動到行尾 (鍵入時) |
| `↑` `↓` | 導覽指令歷史記錄 |
| `!` | 直接執行 Shell 指令 (跳過 AI) |
| `Esc` | 取消目前操作 |
| `Ctrl+C` | 取消操作 / 清除輸入 / 結束 |
| `Ctrl+D` | 關閉工作階段 |
| `Ctrl+L` | 清除螢幕 |
| `Ctrl+G` | 在外部編輯器 ($EDITOR) 中編輯提示 |

## 行編輯 (Line Editing)

| 快捷鍵 | 行動 |
|----------|--------|
| `Ctrl+A` | 移動到行首 |
| `Ctrl+H` | 刪除前一個字元 |
| `Ctrl+W` | 刪除前一個單字 |
| `Ctrl+U` | 從游標刪除到行首 |
| `Ctrl+K` | 從游標刪除到行尾 |
| `Meta+←` `Meta+→` | 按單字移動游標 |

## 教學進階技巧 (Pro tips to teach)

- `@` 是**最重要的**快捷鍵 — 這是您提供精確上下文的方式
- `!git status` 會直接執行 Git，而不經過 AI 處理
- 在執行複雜任務**之前**，使用 `Shift+Tab` 進入計畫 (Plan) 模式
- 針對長提示，使用 `Ctrl+G` 開啟您的 $EDITOR — 這是個改變遊戲規則的技巧
- `Ctrl+S` 讓您可以反覆修改提示而無需重新鍵入
