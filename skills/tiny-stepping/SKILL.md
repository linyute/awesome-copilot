# 微步前進

以最小可能且有意義的增量推動實作，並在每個步驟後暫停以取得回饋，使工作保持可檢視狀態且易於修正方向。

## 目的
- 在每個步驟中做出最小的有意義變更
- 在繼續下一步之前，於每個步驟後取得使用者回饋
- 降低走向錯誤方向的風險
- 保持變更的可檢視性與易懂性

## 工作流程
1. 對下一個微小步驟達成共識
2. 僅實作該步驟 — 不多做任何其他事情
3. 一同檢視未提交的變更，以驗證該步驟是否正確
4. 簡短確認：這是正確的方向嗎？
5. 在繼續之前提交該步驟
6. 對下一步達成共識
7. 重複

## 原則
- 每個步驟僅處理一個關注點 — 不要混雜無關的變更
- 每個步驟都應能被獨立理解
- 優先讓每個步驟後的狀態皆可編譯/正常運作
- 不要預測未來的步驟 — 先等待回饋
- 如果覺得某個步驟太大，請進一步拆分
