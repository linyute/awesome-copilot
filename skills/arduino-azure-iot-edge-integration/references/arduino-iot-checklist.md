# Arduino Azure IoT 檢查清單

在敲定架構或實作指引之前，請使用此檢查清單。

## 0) 官方 Arduino 基準

- 已檢閱來自 <https://www.arduino.cc/en/Guide> 和 <https://docs.arduino.cc/> 的官方參考資料。
- 語言/API 呼叫已對照 <https://docs.arduino.cc/language-reference/> 進行驗證。
- 已檢閱來自 `references/arduino-official-best-practices.md` 的最佳實務。

## 1) 裝置設定檔

- 已記錄 MCU 模型和記憶體限制。
- 已定義感測器清單和取樣策略。
- 已記錄電力模型 (市電、電池、睡眠週期)。

## 2) 連線能力

- 已記錄選定的傳輸方式 (優先選擇 MQTT over TLS)。
- 已定義網路失敗行為。
- 若裝置缺乏實時時鐘 (RTC) 同步，已定義本機時間戳記策略。

## 3) 安全性

- 每個裝置唯一的識別身分。
- 原始檔控制中不得有秘密。
- 已記錄認證輪轉計劃。
- 已記錄韌體更新與復原計劃。

## 4) 邊緣與雲端流程

- 已記錄從邊緣到 IoT Hub 的路由。
- 已定義離線緩衝限制。
- 已記錄重複處理策略。
- 已定義警示閾值與目的地。

## 5) 驗證

- 連線穩定性測試場景。
- 封包遺失與重新連線測試。
- 指令授權測試。
- 韌體版本與健全狀況報告驗證。
