# 事件驅動架構模式

## 事件溯源模式 (Event Sourcing Pattern)

**問題**：需要應用程式狀態所有變更的完整稽核線索。

**解決方案**：使用僅限附加 (append-only) 的儲存來記錄一系列完整的事件，這些事件描述了對定義域中資料所採取的操作。

**使用時機**：
- 需要完整的稽核線索
- 實作時間查詢（特定時間點的狀態）
- 支援事件重放與偵錯
- 實作具有最終一致性的 CQRS

**實作考量**：
- 將事件儲存在僅限附加的記錄中
- 透過重放事件來重建目前狀態
- 實作事件版本控制策略
- 處理事件結構描述 (schema) 的演進
- 考量隨時間增加的儲存空間
- 為效能實作快照
