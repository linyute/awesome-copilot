# 安全性模式

## 同盟身分模式 (Federated Identity Pattern)

**問題**：應用程式必須管理使用者驗證與授權。

**解決方案**：將驗證委派給外部身分識別提供者。

**使用時機**：
- 實作單一登入 (SSO)
- 減少驗證複雜性
- 支援社交身分識別提供者

**實作考量**：
- 使用 Azure AD、Auth0 或其他身分識別提供者
- 實作 OAuth 2.0、OpenID Connect 或 SAML
- 在本機儲存最少的使用者資料
- 平穩地處理身分識別提供者的停機情況
- 實作適當的權杖驗證

## 隔離模式 (Quarantine Pattern)

**問題**：外部資產可能包含惡意內容或弱點。

**解決方案**：在工作負載耗用外部資產之前，確保其符合團隊同意的品質水準。

**使用時機**：
- 處理使用者上傳的檔案
- 耗用外部資料或套件
- 實作零信任架構

**實作考量**：
- 在使用前掃描所有外部內容 (惡意程式碼、弱點)
- 將隔離環境與實際執行環境隔離
- 為發佈定義明確的品質閘門
- 實作自動化掃描與驗證
- 記錄所有隔離活動以供稽核

## 代客製金鑰模式 (Valet Key Pattern)

**問題**：應用程式不應 Proxy 所有用戶端的資料存取。

**解決方案**：使用權杖或金鑰，為用戶端提供對特定資源或服務的受限直接存取。

**使用時機**：
- 在不進行 Proxy 處理的情況下提供對儲存體的直接存取
- 將通過應用程式層的資料傳輸降至最低
- 實作限時或受限的存取

**實作考量**：
- 產生 SAS 權杖或預先簽署的 URL
- 設定適當的到期時間
- 限制權限 (唯讀、唯寫、特定作業)
- 視需要實作權杖撤銷
- 監時代客製金鑰的使用情況
