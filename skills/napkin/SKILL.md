# Napkin — Copilot CLI 的視覺化白板 (Visual Whiteboard for Copilot CLI)

Napkin 為使用者提供一個基於瀏覽器的白板，他們可以在上面繪圖、素描並加入便籤，以視覺化的方式思考點子。Agent 會讀取白板內容（透過 PNG 快照和選填的 JSON 資料），並以分析、建議和後續步驟進行對話式回應。

目標受眾是律師、專案經理 (PM) 和業務利益相關者 — 而非軟體開發人員。保持一切內容易於親近且不含術語。

---

## 啟用 (Activation)

當使用者呼叫此技能時 — 例如說「let's napkin」、「開啟 napkin」、「開始白板」或使用斜線指令 — 請執行下列操作：

1. **複製隨附的 HTML 模板** 從技能資產到使用者的桌面。
   - 模板位於相對於此 `SKILL.md` 檔案的 `assets/napkin.html`。
   - 將其複製到 `~/Desktop/napkin.html`。
   - 如果 `~/Desktop/napkin.html` 已經存在，在覆寫前詢問使用者是要開啟現有的還是重新開始。

2. **在預設瀏覽器中開啟：**
   - macOS：`open ~/Desktop/napkin.html`
   - Linux：`xdg-open ~/Desktop/napkin.html`
   - Windows：`start ~/Desktop/napkin.html`

3. **告訴使用者接下來要做什麼。** 說一些親切且簡單的話：

   ```
   您的 Napkin 已在瀏覽器中開啟！

   繪圖、素描或加入便籤 — 任何能幫助您思考點子的方式都可以。

   當您準備好聽聽我的建議時，請點擊白板上的綠色「分享給 Copilot」按鈕，然後回到這裡說「檢查 Napkin」。
   ```

## 讀取 Napkin (Reading the Napkin)

當使用者說「檢查 Napkin」、「看一看 Napkin」、「你有什麼想法」、「讀取我的 Napkin」或任何類似的話時，請遵循下列步驟：

### 步驟 1 — 讀取 PNG 快照 (主要)

尋找名為 `napkin-snapshot.png` 的 PNG 檔案。依序檢查下列位置（瀏覽器會將其儲存到使用者的預設下載資料夾，該資料夾因人而異）：

1. `~/Downloads/napkin-snapshot.png`
2. `~/Desktop/napkin-snapshot.png`

使用 `view` 工具讀取 PNG。這會將影像作為 Base64 編碼資料傳送給模型，模型可以對其進行視覺解讀。PNG 是 Agent 理解使用者所畫內容的**主要**方式 — 它捕捉了手繪素描、箭頭、空間佈局、註釋、圈選或劃掉的項目，以及畫布上的任何其他內容。

如果在任何位置都找不到 PNG，請**不要**靜默跳過。相反地，告訴使用者：

```
我還沒看到您從 Napkin 傳來的快照。請按照下列步驟操作：

1. 前往瀏覽器中的白板
2. 點擊綠色的「分享給 Copilot」按鈕
3. 回到這裡再次說「檢查 Napkin」

該按鈕會儲存一張我可以查看的螢幕截圖。
```

### 步驟 2 — 讀取剪貼簿以獲取結構化 JSON (輔助)

同時嘗試從系統剪貼簿獲取結構化 JSON 資料。白板會自動隨 PNG 一併複製此資料。

- macOS：`pbpaste`
- Linux：`xclip -selection clipboard -o`
- Windows：`powershell -command "Get-Clipboard"`

JSON 包含便籤和文字標籤的確切文字內容、它們的位置以及顏色。這補充了 PNG 的不足，為您提供可能難以從螢幕截圖中讀取的精確文字。

如果剪貼簿不包含 JSON 資料也沒關係 — 僅憑 PNG 模型就有足夠的資訊可以處理。不要將遺失剪貼簿視為錯誤。

### 步驟 3 — 結合兩個來源進行解讀

將視覺快照和結構化文字結合成對使用者想法或計畫的連貫理解：

- **來自 PNG**：描述您看到的內容 — 素描、圖表、流程圖、分組、箭頭、空間佈局、註釋、圈選項目、劃掉項目、強調標記。
- **來自 JSON**：讀取便籤和標籤的確切文字內容，並注意它們的位置和顏色。
- **結合兩者** 成為單一且具對話性的解讀。

### 步驟 4 — 進行對話式回應

不要傾倒原始資料或技術總結。以看過某人白板草圖的協作者身份進行回應。範例：

- 「我可以看到您繪製了一個三個階段的程序 — 看起來您正在考慮將 [X] 串接到 [Y] 然後是 [Z]。角落的便籤寫著『[文字]』 — 這是您希望我處理的疑慮嗎？」
- 「看起來您將左側的這四個點子分組在一起，並與右側的兩個項目分開。您是否將它們視為兩個不同的類別？」
- 「我看到您畫了連接 [A] 到 [B] 到 [C] 的箭頭 — 這是您構想的工作流程嗎？」

### 步驟 5 — 詢問下一步

最後務必提供下一步建議：

- 「需要我在這基礎上進一步發展嗎？」
- 「我應該將此轉換為結構化文件嗎？」
- 「需要我在 Napkin 上加入我的建議嗎？」

## 在 Napkin 上回應 (Responding on the Napkin)

當使用者希望 Agent 將內容加回白板時：

- Agent **無法** 直接修改 HTML 檔案的畫布狀態 — 那是由瀏覽器中執行的 JavaScript 管理的。
- 相反地，請提供實用的替代方案：
  - 直接在 CLI 這裡提供回應，並建議使用者手動將其加入 Napkin。
  - 提議根據從 Napkin 解讀出的內容建立一份獨立文件（Markdown、備忘錄、檢核表等）。
  - 如果合理，請建立一個預先載入內容的 `napkin.html` 更新複本。

## 語氣與風格 (Tone and Style)

- 使用與新手模式 (Noob-mode) 技能相同且易於親近、非技術性的語氣。
- 絕不要使用開發人員術語而不以淺顯易懂的英文進行解釋。
- 將 Napkin 視為一個具創意、協作性的空間 — 而非正式的輸入機制。
- 無論藝術品質如何，都要鼓勵使用者的素描。
- 將回應框架化為「在您的思考基礎上建構」，而非「分析您的輸入」。

## 錯誤處理 (Error Handling)

**找不到 PNG 快照：**

```
我還沒看到您從 Napkin 傳來的快照。請按照下列步驟操作：

1. 前往瀏覽器中的白板
2. 點擊綠色的「分享給 Copilot」按鈕
3. 回到這裡再次說「檢查 Napkin」

該按鈕會儲存一張我可以查看的螢幕截圖。
```

**桌面不存在白板檔案：**

```
看起來我們還沒開始使用 Napkin。需要我為您開啟一個嗎？
```

## 重要注意事項

- PNG 解讀是**主要**頻道。多模態模型可以讀取並解讀由 `view` 工具傳回的 Base64 影像資料。
- JSON 剪貼簿資料是**輔助**的 — 它提供精確的文字，但無法捕捉手繪圖案。
- 務必先檢查 PNG。如果找不到，提示使用者點擊「分享給 Copilot」。
- 如果剪貼簿沒有 JSON 資料，請僅使用 PNG 繼續進行。
- HTML 模板位於相對於此 `SKILL.md` 檔案的 `assets/napkin.html`。
- 如果新手模式技能也處於作用中，在要求檔案或 Bash 權限時，請使用其風險指標格式（綠色/黃色/紅色）。
