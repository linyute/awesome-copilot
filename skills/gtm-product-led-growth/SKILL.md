# 產品導向成長 (Product-Led Growth, PLG)

建立自助式的獲客與擴展動態。但首先，請弄清楚 PLG 是否真的適合你的產品。

## 何時使用

**觸發情境：**
- 「我們應該建立 PLG 還是銷售導向 (Sales-led) 模型？」
- 「我們該如何推動自助式採用？」
- 「從免費版到付費版的轉換效果不彰」
- 「開發者導向的採用策略」
- 「我們應該投資哪些成長管道？」
- 「我該如何判斷 PLG 是否奏效？」

**背景：**
- 開發者工具與平台
- 具備自助服務潛力的 B2B SaaS
- 價值無需演示 (Demo) 即可一目了然的產品
- 由下而上 (Bottom-up) 的採用動態
- 成長管道優先順序評估

---

## 核心框架

### 1. PLG 的現實檢驗（在承諾前先測試）

**我在同時運行兩種動態中學到的教訓：**

經典的創業辯論。PLG 派：「開發者想要自助服務。」銷售派：「企業需要專人引導。」我們沒有爭論，而是直接測試了 6 個月。同樣的產品，兩套進入市場 (GTM) 動態，並追蹤所有數據。

**結果：**

PLG：高量、低客單價 (ACV 約 $5K)、快速變現、高流失率。銷售導向：低量、高客單價 (ACV 約 $50K)、變現較慢、低流失率。**銷售導向最終帶來的營收是 PLG 的 10 倍，儘管量少了 10 倍。**

**原因：** 產品複雜度 + 買家資歷 = 銷售導向獲勝。該產品需要與現有基礎架構整合、跨團隊的變革管理以及多方關係人的對齊。開發者喜歡自助服務，但他們並非財務買家。

**PLG 奏效的條件：**
- 價值在最初 5 分鐘內即清晰可見
- 實作過程非常簡單
- 個人使用者無需團隊背書即可獲得價值
- 沒有採購/法律障礙
- 買家即使用者

**銷售導向奏效的條件：**
- 產品需要整合/設定
- 多方關係人需要對齊
- 買家不等於使用者
- 交易規模足以支撐人工服務成本
- 客戶需要教育才能看到價值

**在建立 PLG 之前，請先測試你的動態。不要因為 PLG 流行就假設它更好。** PLG 在高量下效率很高，但當產品具備複雜度時，銷售導向可能更獲利。

### 2. 成長等式（將輸入對應到產出）

**模式：**

當你系統化活動與使用者獲取之間的關係時，成長就會產生加乘效應。不是「做更多行銷」— 而是將具體的輸入對應到可衡量的產出。

**如何建立你的成長等式：**

針對每個管道，定義：活動（輸入）→ 流量（產出）→ 轉換。

- **有機搜尋**：1 篇高質量的部落格文章 → 每月 400 名使用者 → 5% 轉換率 = 20 名新使用者
- **付費廣告**：$1K 支出，10 萬次曝光下有 8% 點擊率 = 8K 次點擊 → 以 X% 轉換
- **社群活動**：1 場活動 → 60 名參與者 → 35% 轉換率 = 21 名使用者
- **推薦**：1 個整合夥伴 → N 名被推薦使用者 → 以 Y% 轉換

**為何這很重要：**

一旦驗證了等式，規模化就變成了數學問題。「下個月我需要增加 200 名使用者」→「我需要多寫 10 篇部落格文章」或「我需要多投入 $5K 廣告費」。沒有等式，你就是在瞎猜。

**測試等式：**

1. 從假設開始：「如果我建立 X，它將帶動 Y 轉換」
2. 使用小樣本測試：寫 1 篇部落格文章，衡量實際轉換率
3. 驗證：現實是否符合假設？
4. 帶信心規模化：若符合，則增加輸入
5. 不符合則終止：4 週的數據足以做出決策

**常見錯誤：**

在未經測試的情況下臆測轉換率。假設來自同一管道的所有使用者品質都一樣。在驗證等式之前就進行規模化。

### 3. 管道經濟學（汰弱留強）

**模式：**

每個管道都有其經濟效益。如果不追蹤，你就會過度投資在失敗管道，而對成功管道投資不足。

**針對每個管道追蹤：**
1. **CAC（獲客成本）**：總支出 / 新使用者人數
2. **轉換率**：註冊人數 → 付費人數
3. **留存率**：依來源劃分的 30 天、90 天留存
4. **LTV（生命週期價值）**：依管道劃分的客戶生命週期營收
5. **回本週期**：收回 CAC 需要多長時間

**決策框架：**

- CAC < (LTV × 毛利) → 積極規模化
- CAC ≈ (LTV × 毛利) → 優化，不要規模化
- CAC > (LTV × 毛利) → 4 週內終止

**每月管道檢核**：哪些管道是獲利的？哪些是錢坑？每季重新分配：將 3 倍預算投入勝出管道，終止失敗管道。

**關鍵洞察：管道品質各異**

便宜的 CAC 不代表好的 CAC。有機搜尋可能帶來 $0 CAC 但具備 85% 30 天留存的使用者。付費搜尋可能帶來 $12 CAC 但僅有 45% 30 天留存的使用者。考慮到留存與 LTV，「免費」管道的價值其實高出 10 倍。

**系統化測試：**

每月測試 2 個新管道。每個給予 4 週的數據。如果經濟效益不佳，果斷終止。無論結果如何，都要記錄所學 — 失敗的經驗與成功的經驗同樣有價值。

**常見錯誤：**

追蹤 CAC 時忽略了留存率。一個會導致使用者流失的廉價管道，其成本比能留住使用者的高價管道更昂貴。

### 4. 首次價值實現時間 (Time to First Value, TTFV)（唯一的啟動指標）

**模式：**

使用者在最初的 5-10 分鐘內就會判定產品價值。如果他們不能快速達到「啊哈時刻 (aha moment)」，他們就會放棄。

**啟動審計：**

1. 以新使用者身份註冊你自己的產品
2. 計時達成首次價值所需的時間
3. 計算達到啊哈時刻所需的步驟
4. 你在哪裡卡住了？

**如果 TTFV > 10 分鐘，你就有啟動問題。**

**調整前**：註冊 → 確認電郵 → 填寫個人資料 → 設定配置 → 閱讀文件 → 第一個動作

**調整後**：註冊 → 預載範例資料 → 第一個動作（立即達成啊哈時刻）

**具體的修復方法：**

1. **預載範例資料**：使用者想看到價值，而不是想做設定。立即給他們一個可運作的範例。
2. **跳過非必要設定**：確認電郵、個人資料、配置設定 — 這些都可以等到啊哈時刻之後再做。
3. **漸進式揭露 (Progressive Disclosure)**：不要預先展示所有功能。從一個核心工作流程開始。隨著程度提升逐步揭露複雜性。
4. **演示優於陳述**：互動式教學 > 影片 > 文字文件。讓他們點擊完成一個工作流程。

**常見錯誤：**

假設使用者會閱讀文件。他們不會。他們會點點看 5 分鐘，如果沒用，他們就會離開。

### 5. $5K → $50K 的拐點（PLG 失效的時刻）

**模式：**

PLG 適用於 $1K-$10K 的年營收。在 $20K-$50K 之間，動態會崩潰，因為組織摩擦力開始介入：採購、法律、安全、多方關係人認同。

**混合模式：**

**PLG ($0-$10K)**：自助註冊 → 免費層級 → 付費層級 → 信用卡結帳 → 自動化引導

**銷售輔助 ($10K-$50K)**：自助發現 → 業務根據使用訊號介入 → 人工協商合約 → 專人負責引導

**企業級 ($50K+)**：主動或被動線索 → 演示 → POC → 提案 → 法律/安全審核 → 高管贊助

**PQL（產品合格線索）訊號（何時啟發銷售介入）：**

- **使用深度**：每日活躍、已使用核心功能、接近限制額度
- **擴展訊號**：同一公司有多名使用者、使用團隊功能、整合功能
- **採購訊號**：請求 SSO/合規/SLA、詢問團隊定價

**接洽方式：**

糟的：「嘿，我看到你註冊了。」（冷淡、大眾化、破壞信任）
好的：「你們團隊在 12 個儲存庫中使用了 [特定功能]。我們可以幫你們達成 [特定價值]。有空聊 15 分鐘嗎？」（親切、具體、提供價值）

**常見錯誤：**

業務過早介入小於 $5K 的交易。這會破壞 PLG 動態，嚇跑使用者。讓他們自助服務直到他們需要幫助。

### 6. 成長預測（為不確定性做計畫）

**模式：**

預測永遠是錯的。但計畫依然有價值，因為它強迫思考並建立責任制。

**建立三種情境模型：**

**基準情境 (Baseline)（維持目前軌跡）：**
- 有機搜尋：35% 成長 → 40K 新使用者
- 付費管道：持平 → 2K 新使用者
- 社群：10% 成長 → 400 名新使用者
- 總計：42.4K

**樂觀情境 (Upside)（所有成長倡議皆執行）：**
- 有機：50% 成長（3 倍內容產出）→ 48K
- 付費：2 倍支出，同樣效率 → 4K
- 新倡議（夥伴關係）：起步 → 3K
- 總計：55K

**悲觀情境 (Downside)（關鍵管道失效）：**
- 有機：0% 成長 → 26K
- 付費：單次獲客成本 (CPA) 翻倍 → 1K
- 總計：27K

**將此模型用於：**
- 設定基準目標（基準情境）
- 挑戰目標（樂觀情境）
- 呈報觸發點（如果落入悲觀情境，必須做出改變）
- 資源分配（為了達成樂觀情境，需要改變哪些輸入？）

**每月更新**：將預測與實際情況比較。調整模型。不要預測完就丟在一旁。

**常見錯誤：**

過於樂觀的預測，假設每件事都會奏效。未按月更新。將預測視為目標（它應該是一個範圍，而非一個數字）。

---

### 7. 撰寫教戰手冊 (Playbook) 的習慣

**模式：**

知識會隨著人員離職而流失。目標不是零星的勝利 — 而是將有效的方法系統化。

**在每次成功的活動或實驗後，撰寫 1 頁的教戰手冊：**

```
教戰手冊：[管道/戰術名稱]

目標：[達成什麼成果]
步驟：[編號，且詳細到足以讓不熟悉的人執行]
預期產出：[具體指標]
追蹤指標：[如何衡量]
風險與緩解：[可能出什麼錯]
負責人：[姓名]
最後更新：[日期]
```

**測試方法**：一個未參與其中的人能執行這份手冊嗎？如果不行，就太模糊了。

**每季檢視**。移除不再有效的手冊。更新那些已經演進的手冊。這將成為你的成長作業系統。

**常見錯誤：**

運行實驗卻未記錄教戰。在了解機制前就規模化。成長知識只存在於個別員工的腦袋裡。

## 決策樹

### 我們應該建立 PLG 還是銷售導向模型？

```
使用者能在 10 分鐘內不看文件就獲得價值嗎？
├─ 否 → 需要銷售導向模型
└─ 是 → 他們能自助完成實作嗎？
    ├─ 否 → 需要銷售導向模型
    └─ 是 → 買家即使用者嗎？
        ├─ 否 → 混合模式 (PLG + 銷售輔助)
        └─ 是 → 純 PLG 模式可行
```

### 對於這個管道，該保留、規模化還是終止？

```
CAC < (LTV × 毛利) 嗎？
├─ 否 → 4 週內終止
└─ 是 → 90 天留存率 > 60% 嗎？
    ├─ 否 → 優化 (改善啟動/新手引導)
    └─ 是 → 積極規模化 (投入 3 倍預算)
```

## 常見錯誤

**1. 假設 PLG 永遠奏效**
產品複雜度 + 買家資歷 = 銷售導向獲勝。承諾投入前先測試。

**2. 沒有管道經濟學數據**
每個管道都有其 CAC、留存率和 LTV。追蹤它們，否則你就是在盲飛。

**3. 免費層級太過慷慨或太過受限**
太慷慨：沒有轉換。太受限：無法啟動。請給予使用者 10-20 個啊哈時刻。

**4. 沒有成長等式**
「做更多行銷」不是策略。請按管道對應輸入 → 產出 → 轉換。

**5. 在驗證前就規模化**
在規模化任何管道前，先取得 4 週的數據。如果經濟效益不佳，果斷終止。

**6. 成長知識只在一個人的腦子裡**
將每次成功的實驗記錄為教戰手冊。

## 快速參考

**PLG 就緒度**：10 分鐘內獲得價值 + 自助實作 + 買家即使用者

**成長等式**：活動（輸入）→ 流量（產出）→ 轉換，依管道細分

**管道經濟學**：CAC、轉換、30/90 天留存、LTV、回本週期 — 依管道細分，每月檢核

**終止標準**：CAC > (LTV × 毛利) → 4 週內未改善則終止

**PQL 訊號**：使用深度 + 擴展（多使用者） + 採購意圖（SSO/合規請求）

**銷售接力**：<$10K: PLG → $10K-$50K: 銷售輔助 → >$50K: 全面銷售

**預測**：基準情境 + 樂觀情境 + 悲觀情境，每月更新
