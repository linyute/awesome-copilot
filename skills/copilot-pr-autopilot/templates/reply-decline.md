# 回覆：拒絕並提供理由

當 triage 決定 `decline` 時使用。回覆必須解釋為什麼
拒絕是正確的決定——而不僅僅是您考慮過它。回覆後，務必
解決討論串；未回覆的開放討論串
表示逃避。

```
Considered this, but declining: <源自程式碼或
設計的具體原因>。<選填：您排除的交錯/情境，或
替代成本>。Happy to revisit if <特定觸發條件>.
```

範例（領域中立）：

> 有考慮過將鎖延伸到初始化路徑中，但
> 予以拒絕：在任何並行呼叫端能夠到達此程式碼之前，初始化就已執行完畢，
> 因此競爭視窗只有在 init 回呼傳回後才會開啟。跨模組共用鎖
> 在耦合度上付出的代價，超過了實際風險暴露的合理性。
> 如果遙測數據顯示出實際的交錯，
> 很樂意重新研議。
