# 步驟 1b：入口點與執行流程

識別應用程式如何啟動以及真實使用者如何叫用它。使用 `pixie_qa/00-project-analysis.md` 中的**能力清單**來排列優先順序 — 專注於能測試最有價值且最常用能力的入口點，而不僅僅是你發現的第一個入口點。

---

## 調查內容

### 1. 軟體如何執行

入口點是什麼？如何啟動它？是 CLI、伺服器還是函式庫函式？有哪些必要的參數、設定檔或環境變數？

尋找：

- `if __name__ == "__main__"` 區塊
- 框架入口點 (FastAPI `app`、Flask `app`、Django `manage.py`)
- `pyproject.toml` 中的 CLI 入口點 (`[project.scripts]`)
- 揭示啟動命令的 Docker/compose 設定

### 2. 真實使用者入口點

真實使用者或客戶端如何叫用應用程式？這是評估必須測試的部分 — 而不是繞過請求管線的內部函式。

- **網頁伺服器**：哪些 HTTP 端點接受使用者輸入？使用什麼方法 (GET/POST)？請求本文的形狀為何？
- **CLI**：使用者提供哪些命令列參數？
- **函式庫/函式**：呼叫者匯入並呼叫哪個函式？參數為何？

### 3. 環境與設定

- 應用程式需要哪些環境變數？（服務端點、資料庫 URL、特性旗標）
- 它讀取哪些設定檔？
- 哪些具有合理的預設值，哪些必須明確設定？
