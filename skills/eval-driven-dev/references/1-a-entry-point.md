# 步驟 1a：進入點與執行流程

識別應用程式如何啟動以及真實使用者如何呼叫它。

---

## 調查內容

### 1. 軟體如何執行

進入點是什麼？您如何啟動它？它是 CLI、伺服器還是函式庫函式？有哪些必要的引數、設定檔或環境變數？

尋找：

- `if __name__ == "__main__"` 區塊
- 框架進入點 (FastAPI `app`, Flask `app`, Django `manage.py`)
- `pyproject.toml` 中的 CLI 進入點 (`[project.scripts]`)
- 顯示啟動指令的 Docker/compose 設定

### 2. 真實使用者進入點

真實使用者或用戶端如何呼叫應用程式？這是評估必須執行到的內容 — 而不是略過請求管線的內部函式。

- **網頁伺服器**：哪些 HTTP 端點接受使用者輸入？哪些方法 (GET/POST)？請求主體的形狀為何？
- **CLI**：使用者提供哪些命令列引數？
- **函式庫/函式**：呼叫者匯入並呼叫哪個函式？有哪些引數？

### 3. 環境與設定

- 應用程式需要哪些環境變數？(API 金鑰、資料庫 URL、特性切換 (feature flags))
- 它讀取哪些設定檔？
- 哪些具有合理的預設值，哪些必須明確設定？
