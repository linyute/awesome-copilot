# 錯誤分析：多輪對話 (Error Analysis: Multi-Turn Conversations)

偵錯複雜的多輪對話追蹤。

## 方法 (The Approach)

1. **優先進行端對端檢查** - 對話是否達成了目標？
2. **尋找第一個失敗點** - 向後追溯至根本原因。
3. **簡化** - 在進行多輪對話偵錯前，先嘗試單輪對話。
4. **N-1 測試** - 區分特定輪次的問題與能力問題。

## 尋找第一個上游失敗點 (Find First Upstream Failure)

```
第 1 輪：使用者詢問航班 ✓
第 2 輪：助理詢問日期 ✓
第 3 輪：使用者提供日期 ✓
第 4 輪：助理搜尋了錯誤的日期 ← 第一個失敗點 (FIRST FAILURE)
第 5 輪：顯示錯誤航班（後果）
第 6 輪：使用者感到沮喪（後果）
```

應專注於第 4 輪，而非第 6 輪。

## 先進行簡化 (Simplify First)

在偵錯多輪對話前，先測試單輪對話：

```python
# 如果單輪對話也失敗 → 問題在於檢索/知識
# 如果單輪對話通過 → 問題在於對話上下文
response = chat("電子產品的退貨政策是什麼？")
```

## N-1 測試 (N-1 Testing)

提供第 1 到 N-1 輪作為上下文，測試第 N 輪：

```python
context = conversation[:n-1]
response = chat_with_context(context, user_message_n)
# 與實際的第 N 輪進行比較
```

這可以隔離錯誤是源自上下文還是底層能力。

## 檢查清單 (Checklist)

1. 對話是否達成了目標？（端對端 E2E）
2. 哪一輪首先出錯？
3. 您能透過單輪對話重現嗎？
4. 錯誤是源自上下文還是能力？（N-1 測試）
