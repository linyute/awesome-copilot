# 評估者：概觀 (Evaluators: Overview)

何時以及如何建構自動化評估者。

## 決策框架 (Decision Framework)

```
我應該建構評估者嗎？
        │
        ▼
我能透過提示詞變更來修復它嗎？
    能  → 先修復提示詞
    不能 → 這是重複發生的問題嗎？
          是  → 建構評估者
          否  → 加入觀察名單
```

**不要過早自動化。** 許多問題僅需簡單的提示詞修補。

## 評估者需求 (Evaluator Requirements)

1. **明確標準** - 具體，而非「它好嗎？」
2. **帶標籤的測試集** - 100 個以上帶有人工標籤的範例
3. **測得的準確度** - 部署前需瞭解 TPR/TNR

## 評估者生命週期 (Evaluator Lifecycle)

1. **發現 (Discover)** - 錯誤分析揭示模式
2. **設計 (Design)** - 定義標準與測試案例
3. **實作 (Implement)** - 建構程式碼或 LLM 評估者
4. **校準 (Calibrate)** - 針對人工標籤進行驗證
5. **部署 (Deploy)** - 新增至實驗/CI 管線
6. **監控 (Monitor)** - 追蹤準確度隨時間的變化
7. **維護 (Maintain)** - 隨著產品演進而更新

## 哪些內容「不要」自動化 (What NOT to Automate)

- **罕見問題** - 小於 5 個實例？加入觀察名單即可，不要建構評估者
- **快速修復** - 可透過提示詞變更修復？那就修復它
- **演進中的標準** - 先穩定定義
