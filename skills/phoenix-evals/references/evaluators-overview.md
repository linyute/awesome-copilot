# 評估器：總覽 (Overview)

何時以及如何建立自動化評估器。

## 決策架構 (Decision Framework)

```
我應該建立評估器嗎？
        │
        ▼
我是否可以透過提示 (prompt) 變更來修復它？
    是 (YES) → 先修復提示
    否 (NO)  → 這是一個重複發生的問題嗎？
              是 (YES) → 建立評估器
              否 (NO)  → 加入觀察名單 (watchlist)
```

**不要過早自動化。** 許多問題只需簡單地修正提示即可解決。

## 評估器需求

1. **明確的準則 (Clear criteria)** - 具體具體，而非「它好嗎？」
2. **具備標籤的測試集 (Labeled test set)** - 100 個以上帶有人類標籤的範例
3. **經過測量的準確率 (Measured accuracy)** - 在部署前了解 TPR/TNR

## 評估器生命週期 (Evaluator Lifecycle)

1. **發現 (Discover)** - 錯誤分析揭示模式
2. **設計 (Design)** - 定義準則和測試案例
3. **實作 (Implement)** - 建立程式碼或 LLM 評估器
4. **校準 (Calibrate)** - 針對人類標籤進行驗證
5. **部署 (Deploy)** - 加入實驗/CI 管線
6. **監控 (Monitor)** - 隨時間追蹤準確率
7. **維護 (Maintain)** - 隨著產品演進進行更新

## 哪些內容不應自動化

- **罕見問題** - 少於 5 個案例？加入觀察名單，不要建立評估器
- **快速修復** - 可以透過提示變更來修復？直接修復它
- **演進中的準則** - 先穩定定義
