# 生產環境：護欄 vs 評估者 (Production: Guardrails vs Evaluators)

護欄 (Guardrails) 進行即時阻擋。評估者 (Evaluators) 進行非同步測量。

## 關鍵區別 (Key Distinction)

```
請求 → [輸入護欄] → LLM → [輸出護欄] → 回應
                                  │
                                  └──→ 非同步評估者 (背景執行)
```

## 護欄 (Guardrails)

| 面向 | 需求 |
| ------ | ----------- |
| 時機 | 同步、阻塞 (blocking) |
| 延遲 | < 100 毫秒 |
| 目的 | 防止損害 |
| 類型 | 基於程式碼（確定性） |

**用於：** PII 偵測、提示詞注入、不雅用語、長度限制、格式驗證。

## 評估者 (Evaluators)

| 面向 | 特性 |
| ------ | -------------- |
| 時機 | 非同步、背景執行 |
| 延遲 | 可能需數秒 |
| 目的 | 測量品質 |
| 類型 | 可使用 LLM |

**用於：** 有幫助程度、忠實度、語氣、完整性、引用準確性。

## 決策 (Decision)

| 問題 | 答案 |
| -------- | ------ |
| 必須阻擋有害內容嗎？ | 護欄 |
| 正在測量品質嗎？ | 評估者 |
| 需要 LLM 判斷嗎？ | 評估者 |
| 必須在 100 毫秒內完成嗎？ | 護欄 |
| 誤判會導致使用者不滿嗎？ | 評估者 |

## LLM 護欄：極少使用 (LLM Guardrails: Rarely)

僅在以下情況使用 LLM 護欄：
- 延遲預算 > 1 秒
- 錯誤成本遠大於 LLM 成本
- 流量低
- 具備備援方案

**關鍵原則：** 護欄防止損害（阻擋）。評估者測量品質（記錄）。
