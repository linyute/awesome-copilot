# 生產：護欄 (Guardrails) vs 評估器 (Evaluators)

護欄 (Guardrails) 會進行即時阻塞。評估器 (Evaluators) 則進行非同步測量。

## 關鍵區別 (Key Distinction)

```
請求 → [輸入護欄] → LLM → [輸出護欄] → 回應
                                            │
                                            └──→ 非同步評估器 (ASYNC EVALUATOR) (背景執行)
```

## 護欄 (Guardrails)

| 面向 | 需求 |
| ------ | ----------- |
| 時間點 | 同步、阻塞 |
| 延遲 | < 100ms |
| 目的 | 防止損害 |
| 類型 | 基於程式碼 (決定性) |

**適用於：** PII 偵測、提示詞注入 (prompt injection)、髒話過濾、長度限制、格式驗證。

## 評估器 (Evaluators)

| 面向 | 特性 |
| ------ | -------------- |
| 時間點 | 非同步、背景執行 |
| 延遲 | 可能長達數秒 |
| 目的 | 測量品質 |
| 類型 | 可使用 LLM |

**適用於：** 幫助程度 (helpfulness)、忠實度 (faithfulness)、語氣、完整性、引用準確性。

## 決策 (Decision)

| 問題 | 答案 |
| -------- | ------ |
| 必須阻塞有害內容？ | 護欄 |
| 正在測量品質？ | 評估器 |
| 需要 LLM 判定？ | 評估器 |
| 需要 < 100ms 的回應？ | 護欄 |
| 偽陽性 (False positives) 會導致使用者不滿？ | 評估器 |

## LLM 護欄：極少使用

僅在下列情況使用 LLM 護欄：
- 延遲預算 > 1s
- 錯誤成本 >> LLM 成本
- 低流量
- 存在回退方案 (Fallback)

**關鍵原則：** 護欄防止損害（阻塞）。評估器測量品質（紀錄）。
