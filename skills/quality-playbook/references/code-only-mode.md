# 僅原始碼模式 (Code-only mode)

*最後更新時間：2026-05-03 (v1.5.6 階段 3 — 首次發佈)。*

當品質播放手冊針對目標存放庫執行時，若其 `reference_docs/` 目錄不存在或為空，播放手冊將以 **僅原始碼模式** 運作。此文件說明了該模式的意義、其重要性，以及如何為下一次執行升級為完整文件模式。

## 「僅原始碼模式」的意義

播放手冊正常的階段 1 推導會讀取兩類證據：

- **原始碼證據 (第 3 層以上)** — 原始碼樹本身，外加行內註解、防禦性模式、測試，以及任何與原始碼共存的行內文件。
- **文件證據 (第 1/2 層)** — 操作員放入 `reference_docs/` 的純文字檔案（自由格式的筆記、設計文件、回顧內容、AI 聊天記錄）以及 `reference_docs/cite/` 下的檔案（專案規格、RFC、需求應具備可追溯性的 API 合約）。

僅原始碼模式是指文件證據不可用時的執行狀態。播放手冊會繼續執行 — 不會中止 — 但其衍生的每項需求都將完全依賴原始碼證據。階段 1 的 EXPLORATION.md 會獲得一個「文件狀態：僅原始碼模式」的開場章節，以突顯該模式，讓審查人員在第一時間看到。

## 對僅原始碼執行的預期

在我們的基準測試執行中，僅原始碼模式始終會產生以下結果：

- **衍生的需求總數較少。** 由於缺乏可用於錨定的規格語言，階段 1 沒有可引用的第 1/2 層證據，因此需求集完全退化為第 3 層 (程式碼即規格)。
- **發現的漏洞可能較少。** 當審查人員知道程式碼「應該」執行什麼操作時，程式碼審查 (階段 3) 最為有效 — 違反記錄在案之意圖的漏洞比隱藏在模糊的程式碼即規格背後的漏洞更容易浮現。在沒有文件的情況下，審查人員必須從程式碼本身推斷意圖，這會導致一類違反意圖的缺陷無法被偵測。
- **對程式碼內部訊號的依賴程度更高。** 在缺乏外部文件的情況下，防禦性模式 (錯誤檢查、驗證)、測試名稱和註解樣式的標註具有更大的權重。

僅原始碼模式下的漏洞計數仍然有用 — 它們反映了僅從原始碼中可發現的內容 — 但它們是完整文件執行所能產出內容的下限。

## 如何升級為完整文件執行

在重新執行階段 1 之前，將純文字文件檔案放入目標存放庫的 `reference_docs/` 樹狀結構中：

```
<target-repo>/
  reference_docs/
    project_notes.md         # 第 4 層 — 非正式筆記、AI 聊天記錄
    design_overview.md       # 第 3-4 層 — 內部設計決策
    cite/
      api_spec.md            # 第 1/2 層 — 可引用的規格、RFC、合約
      protocol_v3.txt        # 第 1/2 層 — 正式規格
```

位於 `reference_docs/` 頂層的檔案被視為非正式背景資訊 (第 4 層)。位於 `reference_docs/cite/` 下的檔案被視為可引用的證據 (根據來源的權威性，為第 1 層或第 2 層 — 參見 `schemas.md` §3.1)。`.md` 和 `.txt` 格式皆可識別；其他格式將被忽略。

放入文件後，重新執行播放手冊。階段 1 將偵測到已填充的 `reference_docs/` 並跳過僅原始碼模式的降級。新執行的 EXPLORATION.md、REQUIREMENTS.md 和 BUGS.md 將反映更豐富的證據基礎。

## 選擇性退出：`--require-docs`

若操作員希望執行在 `reference_docs/` 為空時中止，而非進入僅原始碼模式，可以在執行 `python3 -m bin.run_playbook` 時傳遞 `--require-docs` 旗標 (v1.5.6+)。當設定了 `--require-docs` 且在階段 1 進入時 `reference_docs/` 為空，播放手冊會執行以下操作：

1. 向 `quality/run_state.jsonl` 附加一個 `aborted_missing_docs` 事件 (該事件類型已在 `references/run_state_schema.md` 中註冊)。
2. 在 `quality/PROGRESS.md` 中寫入一個清晰的 `ERROR: aborted_missing_docs — reference_docs/ empty and --require-docs set` 區塊。
3. 在進行任何 LLM 工作前中止執行 (以非零狀態碼退出，等同於閘道失敗)。

此旗標預設為關閉。請在合規性/政策環境中使用它，因為在這些環境中，安靜的僅原始碼模式降級會遮蓋真實的流程缺口 (例如，「每次發佈執行都必須引用規格；沒有規格意味著執行不應開始」)。此旗標是 `--no-formal-docs` 的選擇性退出 (opt-OUT) 對應項 (後者抑制了相同僅原始碼模式情況下的 WARN 橫幅，但允許執行繼續)。
