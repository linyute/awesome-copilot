# 新手模式詞彙表 (Noob Mode Glossary)

使用 Copilot CLI 時會遇到的技術術語淺白用語參考。按類別整理。

---

## 🗂️ Git 與版本控制 (Git & Version Control)

### 儲存庫 (Repository, repo)
**淺白用語：** 一個會記住檔案所有變更歷史的專案資料夾。
**類比：** 一個內建時光機的檔案櫃 —— 您可以隨時取出任何文件在過去任何時間點的版本。
**情境範例：** 「我會查看此儲存庫中的檔案」 = 「我會查看此專案資料夾中的檔案」。

### 分支 (Branch)
**淺白用語：** 您專案的一個獨立複本，您可以在其中嘗試變更而不影響原始版本。
**類比：** 影印一份合約來測試修改內容，而不觸動已簽署的正本。
**情境範例：** 「我會建立一個新分支」 = 「我會建立一個可以安全實驗的複本」。

### 提交 (Commit)
**淺白用語：** 儲存您工作的一份快照，並附上關於變更內容的註記。
**類比：** 在每次工作結束時拍一張桌面照片，並貼上一張便利貼說明您做了什麼。
**情境範例：** 「我會提交這些變更」 = 「我會為我剛才做的操作儲存一份快照」。

### 合併 (Merge)
**淺白用語：** 將一個複本中的變更整合回原始版本。
**類比：** 將您在影印本上標註的修改內容正式寫入官方合約中。
**情境範例：** 「讓我們合併此分支」 = 「讓我們將這些變更整合回主版本」。

### 提取請求 (Pull Request, PR)
**淺白用語：** 一份正式請求，表示「我做了這些變更 —— 有人能在我們正式採用前檢閱一下嗎？」。
**類比：** 在文件交給客戶前，先提交給合夥律師進行審閱。
**情境範例：** 「我會開啟一個提取請求」 = 「我會提交這些變更以供檢閱」。

### 複製 (Clone)
**淺白用語：** 從伺服器下載專案的完整複本到您的電腦。
**類比：** 從律師事務所的伺服器取得您專屬的一份共享案件檔案。
**情境範例：** 「複製儲存庫」 = 「將專案下載到您的電腦」。

### 分叉 (Fork)
**淺白用語：** 在伺服器上建立一份他人專案的個人專屬複本。
**類比：** 取得一份範本的複本，您可以隨意自訂它而不影響原始範本。
**情境範例：** 「分叉此儲存庫」 = 「建立此專案的個人複本」。

### 差異 (Diff)
**淺白用語：** 一個顯示兩個版本之間具體變更內容 (新增、刪除或修改) 的對照表。
**類比：** Word 中的「追蹤修訂」，顯示紅色的刪除線與藍色的新增內容。
**情境範例：** 「讓我檢查差異」 = 「讓我看看具體變更了什麼」。

### 暫存區 (Staging Area, Index)
**淺白用語：** 一個等待區域，您在儲存快照前將檔案放在這裡。就像「待提交」堆疊。
**類比：** 桌面上的寄件匣 —— 您已決定要寄出但尚未正式投遞的文件。
**情境範例：** 「我會暫存這些檔案」 = 「我會將這些檔案標記為已準備好儲存」。

### 遠端 (Remote)
**淺白用語：** 存在於伺服器 (如 GitHub) 上的專案複本，與您電腦上的複本相對。
**類比：** 律師事務所雲端儲存空間中的主檔，與您筆記型電腦上的複本。
**情境範例：** 「推送至遠端」 = 「將您的變更上傳至伺服器」。

### 推送 (Push)
**淺白用語：** 將您電腦上已儲存的變更上傳至共用伺服器。
**類比：** 將您本地的編輯內容同步回共享磁碟，讓每個人都能看到。
**情境範例：** 「我會推送這些提交」 = 「我會將這些已儲存的變更上傳至伺服器」。

### 拉取 (Pull)
**淺白用語：** 從共用伺服器下載最新的變更至您的電腦。
**類比：** 用同事所做的任何更新來重新整理您的本地複本。
**情境範例：** 「拉取最新的變更」 = 「從伺服器下載任何更新」。

### 檢出 (Checkout)
**淺白用語：** 切換到專案的不同分支或版本。
**類比：** 從檔案櫃中取出不同版本的文件來處理。
**情境範例：** 「檢出 main 分支」 = 「切換回專案的主版本」。

### 衝突 (Conflict)
**淺白用語：** 當兩個人更改了同一個檔案的同一個部分，且電腦無法自動判斷該保留哪個版本。
**類比：** 兩位律師對合約的同一段落做了不同的修改 —— 需要有人決定最終採用哪個版本。
**情境範例：** 「發生合併衝突」 = 「兩組變更重疊，需要由您決定保留哪一組」。

### 暫存 (Stash)
**淺白用語：** 暫時儲存您目前的進度，以便切換到其他工作，稍後再回來處理。
**類比：** 將目前的文件放入抽屜，以便清理桌面處理緊急任務，之後再重新拿出。
**情境範例：** 「我會暫存您的變更」 = 「我會暫時儲存您目前的進度」。

### HEAD
**淺白用語：** 您目前正在查看或處理的專案版本。
**類比：** 目前在您螢幕上開啟的文件。
**情境範例：** 「HEAD 指向 main」 = 「您目前正在查看主版本」。

### 標記 (Tag)
**淺白用語：** 附加在特定版本上的永久標籤，通常用於標示發行版或里程碑。
**類比：** 在合約的特定版本貼上「最終版 —— 客戶已簽署」的貼紙。
**情境範例：** 「我會將此標記為 v1.0」 = 「我會將此版本標記為正式的 1.0 發行版」。

## 💻 檔案系統與 Shell (File System & Shell)

### 終端機 (Terminal, Console)
**淺白用語：** 您電腦的文字控制面板。您輸入指令而非點擊按鈕。
**類比：** 就像透過傳送簡訊向您的電腦下達指令，而非指指點點。
**情境範例：** 「開啟您的終端機」 = 「開啟您輸入指令的應用程式」。

### Shell (Bash, Zsh)
**淺白用語：** 在終端機內執行並解讀您所輸入指令的程式。
**類比：** 電話線另一端負責執行您請求的接線生。
**情境範例：** 「在您的 Shell 中執行此指令」 = 「在您的終端機中輸入此指令」。

### CLI (命令列介面, Command Line Interface)
**淺白用語：** 您透過輸入指令而非點擊視覺化介面來互動的程式。
**類比：** 在餐廳向服務生口頭點餐 (CLI)，而非在平板選單上點擊 (GUI)。
**情境範例：** 「使用 CLI」 = 「輸入指令而非點擊按鈕」。

### 目錄 (Directory)
**淺白用語：** 電腦上的一個資料夾。
**類比：** 它就是資料夾。我們只是有時會用比較高級的詞。
**情境範例：** 「導覽至此目錄」 = 「前往此資料夾」。

### 路徑 (Path)
**淺白用語：** 檔案或資料夾在電腦上的位址，顯示到達它所需經過的每個資料夾。
**類比：** 就像街道地址：國家 / 省份 / 城市 / 街道 / 建築物 —— 但這是給檔案用的。
**情境範例：** 「`~/Desktop/contracts/nda.md`」 = 「一個名為 nda.md 的檔案，位於桌面上的 contracts 資料夾中」。

### 根目錄 (Root, /)
**淺白用語：** 電腦上的最頂層資料夾 —— 所有其他資料夾都存在於其中。
**類比：** 建築物的大廳 —— 每一層樓與房間都必須從這裡進入。
**情境範例：** 「根目錄」 = 「電腦上最頂層的資料夾」。

### 家目錄 (Home Directory, ~)
**淺白用語：** 您在電腦上的個人資料夾。在 Mac 上，它是 `/Users/您的名稱`。
**類比：** 建築物中您個人的辦公室。
**情境範例：** 「`~/Desktop`」 = 「您個人資料夾內的桌面資料夾」。

### 環境變數 (Environment Variable)
**淺白用語：** 儲存在您電腦上的一個設定，程式可以讀取它。就像貼在螢幕上供應用程式查看的便利貼。
**類比：** 程式可以查看的名牌，用來瞭解您的設定資訊。
**情境範例：** 「設定環境變數」 = 「儲存一個程式可以查閱的設定」。

### 管道 (Pipe, |)
**淺白用語：** 將一個指令的輸出傳送給另一個指令，就像生產線一樣。
**類比：** 在接力賽中將文件從一個人手中傳交給下一個人。
**情境範例：** 「`grep 'term' file.txt | wc -l`」 = 「尋找該詞彙，然後計算它出現的次數」。

### 重新導向 (Redirect, >, >>)
**淺白用語：** 將輸出傳送到檔案而非顯示在螢幕上。`>` 會覆寫檔案；`>>` 則會附加在後。
**類比：** 與其大聲朗讀報告，不如將其寫在紙上。
**情境範例：** 「`echo 'hello' > file.txt`」 = 「將 'hello' 寫入 file.txt (覆寫原本的內容)」。

### 權限 (Permissions)
**淺白用語：** 關於誰可以讀取、編輯或執行檔案的規則。顯示為 `rwx` 代碼或數字如 `755`。
**類比：** 就像 SharePoint 中的文件權限 —— 誰可以檢視、誰可以編輯、誰可以分享。
**情境範例：** 「變更檔案權限」 = 「變更誰被允許讀取或編輯此檔案」。

### 符號連結 (Symlink, Symbolic Link)
**淺白用語：** 指向另一個檔案或資料夾的捷徑。它不是複本 —— 僅僅是一個指標。
**類比：** 文件中的超連結，會帶您前往原始檔案。
**情境範例：** 「建立符號連結」 = 「建立一個指向另一個檔案的捷徑」。

### 標準輸出 / 標準錯誤 (stdout / stderr)
**淺白用語：** 程式輸出的兩個頻道。stdout 是正常輸出；stderr 是錯誤訊息。
**類比：** stdout 是會議中的主要發言者；stderr 是某人傳送的緊急紙條。
**情境範例：** 「重新導向 stderr」 = 「將錯誤訊息傳送到特定地方」。

### 腳本 (Script)
**淺白用語：** 一個包含多個會自動依序執行指令的檔案。
**類比：** 食譜 —— 依序遵循步驟，您就能得到結果。
**情境範例：** 「執行此腳本」 = 「執行此預先寫好的指令清單」。

## 🔧 開發概念 (Development Concepts)

### API (應用程式發展介面, Application Programming Interface)
**淺白用語：** 兩個程式之間互相溝通的一種方式，遵循雙方同意的規則。
**類比：** 餐廳的服務生 —— 將您的訂單傳達給廚房，並將您要求的東西帶回來。
**情境範例：** 「呼叫 API」 = 「傳送請求給另一個程式並獲得回應」。

### 端點 (Endpoint)
**淺白用語：** API 接受請求的特定網址，就像公司內部的特定分機。
**類比：** 政府機關中處理特定類型請求的特定櫃台。
**情境範例：** 「/users 端點」 = 「處理使用者相關請求的特定位址」。

### 伺服器 (Server)
**淺白用語：** 當您請求時提供服務的電腦 (或程式)。它等待請求並給予回應。
**類比：** 圖書館管理員 —— 您向他借書，他找到後遞給您。
**情境範例：** 「啟動伺服器」 = 「開啟負責監聽並回應請求的程式」。

### 用戶端 (Client)
**淺白用語：** 傳送請求給伺服器的程式。您的網頁瀏覽器就是一個用戶端。
**類比：** 向管理員借書的圖書館讀者。
**情境範例：** 「用戶端傳送請求」 = 「您的程式向伺服器請求某樣東西」。

### 資料庫 (Database)
**淺白用語：** 一個有組織的資料集合，程式可以從中搜尋、新增、更新與刪除資料。
**類比：** 一個非常高級的試算表，多個程式可以同時讀取與寫入。
**情境範例：** 「查詢資料庫」 = 「在資料儲存空間中尋找資訊」。

### 相依性 (Dependency)
**淺白用語：** 一個專案為了運作而需要的預建工具或函式庫。
**類比：** 您書架上進行研究所需的參考書。
**情境範例：** 「安裝相依性」 = 「下載此專案所需的所有工具與函式庫」。

### 套件 (Package)
**淺白用語：** 由他人撰寫並打包好供他人使用的程式碼包。
**類比：** 五金行販售的預建工具箱 —— 讓您不必自己打造工具。
**情境範例：** 「安裝套件」 = 「下載並將此預建工具箱加入您的專案」。

### 模組 (Module)
**淺白用語：** 處理一件特定事情的自給自足程式碼片段。
**類比：** 書中的一個章節 —— 它涵蓋一個主題且可以相對獨立地閱讀。
**情境範例：** 「匯入 auth 模組」 = 「載入處理登入/安全性的程式碼片段」。

### 框架 (Framework)
**淺白用語：** 一個預建的基礎，為您提供建構的結構，並具備如何組織程式碼的規則。
**類比：** 法律訴狀範本 —— 它給您結構，由您填入內容實體。
**情境範例：** 「我們正在使用 React 框架」 = 「我們正在名為 React 的預設結構上進行建構」。

### 建構 (Build)
**淺白用語：** 將原始程式碼轉換為實際可執行的形式。
**類比：** 將 Word 文件轉換為最終的 PDF —— 內容相同，但格式已準備好分發。
**情境範例：** 「執行建構」 = 「將程式碼轉換為完成且可執行的形式」。

### 編譯 (Compile)
**淺白用語：** 將人類撰寫的語言翻譯成電腦能理解的語言。
**類比：** 將一份合約從英文翻譯成日文，以便另一方能夠閱讀。
**情境範例：** 「編譯程式碼」 = 「將其翻譯成電腦可讀的形式」。

### Lint / Linter
**淺白用語：** 一種工具，在不執行程式碼的情況下檢查其中的常見錯誤、樣式問題與潛在風險。
**類比：** 程式碼的拼字與語法檢查器。
**情境範例：** 「執行 Linter」 = 「檢查程式碼中的錯誤與樣式問題」。

### 測試 (Test，包含單元測試、整合測試)
**淺白用語：** 自動檢查其他程式碼是否運作正確的程式碼。
**類比：** 一份品質保證 (QA) 檢查清單 —— 「登入頁面能用嗎？搜尋能回傳結果嗎？儲存按鈕真的有存檔嗎？」。
**情境範例：** 「執行測試」 = 「自動驗證一切是否仍正常運作」。

### 執行階段 (Runtime)
**淺白用語：** 程式碼實際執行的環境。也指程式碼正在執行的時間段。
**類比：** 戲劇表演的舞台 (相對於劇本，即程式碼)。
**情境範例：** 「執行階段錯誤」 = 「程式在實際執行時出了問題」。

### 部署 (Deploy)
**淺白用語：** 將完成的程式碼放到人們可以使用的地方 (伺服器、網站、應用程式商店)。
**類比：** 出版完成的書籍 —— 從作者的桌面移到書店的書架上。
**情境範例：** 「部署至正式環境 (production)」 = 「將此功能提供給真實使用者使用」。

## 🌐 網路 (Web & Networking)

### URL (統一資源定位符)
**淺白用語：** 一個網址。您在瀏覽器網址列中輸入的文字。
**類比：** 街道地址，但是給網站用的。
**情境範例：** 「`https://github.com/settings`」 = 「GitHub 網站上的設定頁面」。

### HTTP / HTTPS
**淺白用語：** 網頁瀏覽器與伺服器互相溝通的語言。HTTPS 是安全 (加密) 版本。
**類比：** HTTP 是寄送明信片 (任何人都能讀取)；HTTPS 是寄送密封且上鎖的信封。
**情境範例：** 「發送 HTTP 請求」 = 「傳送訊息給網頁伺服器」。

### JSON (JavaScript 物件標記法)
**淺白用語：** 一種易於人類與電腦讀取的標準資料結構化格式。
**類比：** 一份整理得非常好且帶有欄位標籤的表格，例如：`{ "姓名": "小明", "職稱": "律師" }`。
**情境範例：** 「API 回傳 JSON」 = 「回應內容是以結構化、帶標籤的資料形式傳回」。

### 權杖 (Token)
**淺白用語：** 一個證明您身分的數位鑰匙或通行證，用來代替每次輸入密碼。
**類比：** 大樓進出證 —— 您刷卡即可進入，而不必每次都出示身分證。
**情境範例：** 「您的驗證權杖」 = 「證明您已登入的數位鑰匙」。

### 狀態碼 (Status Code)
**淺白用語：** 網頁伺服器傳回的一個數字，用來告訴您請求是否成功。常見的有：
- **200** = 成功 (「這是您要求的內容」)
- **404** = 找不到 (「該頁面或項目不存在」)
- **500** = 伺服器錯誤 (「我們這邊出了點問題」)
- **401** = 未經授權 (「您需要先登入」)
- **403** = 禁止存取 (「您已登入但沒有權限」)

### Localhost
**淺白用語：** 一種將您自己的電腦當作網頁伺服器來稱呼的方式。用於測試。
**類比：** 在會議室正式報告前，先在自己的辦公室進行預演。
**情境範例：** 「`http://localhost:3000`」 = 「在您自己電腦上的 3000 頻道執行的網站」。

### 連接埠 (Port)
**淺白用語：** 電腦上的編號頻道。不同的服務使用不同的連接埠，就像不同的電視頻道。
**類比：** 無線電頻率 —— 每個電台在各自的頻率廣播，以免互相干擾。
**情境範例：** 「在 3000 連接埠執行」 = 「使用您電腦上的 3000 頻道」。

### REST (RESTful API)
**淺白用語：** 一種建立網頁 API 的常見風格，您使用標準的網址與動作 (GET, POST 等) 來與資料互動。
**類比：** 一套標準化的表單系統 —— 每個人都同意如何提交、擷取、更新與刪除記錄。
**情境範例：** 「一個 RESTful 端點」 = 「一個遵循標準資料存取慣例的網址」。

## 🤖 Copilot CLI 相關

### MCP 伺服器 (Model Context Protocol)
**淺白用語：** 一個為 Copilot CLI 提供額外能力的附加元件 —— 就像網頁瀏覽器的外掛程式。
**類比：** 在手機上安裝一個新應用程式，增加原本沒有的功能。
**情境範例：** 「設定一個 MCP 伺服器」 = 「設定一個能賦予 Copilot 新能力的附加元件」。

### 工具呼叫 (Tool Call)
**淺白用語：** 當 Copilot 要求使用其內建能力 (讀取檔案、執行指令、搜尋網路等) 時。
**類比：** 助理在執行前詢問：「我可以開啟這個檔案櫃嗎？」。
**情境範例：** 「核准此工具呼叫」 = 「准許我使用這項特定能力」。

### 核准提示 (Approval Prompt)
**淺白用語：** Copilot 在執行某些操作前，停下來請求您許可的時刻。
**類比：** 助理說：「在我寄出這封郵件前，您可以先檢閱一下嗎？」。
**情境範例：** 「我需要您的核准」 = 「我在採取行動前正請求您的許可」。

### 上下文視窗 (Context Window)
**淺白用語：** Copilot 同一時間能記住的對話與資訊總量。當它填滿時，較舊的部分會被總結或遺忘。
**類比：** 辦公桌的大小 —— 在您必須歸檔某些文件前，桌上能攤開的文件數量有限。
**情境範例：** 「上下文視窗快滿了」 = 「我的工作記憶體不足，可能需要總結我們之前的對話」。

### 模型 (Model)
**淺白用語：** 驅動 Copilot 的 AI 大腦。不同的模型 (如 Sonnet, GPT, Gemini) 各有優缺點。
**類比：** 不同的搜尋引擎 (如 Google, Bing) —— 它們都能搜尋網路，但運作方式不同，結果也略有差異。
**情境範例：** 「切換到不同模型」 = 「換一個 AI 大腦」。

### 權杖 (Token)
**淺白用語：** AI 模型處理文字的單位。大致上，1 個權杖 ≈ 0.75 個單字。
**類比：** 如果 AI 是以音節而非完整單字來閱讀，每個音節就是一個權杖。
**情境範例：** 「這耗費了 1,000 個權杖」 = 「這大約相當於 750 個單字的 AI 處理量」。

### 技能 (Skill)
**淺白用語：** 您可以為 Copilot CLI 增加的一種處理特定類型任務的專業能力。
**類比：** 您可以請來的專家 —— 就像請來稅務專家 vs. 合約專家。
**情境範例：** 「啟用一項技能」 = 「開啟一項專業能力」。

### 外掛程式 (Plugin)
**淺白用語：** 由第三方提供，用於擴展 Copilot CLI 功能的附加元件。
**類比：** 瀏覽器擴充功能 —— 由他人開發，您安裝後可以增加功能。
**情境範例：** 「安裝一個外掛程式」 = 「為 Copilot 增加一項第三方功能」。

### 工作階段 (Session)
**淺白用語：** 您與 Copilot CLI 之間從開始到關閉的一次連續對話。
**類比：** 一通電話 —— 在您掛斷前，電話中討論的一切都屬於同一次對話。
**情境範例：** 「恢復工作階段」 = 「從上次離開的地方接續之前的對話」。

### 自訂指引 (Custom Instructions)
**淺白用語：** 一個告訴 Copilot 如何表現的檔案 —— 包含您的偏好、規則與樣式要求。
**類比：** 您給新助理的任務簡報：「這是我喜歡的備忘錄格式，以及哪些事需要優先處理」。
**情境範例：** 「切換自訂指引」 = 「開啟或關閉 Copilot 的特定行為規則集」。
