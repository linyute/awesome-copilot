# 每日準備 (Daily Prep)

為下一個工作日產生結構化的準備檔案，包含會議詳情、準備要點、連結的任務以及生產力建議。

## 何時使用

- 工作日結束時：「幫我準備明天的內容」
- 任何時間：「幫我準備週五的內容」或「3 月 25 日的時程如何？」
- 每週規劃：針對多天執行

## 流程

### 1. 確定目標日期

如果使用者指定了日期，則使用該日期。否則，預設為明天（目前日期 + 1 天）。
如果明天是週六，則預設為週一。如果是週日，則預設為週一。
計算輸出路徑：`outputs/YYYY/MM/YYYY-MM-DD-prep.html`

### 2. 透過 WorkIQ 提取行事曆

使用 WorkIQ MCP 工具擷取行事曆。詢問 WorkIQ：

> 「我在 {目標日期} 有哪些會議？對於每個會議，請包含：主題、開始時間、結束時間、召集人、所有參與者及其電子郵件地址、地點、是否為線上會議，以及我是否已接受或拒絕。」

如果回覆不足，請進行後續查詢：

> 「對於 {目標日期} 的會議，哪些被標記為選用或暫定？哪些是週期性會議？」

### 3. 將每個會議分類

根據參與者網域和主題套用以下標籤：

| 標籤 | 準則 |
|-------|----------|
| `[客戶 · 高]` | 來自客戶/夥伴網域的外部參與者，或主題符合已知的客戶名稱 |
| `[內部]` | 僅限公司內部網域參與者 |
| `[社群]` | CoP、社群、公會、學習課程 |
| `[技能提升]` | 訓練、工作坊、認證、學習 |
| `[選用 · 跳過]` | 暫定、重要性低，或已知的週期性選用會議（例如「Office Hours」、「開放 Q&A」） |
| `[個人]` | 私人活動、非工作 |

#### 區域標記

對於每個會議，檢查召集人欄位並套用這些額外標記：

| 條件 | 標記 | 動作 |
|-----------|--------|--------|
| 開始時間 ≥ 15:30 且 < 16:00 (任何召集人) | `⚠️ 下班後` | 建議拒絕 |
| 開始時間 ≥ 16:00 且 **非** 自行發起 | `⚠️ 下班後` | 建議拒絕 |
| 開始時間 ≥ 16:00 且為自行發起 | _(無標記)_ | OK — 這是您選擇安排的 |
| 09:00 之前且 **非** 自行發起 | `⚠️ 過早` | 建議拒絕 — 侵入學習時間窗 |
| 09:00 之前且為自行發起 | _(無標記)_ | OK — 這是您選擇安排的 |
| 與 12:00–13:00 重疊 | `🍽️ 午餐衝突` | 在行事曆備註中註明 |

「自行發起」代表 **您** 是會議召集人（檢查 WorkIQ 的召集人欄位）。

### 4. 理想的一天結構

將此作為所有分析步驟的決策框架。每個會議都必須對照這些區域進行評估。使用者應根據個人習慣調整這些時間和目標。

| 區域 | 時間 | 目的 | 規則 |
|------|------|---------|-------|
| 早晨專注 | 09:00 之前 | 行政、學習、個人工作 | 保護其不受他人會議干擾。標記外部活動。 |
| 客戶時段 | 09:00–12:00 | 客戶 / 外部會議 | 最多 2 個客戶會議。優先選擇早晨進行外部通話。 |
| 午餐 | 12:00–13:00 | 休息 | 受保護。標記任何重疊。 |
| 深度工作 | 13:00–15:30 | 交付成果、專注寫程式碼/寫作 | 盡量減少會議。將非必要會議標記為深度工作干擾。 |
| 受保護（嚴格） | 15:30–16:00 | 工作日結束收尾 | 不論召集人是誰，標記所有會議。 |
| 受保護（彈性） | 16:00+ | 工作日結束 | 僅標記他人的會議。自行發起則 OK。 |

**每日目標：**
- 學習時數：**1.5 小時**（來自早晨專注 + 空檔時間）
- 深度工作時數：**2.5 小時**（13:00–15:30 區域）
- 客戶會議：**最多 2 個**（優先在 09:00–12:00）

### 5. 偵測衝突與時程契合度問題

比較活動的時間窗。在衝突表中標記重疊，並為每個重疊提供建議 — 優先考慮客戶會議，而非內部/選用會議。

同時偵測這些 **時程契合度問題**（在獨立的「時程契合度問題」表中回報）：

| 檢查項 | 條件 | 標記 |
|-------|-----------|------|
| **客戶超載** | >2 個 `[客戶 · 高]` 會議 | 將第 3 個及以後的標記為「考慮改期至另一天」 |
| **深度工作干擾** | 13:00–15:30 區域內的非必要會議 | 「干擾深度工作 — 考慮移至早晨」 |
| **非理想位置** | 09:00–12:00 以外的客戶會議 | 「客戶會議不在偏好的早晨區域」 |
| **過早侵入** | 09:00 之前的他人會議 | 「侵入學習時間窗 — 建議拒絕」 |
| **午餐衝突** | 會議與 12:00–13:00 重疊 | 「與午餐休息衝突」 |

### 6. 從工作空間收集上下文

1. 讀取未完成任務檔案，尋找與明天會議中的客戶名稱或參與者相關的任務
2. 搜尋工作空間資料夾，尋找與那些客戶或主題相關的近期檔案
3. 檢查近期的會議摘要或計畫，以取得相關的準備上下文
4. 使用這些資訊為每個會議產生可執行的準備要點

### 7. 為每個會議產生準備內容

依時間順序為每個會議包含：
- 時間、主題、召集人
- 參與者名單（名字，如果是外部則包含公司）
- 3–5 個根據未完成任務、近期摘要和會議主題產生的可執行準備要點
- 若無可用上下文，請註明在會議中需要詢問/釐清的事項

### 8. 尋找學習與專注時段

為每個會議產生準備內容後，分析當天的時程以尋找空檔時段：

1. **早晨專注確認** — 驗證早晨專注時間窗是否為空。若存在任何非自行發起的活動，請標記。

2. **學習時段** — 在早晨時間窗尋找 ≥ 30 分鐘的空檔，以及任何其他適合提升技能的空閒時段。目標：**1.5 小時/天**。對於每個時段：時間範圍、持續時間、建議活動。

3. **深度工作區塊** — 在 13:00–15:30 區域尋找連續的空閒空檔以處理交付成果。對於每個區塊：時間範圍、持續時間、來自未完成任務的建議任務。

4. **回報總計：**
   - 找到的學習時數 vs. 1.5 小時目標（例如：「1.0 小時 / 1.5 小時目標 — 短少 0.5 小時」）
   - 13:00–15:30 內可用的深度工作時數（例如：「2.0 小時 / 2.5 小時可用」）

### 9. 生產力建議

分析完整的一天並提供：

| 區段 | 包含內容 |
|---------|------------------|
| **時程契合得分** | 評分 0–100%，衡量當天時程與「理想的一天結構」的契合度。準則：(1) 早晨專注清空 (+20%)、(2) 09:00–12:00 內 ≤2 個客戶會議 (+20%)、(3) 12:00–13:00 午餐受到保護 (+15%)、(4) 13:00–15:30 深度工作完整 (+20%)、(5) 15:30 後無會議或 16:00 後僅有自行發起會議 (+15%)、(6) 找到 ≥1 小時學習時段 (+10%)。顯示為：🟢 ≥80%、🟡 50–79%、🔴 <50%。 |
| **當天輪廓** | 總會議時數、可用專注時間、學習時數、深度工作時數、負荷（沉重/適中/輕微）評估 |
| **建議拒絕候選** | 自動包含：(1) 15:30–16:00 的所有會議、(2) 16:00 以後的他人會議、(3) 09:00 之前的他人會議、(4) 第 3 個及以後的客戶會議、(5) 深度工作時段內的選用會議。顯示「回收」欄位，標明恢復的分鐘數。09:00 之前或 16:00 之後的自行發起會議 **不包含** 在自動拒絕中。 |
| **衝突解決** | 針對每個重疊提供具體建議 |
| **學習時段** | 提升技能的空檔 — 來自步驟 8。表格：時間窗、持續時間、建議活動。顯示總計 vs. 1.5 小時目標。 |
| **深度工作區塊** | 13:00–15:30 內處理交付成果的空閒空檔 — 來自步驟 8。表格：時間窗、持續時間、建議任務。 |
| **精力管理** | 若連續超過 3 小時的客戶會議且無休息，請標記 |
| **前 3 大優先事項** | 最具影響力的 3 件待辦事項（會議與任務結合） |

### 10. 寫入檔案

在 `outputs/YYYY/MM/YYYY-MM-DD-prep.html` 建立輸出檔案，作為一個包含內嵌 CSS 的獨立 HTML 檔案（深色主題、顏色編碼的時間軸、響應式佈局）。

若該日期已存在檔案，請先讀取並更新而非覆蓋 — 使用者可能已手動新增備註。

## 範例提示詞

- 「幫我準備明天的內容」
- 「週五的時程如何？」
- 「3 月 28 日的每日準備」
- 「幫我準備下週一的內容 — 著重於客戶會議」

## 需求

- 必須提供 **WorkIQ MCP 工具** 以存取行事曆 (Microsoft 365 / Outlook)
- 包含任務檔案與客戶/專案資料夾的工作空間，用於豐富上下文
- 輸出為獨立的 HTML — 無外部依賴項目
