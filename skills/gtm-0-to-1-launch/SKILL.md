# 0 到 1 的產品發布 (0-to-1 Launch)

將新產品從構思推向首批客戶。目標不是上頭條 — 而是找到 10 位少了你就無法運作的客戶。

## 何時使用

**觸發情境：**
- 「我們該如何發布這款產品？」
- 「首批客戶獲取策略」
- 「我們發布了產品，但沒人在用」
- 「該在 Product Hunt 發布還是進行直接外展 (outreach)？」
- 「我們有知名度但沒有轉換」
- 「我該如何知道這是否奏效？」

**背景：**
- 新產品發布
- 感覺像是新產品的功能發布
- 尋找首批 10-50 位客戶
- 驗證產品市場契合度 (product-market fit, PMF)
- 診斷早期動能停滯的原因

---

## 核心框架

### 1. 媒體報導 ≠ 成長（僅獲得 12 人註冊的發布案例）

**模式：**

協調了一次功能發布，並進行了完整的媒體巡迴。登上 TechCrunch、VentureBeat 和各大產品部落格。盛大的發布日。

**結果：**
- 5 萬次曝光 (impressions)
- 12 人註冊
- 2 位轉換

**失敗原因：**

過度優化媒體熱度，而非使用者價值。該功能尚未準備好進行自助服務。它需要教育、情境說明和專人引導。媒體能為你帶來目光。但沒有啟動 (activation) 的目光 = 虛榮。

**更好的做法：**

直接寄電子郵件給 50 位目標客戶。「我們建立了 [功能]，因為像你們這樣的團隊正深受 [問題] 所苦。想要早期存取權嗎？」親自引導他們完成設定。獲取回饋，持續反覆運算。

**結果：** 50 封郵件 → 15 封回覆（30% 回覆率）→ 8 次試用 → 4 位轉換（50% 試用轉付費）。

**啟示：**

早期客戶來自直接外展，而非媒體報導。媒體報導在後期才重要（A 輪融資公告、重大里程碑）。對於 0 到 1 的階段，它只會分散注意力。

### 2. 三層診斷（為何發布後會停滯）

**模式：**

你發布了產品。你有一定的知名度。但轉換很微弱。問題通常存在於以下三層之一，每一層都需要不同的干預措施。

**第一層：定位問題 (Positioning Problem)**

症狀：
- 訊息聽起來和競爭對手一模一樣
- 差異化需要解釋複雜的技術細節
- 買家認為你與其他替代方案可以互換
- 銷售對話常因比較問題而偏離主題

診斷：你是在「錯誤的戰線上打一場不對稱戰爭」— 在功能上與資金更雄厚的公司競爭。找出競爭對手聲稱具備獨特價值的地方。尋找一個他們無法輕易複製的定位。

修正：佔領一個你在結構上可以擁有的領域（而不僅僅是透過產品功能）。在投入產品資源之前，先透過外展訊息進行測試。

**第二層：體驗問題 (Experience Problem)**

症狀：
- 知名度高但啟動弱
- 使用者註冊了但未完成第一個工作流程
- 多個進入點導致決策癱瘓
- 文件以功能為中心，而非以成果為中心

診斷：沒有預設引導的靈活性是一種負擔，而非功能。使用者面臨「選擇的矛盾」— 選項太多，卻沒有足夠的指引帶領他們到達「啊哈時刻 (aha moment)」。

修正：識別 2-3 個能立即提供價值的「無庸置疑的使用案例」。將新手引導限制在這些特定的使用案例中。將進階功能放在掌握路徑之後。圍繞著「待辦任務 (jobs-to-be-done)」重寫說明內容，而非功能清單。

**第三層：對齊問題 (Alignment Problem)**

症狀：
- 團隊回報「沒有頻寬」來處理客戶
- 不同職能部門針對不同指標進行優化
- 每個想法的權重都一樣（沒有決策機制）
- 沒有清晰的北極星指標將活動與成果連結

診斷：「探索模式」— 即每項倡議都有相同優先級 — 當資源受限時會產生破壞性。

修正：定義一個單一且共享的北極星指標。將其作為每個決策的仲裁標準：「這是否有助於我們贏得客戶？」削減與此不符的活動。每週（而非每季）公開進度。

**如何使用此框架：**

當發布停滯時，在投入資源之前，先診斷是哪一層出了問題。當問題出在定位時，修正體驗會浪費工程時間。當問題出在內部對齊時，修正定位會浪費行銷支出。

### 3. 首批 10 位客戶框架

**原則：** 首批 10 位客戶不是為了營收，而是為了學習。

**你正在學習什麼：**
1. 產品是否真的解決了問題？
2. 啟動流程為何？（他們如何獲得價值？）
3. 會出現哪些反對意見？（價格、功能、整合？）
4. 真正的買家是誰？（職稱、角色、預算權限？）
5. 銷售週期多長？（天、週、月？）

**如何找到他們：**

**管道 1：個人網絡（前 2-3 位）**
- 「我正在建立 [X]，可以給我一些回饋嗎？」
- 將其轉換為付費客戶（不要免費提供 — 免費使用者給出的回饋與付費使用者不同）

**管道 2：直接外展（第 3-20 位客戶）**
- 建立包含 100 個目標帳號的清單
- 針對他們的特定痛點進行個人化處理
- 測試訊息變體 — 哪種切入點能獲得回覆？

**管道 3：天花板時刻鎖定 (Ceiling Moment Targeting)（最高意圖）**
- 最高意圖的潛在客戶是那些已經採用了類似解決方案但遇到限制的人
- 他們已投入時間學習某種工具，但遇到了天花板，且切換成本較低
- 圍繞著限制設計外展內容：「我們看到有些團隊在需要 [某功能] 時會感到 [現有工具] 不敷使用。這正是我們建立的原因。」
- 這些人的轉換率比冷啟動外展高出 3-5 倍，因為他們已經理解問題所在

**管道 4：社群（開發者產品）**
- 「建立了 [X] 來解決 [問題]，正在尋找早期使用者」
- 提供「白手套」式的新手引導
- 最適合使用者聚集在 Slack/Discord/論壇的產品

### 4. 2 週實驗循環

**模式：**

早期階段的速度比完美更重要。限制不在於你是否正確 — 而在於你能多快測試假設並進行反覆運算。

**如何執行：**

- 在開始前為每項測試設定清晰的成功標準
- 每次實驗僅測試一個變數（訊息、管道、定價、功能）
- 最多執行 2 週 — 如果到那時還沒看到訊號，之後也不會有了
- 如果奏效，在一週內分配 3 倍資源
- 如果無效，終止測試並進行下一項
- 無論結果如何，都記錄下你學到的東西

**教戰手冊規則：**

每項成功的實驗在規模化之前都必須成為一份教戰手冊 (playbook)。結構：目標 → 步驟 → 預期產出 → 指標 → 風險。如果一個不熟悉的人無法執行這份手冊，那就表示記錄得不夠詳細。

**為何這很重要：**

零星的勝利無法累積，系統化的實驗則可以。目標不是單一的發布 — 而是建立一個能快速測試假設的重複性機器。

**常見錯誤：**

測試前過度規劃。等待「完美」條件才發布。因為投入了情感精力而堅持失敗的實驗太久。憑藉 70% 的資訊就做出決策。

### 5. 夥伴導向的市場進入 (Partner-Led Market Entry)（當你缺乏分銷管道時）

**模式：**

與其僅透過直接銷售進入新市場，不如利用與成熟業者的合作關係來加速。

**如何執行：**

1. 識別目標細分市場中的領導者
2. 以客戶問題而非合作提案來接洽 — 「如果你的使用者可以存取 [功能] 會怎樣？」將重點從你的需求轉向他們的需求
3. 從小處著手：幫助他們解決一個特定的問題（窄整合，而非全面合作）
4. 在要求更廣泛的承諾之前，先進行 3-6 個月的試點計畫 (pilot) 以證明價值
5. 共同建立參考客戶 (reference customers) — 降低他們的風險
6. 利用他們的進入市場 (GTM) 策略：一旦整合完成，他們就會向自己的客群進行行銷

**超節點模式 (The Supernode Pattern)：**

將自己定位為其他工具自然連接的整合中心。你擁有其他平台需要的關鍵資料或工作流程。這會產生加乘效應 — 每個新夥伴都會讓你對下一個夥伴更有價值。

**類別排序：**

不要到處尋求合作。每季專攻 2-3 個類別：
1. 以真實的使用案例為導向：「我們的使用者每個月要求與 [夥伴] 整合 50 次」
2. 一旦你與頂尖業者合作，競爭對手也會感到壓力而想與你合作
3. 在一個類別中成功建立 2-3 個合作關係後，建立共同的客戶案例

**常見錯誤：**

在沒有清晰整合路徑的情況下啟動合作。期望夥伴在沒有支援的情況下推動知名度。將合作視為銷售管道而非平台擴展。

### 6. PMF 驗證檢查清單

**產品市場契合度 (PMF) 是指客戶拉著你前進，而非你推著他們。**

**留存率 (Retention)：**
- [ ] 40% 以上的第一週使用者在第四週回訪
- [ ] 使用量隨時間增加
- [ ] 客戶在沒有銷售推動的情況下續約

**有機成長：**
- [ ] 正在發生口碑推薦
- [ ] 客戶詢問「我可以加入我的團隊嗎？」
- [ ] 在沒有付費行銷的情況下有主動詢問 (inbound)

**銷售速度：**
- [ ] 銷售週期縮短
- [ ] 試用轉換率 > 30%
- [ ] 客戶說「我們現在就需要這個」

**質性指標：**
- [ ] 如果產品消失，>40% 的人會感到非常失望（Sean Ellis 測試）
- [ ] 客戶能清楚說明產品用途（清晰的使用案例）
- [ ] 客戶公開倡導產品

**如果你沒有達到這些指標，表示你還沒有達到 PMF。不要規模化行銷/銷售。**

## 決策樹

### 為何我們的發布停滯不前？

```
潛在客戶理解你是什麼嗎？
├─ 否 → 第一層：定位問題
│         修正：在更改產品之前測試新的訊息
└─ 是 → 繼續...
    │
    使用者註冊後會啟動嗎？
    ├─ 否 → 第二層：體驗問題
    │         修正：將新手引導限制在 2-3 個使用案例，引導至「啊哈時刻」
    └─ 是 → 繼續...
        │
        團隊是否在重要事項上達成一致？
        ├─ 否 → 第三層：對齊問題
        │         修正：單一北極星指標，每週公開進度，削減非核心事項
        └─ 是 → 繼續反覆運算，你走在正確的軌道上
```

### 媒體發布還是直接外展？

```
自助服務準備好了嗎？（使用者在 10 分鐘內能獲得價值）
├─ 否 → 僅限直接外展（媒體報導不會產生轉換）
└─ 是 → 你有超過 100 萬美元的融資要宣布嗎？
    ├─ 是 → 兩者並行（媒體負責知名度，外展負責轉換）
    └─ 否 → 先直接外展，稍後再進行媒體報導
```

## 常見錯誤

**1. 優化頭條而非啟動**
5 萬次曝光和 12 人註冊。媒體報導 ≠ 成長。

**2. 發布前沒有目標客戶清單**
隨機散播 (Spray-and-pray) 在 0 到 1 階段行不通。先建立 100 個帳號的清單。

**3. 沒有預設引導的靈活性**
給使用者所有選項會讓他們癱瘓。挑選 2-3 個無庸置疑的使用案例並強力引導。

**4. 免費提供產品**
免費使用者會給予客氣的回饋。付費使用者會給予誠實的回饋。

**5. 在學習之前就規模化**
首批 10 位客戶是為了學習，而非營收。記錄一切。

**6. 過度規劃，測試不足**
帶有清晰終止標準的 2 週實驗。快速行動，記錄所學。

**7. 診斷錯誤的層級**
當問題出在體驗時修正定位 = 浪費行銷。當問題出在定位時修正體驗 = 浪費工程。

## 快速參考

**三層診斷：**
第一層：定位（訊息聽起來像競爭對手）→ 測試新訊息
第二層：體驗（有知名度但無啟動）→ 引導至「啊哈時刻」
第三層：對齊（團隊分散）→ 單一北極星指標，每週公開進度

**首批 10 位客戶：**
個人網絡 (2-3) → 直接外展 (3-20) → 天花板時刻鎖定（最高意圖）→ 社群（開發者產品）

**2 週實驗循環：**
假設 → 成功標準 → 測試（最長 2 週）→ 終止或 3 倍資源 → 記錄教戰手冊

**PMF 訊號：**
40% 以上的第一週到第四週留存率 + 口碑 + 銷售週期縮短 + >40% 感到非常失望

**夥伴導向的進入：**
客戶問題優先 → 窄試點計畫 → 共同建立參考客戶 → 利用他們的 GTM 策略
