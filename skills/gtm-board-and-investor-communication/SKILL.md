# 董事會與投資人溝通

建構能建立信任並推動決策的董事會會議、投資人更新和高管溝通 — 而不是沒人看的幻燈片。

## 何時使用

**觸發情境：**
- 「我該如何準備我們的董事會會議？」
- 「投資人更新中應該包含哪些內容？」
- 「我們沒達到預期數字，該如何溝通？」
- 「董事會簡報結構」
- 「我們應該多久更新一次投資人資訊？」
- 「我們的董事會會議沒有產值」

**背景：**
- 種子輪至成長階段的公司
- 董事會會議的準備與後續追蹤
- 每月/每季的投資人更新
- 處理未達標情況與壞消息
- 將策略從董事會層層傳遞至整個組織

---

## 核心框架

### 1. 先講故事，再秀資料

**模式：**

大多數董事會會議都以大量的資料堆疊開始。一頁又一頁的指標，然後是 10 分鐘的問答時間，董事會成員試圖弄清楚這一切代表什麼。

請翻轉這個做法。應該由敘事主導，資料則是用來確認。

**如何運作：**

以你們目前所處的階段開場。不是「這是我們的 ARR」，而是「這是我們進入本季時的信念、我們學到了什麼，以及這將我們帶到了什麼樣的位置」。接著展示驗證這套敘事的資料。

**董事會會議結構：**

**會前閱讀資料 (Pre-Read)（會議前 48-72 小時寄出）**
- 財務儀表板 (ARR、消耗率、資金存續期、案源管道)
- 帶有健康指標 (綠/黃/紅) 的計分卡
- 涵蓋市場背景與策略更新的 2-3 頁敘事摘要
- 規則：如果內容可以閱讀，就不要現場簡報。

**會議過程（90-120 分鐘）**

1. **市場背景與會前資料提問** (15 分鐘)
   - 自上次會議以來市場有何變化？
   - 董事針對會前資料中不明確之處提問
   - 不再重複介紹資料內容

2. **執行長策略更新 — 「三件事」** (15 分鐘)
   - **什麼奏效** (2-3 個展現動能的領域)
   - **什麼不奏效** (1-2 個具體缺口，非模糊辭令)
   - **我們對此採取什麼行動** (具體的變更，而非承諾)
   - 這是敘事，不是數字

3. **決策項** (30-45 分鐘)
   - 1-3 個需要董事會輸入或批准的議題
   - 有備而來：「我們正在 X 與 Y 之間做抉擇，董事們有什麼想法？」
   - 帶著決策離開，而不是「我們下季再議」

4. **深入探討 (Deep Dive)** (20-30 分鐘)
   - 深度探索一個議題（每次會議輪換）
   - 邀請負責該項目的職能領導人參加
   - 範例：定價策略、競爭格局、組織設計、市場擴張

5. **結語** (10 分鐘)
   - 總結決策與行動負責人
   - 閉門會議 (Closed session)（僅限董事，不含管理層）
   - 執行長與董事會主席後續彙報 (Debrief)

**常見錯誤：**

- 在敘事之前先丟資料（董事會會在沒有情境的情況下迷失在數字中）
- 多個競爭的敘事（董事會無法綜合資訊 — 請挑選一條主線）
- 試圖深入涵蓋太多主題（導致所有決策都草草了事）
- 沒有深度探討（董事會變成了橡皮圖章）
- 會議中充斥著好消息（董事會不信任只分享成功的執行長）

### 2. 「三件事」敘事模型

**模式：**

業務過於複雜，無法用一個標題概括。「三件事」模型 — 什麼奏效、什麼不奏效、什麼正在改變 — 讓董事會在幾分鐘內掌握現況與軌跡。

**什麼奏效（2-3 個領域）：**
務必具體。不要說「銷售進展順利」，而要說「結案了首筆六位數的企業交易，驗證了商業動態」。量化動能：
- 新產品發布獲得採用
- 銷售動態趨於成熟（首個大客戶、可重複的動態正在成型）
- 社群/生態系統里程碑

**什麼不奏效（1-2 個領域）：**
同樣要具體。不要說「我們面臨了一些挑戰」，而要說「自助服務轉換微弱 — 知名度高但轉換率僅 0.1%」。董事會需要理解實際的問題，而不是委婉說法。

**我們對此採取什麼行動（針對每個「不奏效」）：**
說明具體的變更。哪些產品變更？哪些組織變更？時間表為何？投入了哪些資源？

**為何這有效：**

董事會成員閱讀大量的更新。三件事模型為他們提供了一個心智歸檔系統：動能（感覺良好）、風險（注意）、執行力（這個團隊能處理問題）。沒有它，董事會要麼過度關注單一不佳指標，要麼錯過埋在 40 頁簡報中的真實問題。

### 3. 進度是方向性的，而非絕對的

**模式：**

如果你目標是 5 萬卻達到了 10 萬使用者，或是目標 20 萬卻沒達到，這並不重要。情境才是一切。展示達成目標的進度，而不僅僅是數字。

**如何框架進度：**

- **按計畫進行 (On track)**：「目標 10 萬活躍開發者。目前 7 萬。所有倡議均按計畫執行。信心：高。」
- **有風險 (At risk)**：「目標 10 萬活躍開發者。目前 5 萬。獲取成本高於預期。正在測試新管道。調整後目標：8.5 萬。信心：中。」
- **超前 (Ahead)**：「目標 10 萬活躍開發者。目前 9.5 萬。超出獲取預測。新合作夥伴加速成長。信心：極高。」

**規則：**
1. 預先設定目標（每個指標都需要目標）
2. 以達成目標的百分比顯示進度，而非僅顯示絕對數字
3. 承認未達標，解釋轉向 (pivots)
4. 根據軌跡說明信心程度

**常見錯誤：**

未達標時更改目標。董事會會追蹤這一點。如果 Q1 目標是 10 萬，你只達到 6 萬，不要把 Q2 目標改成 7.5 萬並稱之為「修正後的指引」。請陳述未達標的原因，解釋原因，並展示恢復計畫。公信力的累積比指標的成長更快。

---

### 4. 四層指標階層

**模式：**

太多指標會讓董事會混亂，太少則會錯過重要訊號。將指標分為四層，且每季都不要更改。

**第一層：北極星指標 (1 個指標)**
最能代表公司健康狀況的單一指標。
- 範例：ARR、活躍使用者、完成的任務
- 展示趨勢線，而不僅僅是現狀
- 如果董事會忘記所有其他內容，這是他們唯一應該記住的指標

**第二層：驅動指標 (3-5 個指標)**
預測北極星指標的領先指標。
- 使用者獲取率
- 留存曲線（上個月活躍的使用者中有多少本月仍活躍？）
- 功能採用率（使用關鍵功能的使用者百分比）
- DAU/MAU 比率
- 案源管道覆蓋率
- 追蹤原因：在影響營收前發出早期預警

**第三層：健康指標 (3-5 個指標)**
團隊與維運健康狀況的訊號。
- 消耗率與資金存續期
- 實際人數 vs 計畫人數
- 工程速率 / 出貨節奏
- NPS 或客戶滿意度
- 追蹤原因：成長的產能限制

**第四層：紅旗指標 (2-3 個閾值)**
如果跨過某個閾值，就需要立即採取行動的指標。
- 流失率高於 X%
- CAC 高於 $Y
- 留存率下降大於 Z%
- 追蹤原因：觸發策略對話，而不僅僅是監控

**規範守則：**

1. **每季使用相同指標。** 建立 4-6 季的追蹤紀錄。可以加入新指標，但絕不要刪除舊指標。
2. **每個指標一句情境說明。** 這個指標告訴我們什麼？是好是壞？影響為何？
3. **務必包含實際值 vs 計畫值。** 展示指標時務必對比計畫。其中的差距才是對話的重點。
4. **必要時進行拆解。** 總使用量會隱藏細分市場。按客戶類型、細分市場、產品線拆解 — 任何平均值會遮蓋現實的地方。
5. **使用健康指示燈。** 以紅黃綠狀態讓董事會快速評估。讓董事會能掃視儀表板並聚焦在黃色與紅色區域。

**常見錯誤：**

根據哪項指標看起來不錯來決定本季要展示什麼。董事會會立即察覺。這是失去數字公信力最快的方法。每次都展示相同的儀表板 — 當某個指標表現不佳時，那正是需要討論的時候。

### 5. 在董事會詢問前先告知壞消息

**模式：**

當壞消息被延遲或美化時，董事會的信心會受到侵蝕。他們可以處理壞消息，但不能接受意外。而且他們討厭決策緩慢。

**預通報 (Pre-Brief) 模式：**

在壞消息傳到全體董事會之前，在會議前 48-72 小時先與領頭董事 (lead director) 溝通。

- **對象：** 董事會主席或最有經驗的董事
- **內容：** 問題所在、你的評估、你的計畫
- **原因：** 領頭董事可以協助向全體董事會說明情況，避免突發衝擊，且他們可能見過類似模式。

接著寄發簡短郵件給全體董事：

「在我們下次會議前，想先提醒一件事。[具體問題]。這是我們目前所知、尚不清楚的部分，以及我們正在採取的行動。我們會在董事會會議上提供完整更新。」

**壞消息框架：**

對於每一則壞消息，包含四個元素：

1. **發生了什麼** (具體，一句話 — 不要防衛)
   - 「我們未達到 Q1 獲得 10 個新企業客戶的目標；實際達成 7 個」

2. **為何發生** (根本原因，而非藉口)
   - 「由於我們未及早診斷出的產品缺口，導致客戶評估時程滑落」
   - 承擔責任：「我們誤判了需求訊號」，而非「市場發生了變化」

3. **影響為何** (誠實說明嚴重性)
   - 「影響 Q1 ARR 約 5 萬美元，使我們低於原先的指引」

4. **我們對此採取什麼行動** (具體行動、負責人、時間表)
   - 「更改了銷售動態，預先進行產品就緒性資格審核；將 Q2 目標修訂為 15 個 (包含 Q1 積壓的 7 個)」

**「不該」如何溝通未達標情況：**
- 「我們沒達到 Q1 目標，但案源管道很強」
- 「外部因素減緩了交易速度，但我們對 Q2 很有信心」
- 「我們遇到了一些實作挑戰，但現在都解決了」

這些說法都太模糊、太具防衛性。既沒解釋根本原因，也沒有計畫。

**持續追蹤模式：**

後續的每一次董事會更新都應引用之前提出的問題：「上季我提醒了 [問題]。這是目前的更新：[進度]。狀態：[已解決 / 進行中 / 升級中]。」

這能建立持續追蹤的紀錄。董事會會記得誰在追蹤自己的問題。

**常見錯誤：**

將壞消息夾在好消息中間，希望沒人注意到。董事會成員會立即識破。這對信任的損害比壞消息本身更嚴重。

### 6. 每月投資人更新

**模式：**

大多數創辦人頂多每季才寄一次投資人更新。最好的創辦人則是每月都寄 — 即使情況不妙。特別是情況不妙時。

**為何要每月更新：**
- 無需安排通話就能讓投資人參與其中
- 當你需要幫助（引薦、建議、橋接融資）時，投資人已有情境背景
- 一致性展現了維運規範；不規律的更新則顯得混亂

**格式：**

**郵件主旨：** [公司名稱] — [月份] 更新

**開場（1 個段落）：**
一個大標題。簡短情境。

**TL;DR（3 個重點）：**
- 一個數字（目前最重要的指標）
- 一個勝利（具體 — 客戶名稱、交易規模、里程碑）
- 一個挑戰（以及你正採取的行動）

**關鍵指標（每個月相同的表格）：**

| 指標 | 本月 | 上月 | 3 個月平均 | vs 計畫 |
|--------|-----------|------------|----------|---------|
| ARR | | | | |
| MRR 成長 | | | | |
| 消耗率 | | | | |
| 資金存續期 (月) | | | | |
| 案源管道 | | | | |
| 客戶數 | | | | |

不要包含與上次更新相比沒有變化的指標 — 專注於新動態。

**關鍵更新（2-3 個重點）：**
- 重大人事聘僱、產品發布、夥伴/客戶贏單
- 策略性轉向或路線修正
- 內容要具體到足以引起關注

**請求協助 (Asks)（1-2 個重點）：**
- 你需要什麼幫助？引薦、徵才線索、策略建議
- 務必具體：「尋求引薦至 [公司類型] 的工程副總裁」，而非「尋求工程人才」

**節奏規範：**
- 每月同一天寄送。每月第一個週一或最後一個週五 — 選一個日期並永不缺席
- 重大新聞發生後的一週內寄送（不要等到下次例行更新）
- 在積極募資期間：每週更新給潛在投資人，每月更新給既有投資人

**常見錯誤：**

只有在情況好時才寄更新。六個月的好消息接著一陣沉默，會讓投資人清楚知道你有些事沒說。

### 7. 消耗預算關乎規範；營收關乎動能

**模式：**

向董事會進行的財務溝通有兩個獨立的故事。不要混為一談。

**消耗預算溝通：**
- 月平均消耗、资金存續月數、YoY 改善情況
- 效率提升（你削減了什麼、優化了什麼？）
- 資本計畫：已獲得的資金、目標資金、時間表

消耗預算的故事是：「我們是規範嚴明的經營者，能讓資本發揮更大效用。」

**營收溝通：**
- MoM 成長、CAC、LTV、回本週期 (payback period)
- 客戶細分（自助服務 vs 企業，佔比為何？）
- 變現計畫與時間表

營收的故事是：「我們擁有了動能，並且知道如何擴大它。」

**連結兩者：**
最好的財務溝通能展示：「我們在減少消耗的同時加速成長。」這是營運槓桿 (operating leverage) 的證據，也是董事會最想聽到的內容。

**常見錯誤：**
將消耗與成長混為一談。如果董事會聽到「我們成長很快」的同時消耗率也在攀升，他們會認為你在花錢買成長。如果董事會聽到「我們削減了 40% 的消耗」但成長停滯，他們會認為你正處於生存模式。請分別講述這兩個故事，然後再將它們連結起來。

---

### 8. 將策略從董事會傳遞至組織

**模式：**

在董事會會議室決定的策略，如果團隊不知道，那就沒有意義。建立一個系統性的溝通傳遞鏈。

**四個層級：**

**第一級：執行團隊（每週）**
- 策略優先事項與新興風險
- 需要對齊的關鍵決策

**第二級：擴大領導層（雙週）**
- 策略如何影響各職能部門
- 關鍵優先事項以及如何衡量成功

**第三級：全體員工會議 (All-Hands)（每月，60 分鐘）**
- 領導層更新 (15 分鐘)：我們目前的進度、有什麼變化
- 慶祝勝利 (10 分鐘)：客戶、產品發布、人才
- 深入探討 (20 分鐘)：詳細說明一項策略倡議
- Q&A (15 分鐘)：真實的問題，真實的回答

**第四級：團隊特定（持續進行）**
- 團隊目標如何連結至公司策略
- 對於該團隊具體而言，成功代表什麼

**測試：**

隨機找一名員工。他們能解釋公司的策略嗎？如果不行，傳遞鏈就斷了。僅存在於高管腦袋裡的策略不是策略 — 而是秘密。

**將 OKR 連結至傳遞鏈：**
- 公司 OKR 來自策略
- 職能部門 OKR 對齊公司
- 團隊 OKR 對齊職能部門
- 個人理解自己如何做出貢獻

**常見錯誤：**

訊息不一致。如果不同的領導人對公司走向有不同的說法，組織就會為了衝突的目標而優化。請寫下策略。一頁紙。讓每個人都使用相同的詞彙。

## 決策樹

### 董事會會議 vs 投資人更新

```
這是正式的董事會會議嗎？
├─ 是 → 完整結構：會前資料 + 三件事敘事 + 決策 + 深入探討
└─ 否 → 繼續...
    │
    這是重大事件（募資結案、嚴重未達標、關鍵聘僱）嗎？
    ├─ 是 → 立即進行臨時更新（不要等待預定時程）
    └─ 否 → 每月投資人更新格式
```

### 如何框架一項指標

```
達到目標了嗎？
├─ 是 → 陳述事實，繼續前進。不要過度慶祝。
└─ 否 → 繼續...
    │
    你了解原因嗎？
    ├─ 是 → 根本原因 + 計畫 + 修訂後的時間表
    └─ 否 → 說「我們正在診斷中」+ 你正在做什麼來找出原因
        │
        未達標的情況是否重大（影響存續期、策略或董事會信心）？
        ├─ 是 → 在董事會會議前預通報領頭董事
        └─ 否 → 包含在「三件事」敘事的「什麼不奏效」中
```

## 常見錯誤

**1. 董事會會議變成進度報告**
董事會成員識字。寄出会前資料。會議應用於決策和討論，而非資料導覽。

**2. 每季更改指標**
指標的一致性可以建立信任。當你把表現不佳的指標換成表現良好的指標時，你就告訴了董事會你的數字不可信。

**3. 沒有明確的請求**
每次投資人更新、每次董事會會議都應該有一個請求。引薦、建議、批准、耐心。投資人想要幫忙 — 如果你從不開口，他們會失去參與感。

**4. 辯解而非描述**
當你花了 10 分鐘解釋為何未達標時，董事會聽到的只有防衛心。請陳述未達標的情況、原因以及計畫。然後繼續。

**5. 策略僅留在董事會會議室**
如果你的團隊無法解釋策略，表示你沒有溝通好。層層傳遞，否則它就不存在。

**6. 董事會會議中的驚嚇**
壞消息絕不應該在董事會會議上第一次出現。預通報領頭董事，寄發預警郵件。讓董事會在會議前先消化。

## 快速參考

**董事會會議流程：**
會前資料（48-72 小時前）→ 提問 (15 分鐘) → 三件事敘事 (15 分鐘) → 決策項 (30-45 分鐘) → 深入探討 (20-30 分鐘) → 閉門會議 (10 分鐘)

**三件事模型：**
什麼奏效 + 什麼不奏效 + 我們對此採取什麼行動

**指標階層：**
北極星 (1) → 驅動指標 (3-5) → 健康指標 (3-5) → 紅旗指標 (2-3 個閾值)

**壞消息規程：**
預通報領頭董事（48-72 小時）→ 向全體董事寄發預警郵件 → 發生了什麼 → 原因 → 影響 → 計畫 → 在後續每次會議中追蹤

**每月投資人更新：**
TL;DR (3 個重點) → 相同的指標表格 → 關鍵更新 (2-3) → 具體請求 (1-2)

**溝通傳遞鏈：**
執行團隊（每週）→ 領導層（雙週）→ 全體員工（每月）→ 團隊（持續進行）
