# 重構方法以降低認知複雜度 (Refactor Method to Reduce Cognitive Complexity)

## 目標 (Objective)
重構方法 `${input:methodName}`，透過將邏輯擷取到聚焦的輔助方法中，將其認知複雜度降低至 `${input:complexityThreshold}` 或以下。

## 說明 (Instructions)

1. **分析目前的方法** 以識別認知複雜度的來源：
   - 巢狀條件語句
   - 多個 if-else 或 switch 鏈
   - 重複的程式碼區塊
   - 包含條件的多個迴圈
   - 複雜的布林運算式

2. **識別擷取機會**：
   - 可以擷取到獨立方法中的驗證邏輯
   - 重複出現的特定型別或特定情況的處理
   - 複雜的轉換或計算
   - 出現多次的通用模式

3. **擷取聚焦的輔助方法**：
   - 每個輔助方法應具有單一且明確的職責
   - 將驗證邏輯擷取為獨立的 `Validate*` 方法
   - 將特定型別的邏輯擷取為處理常式 (handler) 方法
   - 為常用作業建立公用程式方法
   - 使用適當的存取層級（static, private, async）

4. **簡化主要方法**：
   - 減少巢狀深度
   - 用較小的協調呼叫取代龐大的 if-else 鏈
   - 在適當之處使用 switch 語句以實現更整潔的派送 (dispatch)
   - 確保主要方法讀起來像是一個高階流程

5. **保留功能**：
   - 維持相同的輸入/輸出行為
   - 保留所有驗證與錯誤處理
   - 保留例外狀況型別與錯誤訊息
   - 確保所有參數都正確傳遞給輔助方法

6. **最佳實作**：
   - 當輔助方法不需要執行個體狀態時，將其設為靜態 (static)
   - 儘早使用 null 檢查與防護子句 (guard clauses)
   - 避免建立不必要的區域變數
   - 考慮對多個傳回值使用元組 (tuples)
   - 將相關的輔助方法群組在一起

## 實作方法 (Implementation Approach)

- 在重構主要流程前先擷取輔助方法
- 進行增量測試以確保沒有功能倒退 (regressions)
- 使用能描述所擷取職責的具意義名稱
- 將擷取的方法放在靠近其使用位置的地方
- 考慮將重複的程式碼模式改為泛型方法

## 結果 (Result)

重構後的方法應：
- 認知複雜度降低至目標閾值 `${input:complexityThreshold}` 或以下
- 具備更好的可讀性與可維護性
- 具有明確的關注點分離 (separation of concerns)
- 更易於測試與偵錯
- 保留所有原始功能

## 測試與驗證 (Testing and Validation)

**關鍵事項：完成重構後，您必須：**

1. **執行所有現有測試**，包括與重構方法及其周邊功能相關的測試
2. **強制性：明確驗證測試結果顯示 "failed=0"**
   - **絕不要假設測試已通過** — 務必檢查實際的測試輸出
   - 搜尋包含成功/失敗計數的摘要行（例如：「passed=X failed=Y」）
   - **如果摘要顯示 "failed=0" 以外的任何數字，則代表測試失敗**
   - 如果測試輸出在檔案中，請閱讀整個檔案以找到並驗證失敗計數
   - 執行測試並不等於驗證測試已通過
   - **在您明確確認零失敗之前，請勿繼續**
3. **如果任何測試失敗 (failed > 0)：**
   - 清楚說明有多少個測試失敗
   - 分析每個失敗以了解哪些功能損壞了
   - 常見原因：null 處理、空集合檢查、條件邏輯錯誤
   - 識別重構程式碼中的根本原因
   - 修正重構後的程式碼以恢復原始行為
   - 重新執行測試並驗證輸出中的 "failed=0"
   - 重複此過程，直到所有測試通過 (failed=0)
4. **驗證編譯** — 確保沒有編譯錯誤
5. **檢查認知複雜度** — 確認該計量指標位於或低於目標閾值 `${input:complexityThreshold}`

## 確認檢查清單 (Confirmation Checklist)
- [ ] 程式碼編譯無誤
- [ ] **測試結果明確標示為 "failed=0"**（透過閱讀輸出內容驗證）
- [ ] 所有測試失敗皆已分析並修正（若有發生）
- [ ] 認知複雜度位於或低於目標閾值 `${input:complexityThreshold}`
- [ ] 所有原始功能皆已保留
- [ ] 程式碼遵循專案慣例與標準
