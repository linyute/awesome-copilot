# HTML CSS 樣式色彩指南

在更新或建立用於瀏覽器渲染的 HTML/CSS 樣式時，請遵循這些指南。色彩名稱
代表其各自色調範圍的完整光譜（例如，「藍色」包括海軍藍、天藍色等）。

## 色彩定義

- **暖色調**: 橘色、紅色和黃色
- **冷色調**: 藍色、綠色和紫色
- **中性色**: 灰色和灰階變化
- **二元色**: 黑色和白色
- **60-30-10 規則**
  - **主色**: 使用 60% 的時間（*冷色或淺色*）
  - **輔助色**: 使用 30% 的時間（*冷色或淺色*）
  - **強調色**: 使用 10% 的時間（*互補暖色*）

## 色彩使用指南

透過將 **60-30-10 規則** 應用於背景、按鈕、卡片等平面設計元件，
來平衡所使用的色彩。

### 背景顏色

**切勿使用：**

- 紫色或洋紅色
- 紅色、橘色或黃色
- 粉紅色
- 任何暖色調

**推薦使用：**

- 白色或乳白色
- 淺冷色（例如：淺藍色、淺綠色）
- 細微的中性色調
- 色彩偏移極小的淺色漸層

### 文字顏色

**切勿使用：**

- 黃色（對比度和可讀性差）
- 粉紅色
- 淺色背景上的純白色或淺色文字
- 深色背景上的純黑色或深色文字

**推薦使用：**

- 深中性色（例如：#1f2328, #24292f）
- 近黑色變化（#000000 到 #333333）
  - 確保背景為淺色
- 深灰色（#4d4d4d, #6c757d）
- 用於可存取性的高對比度組合
- 近白色變化（#ffffff 到 #f0f2f3）
  - 確保背景為深色

### 應避免的顏色

除非設計規範或使用者要求明確規定，否則請避免：

- 明亮的紫色和洋紅色
- 明亮的粉紅色和霓虹色
- 高飽和度的暖色調
- 對比度低的顏色（未通過 WCAG 可存取性標準）

### 應謹慎使用的顏色

**暖色調**（紅色、橘色、黃色）：

- 預留給關鍵警報、警告或錯誤訊息
- 僅在傳達緊急性或重要性時使用
- 限制在小的強調區域，而非大區塊
- 在使用暖色調之前，請考慮圖示或粗體文字等替代方案

## 漸層

使用細微的色彩過渡應用漸層，以保持專業的美感。

### 最佳做法

- 保持最小的色彩偏移（例如：#E6F2FF 到 #F5F7FA）
- 在同一色系內使用漸層
- 避免在單一漸層中組合暖色和冷色
- 對於背景，偏好線性漸層而非放射狀漸層

### 適用的使用場景

- 背景容器和區塊
- 按鈕懸停狀態和互動元件
- 落地陰影和深度效果
- 頁首和導覽列
- 卡片元件和面板
