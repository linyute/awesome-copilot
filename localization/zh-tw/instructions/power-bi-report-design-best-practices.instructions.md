# Power BI 報表設計和視覺化最佳實踐

## 概述
本文件根據 Microsoft 的官方指導方針和使用者體驗最佳實踐，提供了設計有效、可存取和高性能 Power BI 報表和儀表板的全面說明。

## 基本設計原則

### 1. 資訊架構
**視覺層次結構** - 按重要性組織資訊：
- **主要**：關鍵指標、KPI、最關鍵的洞察（左上角、標題區域）
- **次要**：支援細節、趨勢、比較（主體）
- **第三級**：篩選器、控制項、導航元素（側邊欄、頁腳）

**內容結構**：
```
報表頁面佈局：
┌─────────────────────────────────────┐
│ 標題：標題、KPI、關鍵指標           │
├─────────────────────────────────────┤
│ 主要內容區域                        │
│ ┌─────────────┐ ┌─────────────────┐ │
│ │  主要       │ │  支援           │ │
│ │  視覺效果   │ │  視覺效果       │ │
│ └─────────────┘ └─────────────────┘ │
├─────────────────────────────────────┤
│ 頁腳：篩選器、導航、備註            │
└─────────────────────────────────────┘
```

### 2. 使用者體驗原則
**清晰度**：每個元素都應具有明確的目的和意義
**一致性**：在所有報表中使用一致的樣式、顏色和互動
**上下文**：提供足夠的上下文供使用者正確解釋資料
**行動**：引導使用者獲得可操作的洞察和決策

## 圖表類型選擇指南

### 1. 比較視覺效果
```
長條圖/柱狀圖：
✅ 比較類別或實體
✅ 按值排名項目
✅ 顯示離散時間段內的變化
✅ 當類別名稱很長時（使用水平長條）

最佳實踐：
- 軸從零開始以進行準確比較
- 按值排序類別以進行排名
- 在類別組內使用一致的顏色
- 限制為 7-10 個類別以提高可讀性

範例使用案例：
- 按產品類別劃分的銷售額
- 按地區劃分的收入  
- 按部門劃分的員工數量
- 按服務類型劃分的客戶滿意度
```

```
折線圖：
✅ 顯示連續時間段內的趨勢
✅ 比較多個指標隨時間的變化
✅ 識別模式、季節性、週期
✅ 預測和預計場景

最佳實踐：
- 使用一致的時間間隔
- 顯示絕對值時 Y 軸從零開始
- 對多個系列使用不同的線條樣式
- 對稀疏資料包含資料點標記

範例使用案例：
- 月度銷售趨勢
- 網站流量隨時間的變化
- 股票價格走勢
- 性能指標追蹤
```

### 2. 構成視覺效果
```
圓餅圖/環圈圖：
✅ 整體與部分關係
✅ 最多 5-7 個類別
✅ 當百分比比絕對值更重要時
✅ 簡單的構成場景

限制：
❌ 難以比較大小相似的區段
❌ 不適用於許多類別
❌ 難以顯示隨時間的變化

替代方案：使用堆疊長條圖以提高可讀性

範例使用案例：
- 按競爭對手劃分的市場份額
- 按類別劃分的預算分配
- 按類型劃分的客戶細分
```

```
堆疊圖：
✅ 同時顯示構成和總計
✅ 比較跨類別的構成
✅ 主要類別中的多個子類別
✅ 當您需要部分和整體視角時

類型：
- 100% 堆疊：專注於比例
- 常規堆疊：顯示絕對值和相對值
- 群集：並排比較子類別
- 當您需要部分和整體視角時

範例使用案例：
- 按產品類別和地區劃分的銷售額
- 按服務類型隨時間劃分的收入明細
- 按部門和級別劃分的員工分佈
```

散佈圖：
✅ 兩個連續變數之間的相關性
✅ 異常值識別

- 風險與回報分析

```

```
熱圖：

配置：
- 使用對色盲友好的顏色比例
- 空間允許時包含資料標籤
- 提供清晰的帶有值範圍的圖例

範例使用案例：

```
何時啟用交叉篩選：

- 為選擇提供清晰的視覺回饋

- 記錄互動設計決策
```

## 視覺設計和格式

### 1. 顏色策略
```
顏色使用層次結構：

語義顏色（一致的意義）：
- 綠色：積極表現、增長、成功、達標
- 紅色：消極表現、下降、警報、超預算
- 藍色：中性資訊、基本指標、企業品牌
- 橙色：警告、需要注意、中度關注
- 灰色：非活動、禁用或參考資訊

品牌整合：
✅ 一致使用企業調色板
✅ 維護可存取性標準（最小對比度 4.5:1）
✅ 考慮色盲可存取性（8% 的男性受影響）
✅ 在不同上下文（投影機、行動裝置、列印）中測試顏色

顏色應用：
主色：用於關鍵指標和亮點的主要品牌顏色
輔色：用於類別的輔助品牌顏色
強調色：用於警報和呼叫的高對比度顏色
中性色：背景、文字、邊框、非活動狀態
```

```
無障礙優先的顏色設計：

色盲考量：
✅ 不要僅僅依靠顏色來傳達資訊
✅ 使用圖案、形狀或文字標籤作為替代方案
✅ 使用色盲模擬工具進行測試
✅ 使用高對比度顏色組合
✅ 提供替代視覺提示（圖示、圖案）

實施：
- 紅綠組合：添加藍色或使用不同的飽和度
- 使用 Colour Oracle 等工具進行測試
- 在顏色是主要區分因素的地方包含資料標籤
- 考慮報表的灰度版本以供列印
```

### 2. 字體排版和可讀性
```
字體層次結構：

報表標題：18-24pt，粗體，企業字體或清晰的無襯線字體
頁面標題：16-20pt，半粗體，與報表標題一致
區段標題：14-16pt，半粗體，用於內容分組
視覺效果標題：12-14pt，半粗體，描述性和簡潔
正文：10-12pt，常規，用於文字方塊和描述
資料標籤：9-11pt，常規，清晰且不重疊
標題/圖例：8-10pt，常規，補充資訊

可讀性指南：
✅ 資料視覺化最小字體大小為 10pt
✅ 文字和背景之間的高對比度
✅ 整個報表中使用一致的字體系列（最多 2 個系列）
✅ 文字元素周圍有足夠的空白
✅ 文字左對齊以提高可讀性（居中標題除外）
```

```
內容寫作最佳實踐：

標題和標籤：
✅ 清晰、描述性和面向行動
✅ 避免行話和技術縮寫
✅ 始終使用一致的術語
✅ 相關時包含時間段和上下文

範例：
好：「按產品類別劃分的月度銷售收入」
差：「銷售資料」

好：「客戶滿意度分數（1-10 分）」
差：「CSAT」

資料敘事：
✅ 使用副標題提供上下文
✅ 必要時包含方法論說明
✅ 解釋不尋常的資料點或異常值
✅ 在文字方塊中提供可操作的洞察
```

### 3. 佈局和間距
```
視覺間距：
網格系統：使用一致的間距倍數（8px、16px、24px）
填充：內容區域周圍有足夠的空白
邊距：視覺元素之間的一致邊距
對齊：使用對齊指南以獲得專業外觀

視覺分組：
相關內容：將相關視覺效果與一致的間距分組
分離：使用空白將不相關的內容區域分開
視覺層次結構：使用大小、顏色和間距來指示重要性
平衡：在頁面中均勻分佈視覺權重
```

### 1. 行動佈局策略
```
行動優先設計原則：
✅ 縱向佈局作為主要佈局

文字大小：增加字體大小以提高行動可讀性

測試策略：

✅ 檢查長時間使用期間的電池使用情況

```

## 應避免的常見反模式

### 1. 設計反模式
```
❌ 圖表垃圾：
- 不必要的 3D 效果
- 過度動畫
- 不增加價值的裝飾元素
- 過於複雜的視覺效果

❌ 資訊過載：
- 單頁上過多的視覺效果
- 雜亂的佈局，空白不足
- 多個相互競爭的焦點
- 壓倒性的顏色使用

❌ 糟糕的顏色選擇：
- 沒有替代方案的紅綠組合
- 低對比度
- 不一致的顏色含義
- 過度使用鮮豔或飽和的顏色
```

### 2. 互動反模式
```
❌ 導航混亂：
- 不一致的導航模式
- 隱藏或不明確的導航選項
- 損壞或意外的鑽取行為
- 循環導航迴圈

❌ 性能問題：
- 過多的視覺效果導致載入緩慢
- 低效的交叉篩選
- 不必要的即時重新整理
- 沒有適當篩選的大型資料集

❌ 行動裝置不友好：
- 小觸控目標
- 水平滾動要求
- 行動裝置上不可讀的文字
- 無法運作的行動互動
```

請記住：始終根據您的特定使用者和使用案例進行設計。及早並經常使用真實使用者和真實資料進行測試，以確保您的報表有效地傳達洞察並實現資料驅動的決策。
