# Issue 範本

複製並自訂這些範本以供 issue 內容使用。

## 錯誤回報 (Bug Report) 範本

```markdown
## 說明
[清晰的錯誤說明]

## 重現步驟
1. [第一步]
2. [第二步]
3. [依此類推...]

## 預期行為
[應該發生的情況]

## 實際行為
[實際發生的情況]

## 環境
- 瀏覽器：[例如：Chrome 120]
- 作業系統：[例如：macOS 14.0]
- 版本：[例如：v1.2.3]

## 螢幕截圖/日誌
[如果適用]

## 其他背景資訊
[任何其他相關資訊]
```

## 功能要求 (Feature Request) 範本

```markdown
## 摘要
[一行功能說明]

## 動機
[為什麼需要此功能？它解決了什麼問題？]

## 建議解決方案
[此功能應如何運作？]

## 驗收標準
- [ ] [標準 1]
- [ ] [標準 2]
- [ ] [標準 3]

## 曾考慮的替代方案
[考慮過的其他方法以及未被採用的原因]

## 其他背景資訊
[模型圖、範例或相關 issue]
```

## 任務 (Task) 範本

```markdown
## 目標
[需要完成的事項]

## 詳情
[工作的詳細說明]

## 檢查清單
- [ ] [子任務 1]
- [ ] [子任務 2]
- [ ] [子任務 3]

## 相依性
[任何阻礙因素或相關工作]

## 備註
[額外背景資訊或考量事項]
```

## 最小化範本

適用於簡單的 issue：

```markdown
## 說明
[內容與原因]

## 任務
- [ ] [任務 1]
- [ ] [任務 2]
```
