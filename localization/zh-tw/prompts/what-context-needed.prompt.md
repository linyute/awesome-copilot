# 你需要什麼上下文？

在回答我的問題之前，告訴我你需要檢視哪些檔案。

## 我的問題

{{question}}

## 指引

1. 根據我的問題，列出你需要檢查的檔案
2. 說明每個檔案為何相關
3. 註記你在本次對話中已經看過的任何檔案
4. 識別你不確定的部分

## 輸出格式

```markdown
## 我需要的檔案

### 必須檢視 (準確回答所需)
- `path/to/file.ts` — [為什麼需要]

### 建議檢視 (有助於完整回答)
- `path/to/file.ts` — [為什麼有幫助]

### 已擁有
- `path/to/file.ts` — [來自稍早的對話]

### 不確定性
- [在沒看到程式碼的情況下不確定的部分]
```

在我提供這些檔案後，我會再次詢問我的問題。
