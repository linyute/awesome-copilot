# Copilot 聊天模式和提示的 AI 模型推薦

## 任務

分析 `.agent.md` 或 `.prompt.md` 檔案以理解其目的、複雜度及所需功能，然後從 GitHub Copilot 的可用選項中推薦最適合的 AI 模型。根據任務特性、模型優勢、成本效益和效能權衡提供理由。

## 範圍與前置條件

- **輸入**: `.agent.md` 或 `.prompt.md` 檔案路徑
- **可用模型**: GPT-4.1, GPT-5, GPT-5 mini, GPT-5 Codex, Claude Sonnet 3.5, Claude Sonnet 4, Claude Sonnet 4.5, Claude Opus 4.1, Gemini 2.5 Pro, Gemini 2.0 Flash, Grok Code Fast 1, o3, o4-mini (附帶棄用日期)
- **模型自動選擇**: 可用於 VS Code (2025 年 9 月+) - 從 GPT-4.1, GPT-5 mini, GPT-5, Claude Sonnet 3.5, Claude Sonnet 4.5 中選擇 (排除高級乘數 > 1)
- **背景資訊**: GitHub Copilot 訂閱層級 (免費版: 2K 完成次數 + 每月 50 次聊天，僅限 0x 模型; 專業版: 無限制 0x + 每月 1000 次高級; 專業版+: 無限制 0x + 每月 5000 次高級)

## 輸入

必填:

- `${input:filePath:Path to .agent.md or .prompt.md file}` - 絕對或工作區相對的檔案路徑以供分析

選填:

- `${input:subscriptionTier:Pro}` - 使用者的 Copilot 訂閱層級 (免費版, 專業版, 專業版+) - 預設為專業版
- `${input:priorityFactor:Balanced}` - 優化優先級 (速度, 成本, 品質, 平衡) - 預設為平衡

## 工作流程

### 1. 檔案分析階段

**讀取並解析檔案**:

- 讀取目標 `.agent.md` 或 `.prompt.md` 檔案
- 提取前置內容 (描述、模式、工具、如果指定模型)
- 分析主體內容以識別:
  - 任務複雜度 (簡單/中等/複雜/進階)
  - 所需推理深度 (基本/中級/進階/專家)
  - 程式碼生成需求 (最少/中等/廣泛)
  - 多輪對話需求
  - 上下文視窗需求 (小/中/大)
  - 專業功能 (圖像分析、長上下文、即時資料)

**分類任務類型**:

根據內容分析識別主要任務類別:

1. **簡單重複任務**:

   - 模式: 格式化、簡單重構、添加註釋/文件字串、基本 CRUD
   - 特性: 直觀邏輯、最少上下文、偏好快速執行
   - 關鍵字: 格式化、註釋、簡單、基本、添加文件字串、重命名、移動

2. **程式碼生成與實作**:

   - 模式: 編寫函式/類別、實作功能、API 端點、測試
   - 特性: 中等複雜度、領域知識、慣用語法程式碼
   - 關鍵字: 實作、建立、生成、編寫、建構、鷹架

3. **複雜重構與架構**:

   - 模式: 系統設計、架構審查、大規模重構、效能優化
   - 特性: 深度推理、多個元件、權衡分析
   - 關鍵字: 架構、重構、優化、設計、擴展、審查架構

4. **偵錯與問題解決**:

   - 模式: 錯誤修復、錯誤分析、系統性故障排除、根本原因分析
   - 特性: 逐步推理、偵錯上下文、驗證需求
   - 關鍵字: 偵錯、修復、故障排除、診斷、錯誤、調查

5. **規劃與研究**:

   - 模式: 功能規劃、研究、文件分析、ADR 建立
   - 特性: 只讀、上下文收集、決策支援
   - 關鍵字: 規劃、研究、分析、調查、文件、評估

6. **程式碼審查與品質分析**:

   - 模式: 安全分析、效能審查、最佳實踐驗證、合規性檢查
   - 特性: 批判性思維、模式識別、領域專業知識
   - 關鍵字: 審查、分析、安全、效能、合規、驗證

7. **專業領域任務**:

   - 模式: Django/框架特定、無障礙 (WCAG)、測試 (TDD)、API 設計
   - 特性: 深度領域知識、框架慣例、標準合規
   - 關鍵字: django、無障礙、wcag、rest、api、測試、tdd

8. **進階推理與多步驟工作流程**:
   - 模式: 演算法優化、複雜資料轉換、多階段工作流程
   - 特性: 進階推理、數學/演算法思維、序列邏輯
   - 關鍵字: 演算法、優化、轉換、序列、推理、計算

**提取功能需求**:

根據前置內容和主體指令中的 `tools`:

- **只讀工具** (搜尋、擷取、用法、githubRepo): 較低複雜度，適合更快的模型
- **寫入操作** (編輯/編輯檔案、新增): 中等複雜度，準確性重要
- **執行工具** (執行命令、執行測試、執行任務): 驗證需求，迭代方法
- **進階工具** (context7/	extit{}, sequential-thinking/	extit{})
- **多模態** (圖像分析參考): 需要具備視覺功能模型

      └─ 平衡 → GPT-4.1, Claude Sonnet 4, GPT-5

### 3. 推薦生成階段

**主要推薦**:

- 根據任務分析和決策樹識別單一最佳模型
- 提供與檔案內容特性相關的具體理由
- 解釋乘數成本對使用者訂閱層級的影響

**替代推薦**:

- 建議 1-2 個替代模型並解釋權衡
- 包含可能偏好替代方案的情境
- 考慮優先級因素覆蓋 (速度 vs. 品質 vs. 成本)

**自動選擇指南**:

- 評估任務是否適合自動模型選擇 (排除高級模型 > 1x)
- 解釋手動選擇何時有利於讓 Copilot 選擇
- 注意特定任務自動選擇的任何限制

**棄用警告**:

- 標記檔案是否目前指定了已棄用模型 (o3, o4-mini, Claude Sonnet 3.7, Gemini 2.0 Flash)
- 提供遷移路徑到推薦的替代方案
- 包含棄用時間表 (例如: "o3 將於 2025-10-23 棄用")

**訂閱層級考量**:

- **免費層級**: 僅推薦 0x 乘數模型 (GPT-4.1, GPT-5 mini, Grok Code Fast 1)
- **專業版層級**: 平衡 0x (無限制) 和 1x (每月 1000 次) 模型
- **專業版+ 層級**: 1x 模型有更多自由 (每月 5000 次)，為特殊情況證明 10x 使用的合理性

### 成功標準

- 推薦由特定檔案特性證明
- 成本影響清晰且適合訂閱層級
- 替代模型涵蓋不同的優先級因素 (速度 vs. 品質 vs. 成本)
- 前置內容更新已準備好複製貼上 (無佔位符)
- 使用者可以立即根據推薦採取行動 (清晰步驟)
- 報告可讀且可掃描 (良好結構、表格、表情符號標記)

### 失敗觸發器

- 檔案路徑無效或不可讀 → 停止並請求有效路徑
- 檔案不是 `.agent.md` 或 `.prompt.md` → 停止並澄清檔案類型
- 無法從內容確定任務複雜度 → 請求更具體的檔案或澄清
- 模型推薦與文件功能矛盾 → 使用 Context7 驗證目前資訊
- 訂閱層級無效 (非免費/專業版/專業版+) → 預設為專業版並註明假設

## 進階使用案例

### 分析多個檔案

如果使用者提供多個檔案:

1. 單獨分析每個檔案
2. 為每個檔案生成單獨的推薦
3. 提供比較推薦的摘要表格
4. 注意任何模式 (例如: "所有與偵錯相關的模式都受益於 Claude Sonnet 4.5")

### 比較分析

如果使用者詢問 "此檔案 X 和 Y 哪個模型更好？":

1. 僅專注於這兩個模型的比較
2. 使用並排表格格式
3. 宣布獲勝者並附帶具體理由
4. 包含訂閱層級的成本比較

### 遷移規劃

如果檔案指定了已棄用模型:

1. 在報告中優先提供遷移指南
2. 測試目前行為預期與替代模型功能
3. 如果預期有重大變更，提供分階段遷移
4. 如果需要，包含回滾計畫

## 範例

### 範例 1: 簡單格式化任務

**檔案**: `format-code.prompt.md`
**內容**: "使用 Black 風格格式化 Python 程式碼，添加類型提示"
**推薦**: GPT-5 mini (0x 乘數，最快，足以應付重複格式化)
**替代**: Grok Code Fast 1 (0.25x，更快，預覽功能)
**理由**: 任務簡單重複；不需要高級推理；速度優先

### 範例 2: 複雜架構審查

**檔案**: `architect.agent.md`
**內容**: "審查系統設計的可擴展性、安全性、可維護性；分析權衡；提供 ADR 級別的建議"
**推薦**: Claude Sonnet 4.5 (1x 乘數，專家推理，非常適合架構)
**替代**: Claude Opus 4.1 (10x，用於超大型程式碼庫 >500K tokens)
**理由**: 需要深度推理、架構專業知識、設計模式知識；Sonnet 4.5 在這方面表現出色

### 範例 3: Django 專家模式

**檔案**: `django.agent.md`
**內容**: "Django 5.x 專家，具備 ORM 優化、非同步視圖、REST API 設計；使用 context7 獲取最新的 Django 文件"
**推薦**: GPT-5 (1x 乘數，進階推理，優秀的程式碼品質)
**替代**: Claude Sonnet 4.5 (1x，替代視角，對框架強大)
**理由**: 領域專業知識 + context7 整合受益於進階推理；1x 成本對於專家模式是合理的

### 範例 4: 免費層級使用者與規劃模式

**檔案**: `plan.agent.md`
**內容**: "具有只讀工具 (搜尋、擷取、githubRepo) 的研究和規劃模式"
**訂閱**: 免費 (2K 完成次數 + 每月 50 次聊天請求，僅限 0x 模型)
**推薦**: GPT-4.1 (0x，平衡，包含在免費層級)
**替代**: GPT-5 mini (0x，更快但上下文較少)
**理由**: 免費層級僅限 0x 模型；GPT-4.1 為規劃任務提供品質和上下文的最佳平衡

### 模型乘數成本參考

| 乘數  | 意義                                | 免費層級 | 專業版使用  | 專業版+ 使用 |
| ----- | ----------------------------------- | -------- | ----------- | ------------ |
| 0x    | 包含在所有方案中，無高級計數        | ✅        | 無限制      | 無限制       |
| 0.25x | 4 個請求 = 1 個高級請求             | ❌        | 4000 次使用 | 20000 次使用 |
| 0.33x | 3 個請求 = 1 個高級請求             | ❌        | 3000 次使用 | 15000 次使用 |
| 1x    | 1 個請求 = 1 個高級請求             | ❌        | 1000 次使用 | 5000 次使用  |
| 1.25x | 1 個請求 = 1.25 個高級請求          | ❌        | 800 次使用  | 4000 次使用  |

### 模型變更日誌與棄用 (2025 年 10 月)

- ❌ o4-mini (0.33x) → 替換為 GPT-5 mini (0x) 以降低成本，GPT-5 (1x) 以提高品質
- ❌ Claude Sonnet 3.7 (1x) → 替換為 Claude Sonnet 4 或 4.5
- ❌ Claude Sonnet 3.7 Thinking (1.25x) → 替換為 Claude Sonnet 4.5
- ❌ Gemini 2.0 Flash (0.25x) → 替換為 Grok Code Fast 1 (0.25x) 或 GPT-5 mini (0x)

- 🧪 Grok Code Fast 1 (0.25x) - 預覽，預覽期間免費

**穩定生產模型**:

- ✅ GPT-4.1, GPT-5, GPT-5 mini, GPT-5 Codex (OpenAI)
- ✅ Claude Sonnet 3.5, Claude Sonnet 4, Claude Opus 4.1 (Anthropic)

### 自動模型選擇行為 (2025 年 9 月+)

**包含在自動選擇中**:

- GPT-4.1 (0x)
- GPT-5 mini (0x)
- GPT-5 (1x)
- Claude Sonnet 3.5 (1x)
- Claude Sonnet 4.5 (1x)

**從自動選擇中排除**:

- 乘數 > 1 的模型 (Claude Opus 4.1, 已棄用 o3)
- 被管理員策略阻止的模型
- 訂閱方案中不可用的模型 (免費層級中的 1x 模型)

**自動選擇時**:

- Copilot 分析提示複雜度、上下文大小、任務類型
- 根據可用性和速率限制從合格池中選擇
- 對自動選擇的模型應用 10% 乘數折扣
- 在聊天視圖中將滑鼠懸停在回應上時顯示選定的模型
