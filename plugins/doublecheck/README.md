# Doublecheck

針對 AI 生成內容的三層驗證流程。擷取可驗證的主張、透過網路搜尋尋找來源、針對幻覺模式執行對抗性審核，並產生包含來源連結的結構化報告，以便人類在採取行動前進行驗證。

## 為何存在

AI 幻覺是模型層級的問題。沒有外掛程式可以修復它們。但幻覺的**後果** —— 根據虛構的引用採取行動、依賴編造的統計數據、引用不存在的判例法 —— 可以透過讓驗證變得快速且結構化來減輕。

Doublecheck 不會告訴您什麼是真實的。它會從 AI 輸出中擷取每一項可驗證的主張，搜尋您可以獨立檢查的來源，並標記任何符合已知幻覺模式的內容。最後由您做出決定。

## 包含內容

| 元件 | 類型 | 描述 |
|-----------|------|-------------|
| `doublecheck` | 技能 | 核心驗證流程。執行三層並產生結構化報告。 |
| `Doublecheck` | 代理程式 | 用於後續問題和更深入調查的互動式驗證模式。 |

## 三個層級

**第一層：自我稽核。** 批判性地重新閱讀目標文字。擷取每一項可驗證的主張（事實、統計數據、引用、日期、因果斷言）。檢查內部矛盾。為下游驗證對主張進行分類。

**第二層：來源驗證。** 針對每一項擷取的主張，執行網路搜尋以尋找支持或反駁的證據。提供可點擊的 URL 供獨立的人工審閱。對引用進行額外審查，因為引用是幻覺風險最高的類別。

**第三層：對抗性審核。** 完全切換立場 —— 假設輸出包含錯誤並主動嘗試尋找它們。根據幻覺模式檢核表進行檢查：虛構的引用、無來源的統計數據、對不確定話題表現出的自信具體性、時間混淆、過度概括以及缺少限定詞。

## 信心評等

每項主張都會得到一個最終評等：

| 評等 | 意義 |
|--------|---------|
| 已驗證 (VERIFIED) | 已找到並連結支持來源 |
| 似是而非 (PLAUSIBLE) | 與一般知識一致，未找到具體來源 |
| 未驗證 (UNVERIFIED) | 無法找到支持或反駁的證據 |
| 有爭議 (DISPUTED) | 從可靠來源找到反駁證據 |
| 虛構風險 (FABRICATION RISK) | 符合幻覺模式（例如：各處都找不到的引用） |

## 使用方式

### 持續模式（「始終開啟」）

啟用 doublecheck 模式，它將在您的後續對話中保持開啟。Copilot 的每一個實質回應都將在底部包含一個內嵌驗證摘要 —— 針對每一項事實主張提供信心評等和來源連結。

要啟用，只需說：

```
use doublecheck
```

一旦啟用：
- 簡單的事實查詢和單一主張的回答會自動獲得內嵌驗證摘要
- 事實分析、法律分析、法規解釋、合規指引以及包含案例引用或法規參考的內容，將自動獲得完整的驗證報告而非內嵌摘要
- 如果在內嵌驗證期間有任何主張評等為「有爭議」(DISPUTED) 或「虛構風險」(FABRICATION RISK)，則會自動產生完整報告
- 程式碼、創意寫作和閒談將被跳過（不適用驗證）
- 您可以對任何回應要求完整的深入驗證，只需說「完整報告」(full report)（或舊稱「執行完整驗證」(run full verification)）
- 隨時可以使用「關閉 doublecheck」(turn off doublecheck) 來關閉它

隨時關閉它：

```
turn off doublecheck
```

這是針對準確性至關重的工作階段（法律研究、合規分析、法規指引、主管簡報）的建議模式。

### 單次驗證

如果您不想要持續模式，可以根據需求驗證特定文字：

```
use doublecheck to verify: [貼上您想要檢查的文字]
```

這將執行完整的三層流程，並產生詳細的驗證報告，其中包含擷取的每一項主張、評等和來源。

### 互動式代理程式模式

用於對話式的來回討論：

```
@doublecheck [貼上文字或描述您想要驗證的內容]
```

代理程式模式讓您可以：
- 取得完整的驗證報告
- 針對特定的標記主張提出後續問題
- 要求更深入的調查（「深入研究 C3」）
- 取得評估來源是否可靠的協助

### 何時使用

- 在根據 AI 生成的法律分析、案例引用或法規指引採取行動之前
- 在將 AI 生成的統計數據或資料點納入文件之前
- 在審閱將與客戶、領導層或外部各方分享的 AI 輸出時
- 在錯誤會帶來實際後果的領域（法律、醫療、財務、安全）工作時
- 任何當您想到「我可能應該加倍檢查這個」的時候

### 何時不該使用

- 用於「準確性」並非目標的創意或主觀內容
- 用於程式碼審查（請改用程式碼特定的審查工具）
- 作為領域專業知識的替代品 —— 該工具協助您更快驗證，它不能取代對該領域的瞭解

## 限制

請注意此工具無法做到的事情：

- **相同的模型，相同的偏見。** 驗證流程使用的模型類型與可能產生原始輸出的模型相同。它可以捕捉到許多問題 —— 特別是像缺失引用這樣的結構性模式 —— 但它具有相同的基本知識限制。
- **網路搜尋並不全面。** 付費內容、近期發表的資料以及利基資料庫可能不會出現在搜尋結果中。主張「未驗證」可能意味著它在付費牆後，而不代表它是錯誤的。
- **「已驗證」(VERIFIED) 意味著「已找到來源」，不代表「絕對正確」。** 來源本身也可能出錯、過時或被誤解。支持連結加速了您的驗證過程；它並未完成它。
- **該工具無法捕捉它不知道自己不知道的事。** 如果一個幻覺足夠精妙，足以通過所有三個層級，那麼人類專家就是您的最後一道防線。

誠實的框架：此工具提高了驗證品質的下限，並大幅減少了識別需要人類關注的主張所需的時間。它並未提高上限。關鍵決策應始終涉及人類領域專業知識。
