# Roundup

狀況簡報產生器，能學習您的溝通方式。

Roundup 觀察您在 GitHub、電子郵件、Teams、Slack 和其他工具上的工作，然後以您自己的語氣為您定義的任何對象草擬狀態更新和簡報。

## 問題所在

經理和團隊負責人每週花費數小時從分散的來源彙整狀態更新：瀏覽 GitHub 中的 PR 活動、閱讀電子郵件串以獲取決策、檢查 Teams 或 Slack 以獲取背景資訊，然後針對不同對象以不同詳細程度重寫所有內容。

撰寫這些更新所需的資料已經存在，工作量在於蒐集和包裝。Roundup 將蒐集與草擬自動化。您只需進行品質控管並點擊發送即可。

## 如何運作

**首次使用 (5 分鐘)：** 執行 setup 技能。它會詢問您的角色、目標對象以及團隊工作發生的位置。然後，貼上您已經寫過的更新範例。從這些範例中，它會學習您的格式、語氣、結構，以及您包含或略過哪些內容。

**之後每次使用：** 告訴 Roundup 您要為誰撰寫。它會從您設定的來源拉取最新資料、進行綜合整理，並草擬一份符合您風格的簡報。

## 安裝

```bash
copilot plugin install roundup@awesome-copilot
```

## 開始使用

### 首次設定

```
use roundup-setup
```

設定流程會引導您完成：
- 描述您的角色與團隊
- 貼上您已經寫過的更新範例
- 定義您的目標對象 (領導層、團隊、合作夥伴等) 以及他們關心的內容
- 識別您的團隊工作與對話發生的地方

### 產生簡報

```
use roundup
```

或者更明確一點：

```
use roundup -- 產生一份涵蓋過去一週的領導層簡報
```

```
use roundup -- 草擬一份自週一以來的團隊更新
```

## 包含內容

| 技能 | 功能 |
|-------|-------------|
| `roundup-setup` | 互動式入門，學習您的風格、目標對象與資料來源 |
| `roundup` | 依需求從您的設定產生簡報草稿 |

## 它可以從哪裡拉取資料

Roundup 可適應您環境中可用的任何工具：

| 來源 | 尋找內容 |
|--------|-------------------|
| GitHub | 您已設定之儲存庫中的 PR、issue、提交、審查活動 |
| M365 / WorkIQ | 電子郵件串、Teams 訊息、行事曆事件、共用文件 |
| Slack | 頻道訊息、串、公告 |
| Google Workspace | Gmail、行事曆、Drive 活動 |

在設定期間，Roundup 會發現您可以存取哪些工具並進行相應設定。如果您稍後添加新工具，請重新執行設定以將其包含進來。

除了您在 Copilot CLI 環境中已經設定的內容外，無需 API 金鑰或額外設定。

## 它的獨特之處

**它從您的範例中學習。** Roundup 不會強迫您套用範本，而是分析您的實際寫作方式並複製該風格。您的簡報聽起來就像是您自己寫的一樣。

**它適用於您擁有的任何工具。** 只有 GitHub？沒問題。完整的 M365 套件？很好。Slack 和 Google 的混合？也可以運作。設定流程會發現您的環境並進行調整。

**多種對象，一次設定。** 定義具有不同詳細程度和格式的不同對象設定檔。相同的基礎資料，針對每個對象產生不同的輸出。

**您的設定檔是可讀的檔案。** Roundup 所學到的一切都存在於 `~/.config/roundup/config.md` 中 — 您可以隨時開啟、閱讀和手動編輯的純 Markdown 檔案。沒有不透明的設定或隱藏狀態。

## 重新設定

若要從頭開始重新執行設定：

```
use roundup-setup
```

它會在覆寫您現有的設定之前詢問您。

若要進行微調，只需在任何文字編輯器中開啟 `~/.config/roundup/config.md`。該檔案以純英文編寫並按章節組織。您的變更會在下次執行時生效。
